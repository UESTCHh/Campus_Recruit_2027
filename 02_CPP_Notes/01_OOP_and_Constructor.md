## 一、 必备的大厂规范：头文件防卫式声明 (Header Guards)

在编写任何类的头文件（`.h`）时，为了防止该头文件被同一个 `.cpp` 文件重复包含（会导致多次定义的编译错误），必须加上防卫式声明。

### 💻 标准写法
```cpp
#ifndef __COMPLEX__
#define __COMPLEX__

class Complex {
    // 类的完整声明...
};

#endif // __COMPLEX__
```
#### 💡 核心要点： 
* 哪怕这个头文件只有一行代码，这三句 #ifndef、#define、#endif 也绝不能省！

## 二、 构造函数的底裤：初始化列表 (Initialization List)

在编写 `Complex` 类的构造函数时，存在两种写法。面试时如果写错，会被直接认为 C++ 基础不扎实。

### 1. 规范写法（大厂强制要求）
```cpp
class Complex {
public:
    // 采用初始化列表
    Complex(double r = 0, double i = 0) : re(r), im(i) { } 
private:
    double re, im;
};
```
### 2. 业余写法（极其容易被喷）
```cpp
class Complex {
public:
    Complex(double r = 0, double i = 0) { 
        re = r; // 这是赋值 (Assignment)，不是初始化！
        im = i; 
    } 
private:
    double re, im;
};
```
#### 💡 面试核心考点：为什么必须用初始化列表？
从底层效率来看，一个对象的创建分为两步：“初始化”和“赋值”。

* 初始化列表是在对象分配内存的瞬间，直接把值塞进去（一步到位）。

* 在大括号内赋值，编译器会先调用成员变量的默认构造函数（浪费一次时间），然后再执行赋值操作（又浪费一次时间）。对于内置类型（如 double）差别不大，但如果成员变量是庞大的自定义类，性能损耗极其严重！

## 三、 铁律：防守严密的 const 成员函数与 inline 内联
### 1. const 成员函数
在设计获取实部和虚部的函数时，必须在函数括号后加上 const：

```cpp
double real() const { return re; }
double imag() const { return im; }
```
#### 💡 面试核心考点：不加 const 会引发什么惨案？
const 在这里是向编译器发誓：“我这个函数绝对不会修改类里面的任何数据”。
假设我不加 const，而在外界使用时写了如下代码：

```cpp
const Complex c1(2, 1); // 用户创建了一个常量复数，明确表示不想被修改
cout << c1.real();      // 报错！
```
* 报错原因： 编译器看到 c1 是常量，而你的 real() 并没有加上 const 保证不修改数据。编译器为了安全起见，会直接拒绝调用，导致编译失败。
* 结论： 只要成员函数不改变对象的数据，必须、一定、绝对要加 const。
### 2. inline 内联函数
* 函数在类定义内部直接实现（如上面的 real()），它就会自动成为候选的 inline 内联函数。内联函数在编译时会像宏一样展开，极大地提升了执行效率（免去了函数调用的开销）。

## 四、 参数传递与返回值：值传递 vs 引用传递
在设计复数加法等操作时，参数通常写成 const Complex& 而不是 Complex。

```cpp
Complex& operator+=(const Complex& r); // 优秀的写法
```
#### 💡 面试核心考点：为什么要传引用？为什么又要加 const？
* 传引用的本质是为了“快”：传值（Pass by value）会把整个对象完整地复制一份压入函数栈。如果对象很大（比如包含了几万个数据的类），极其耗时。而传引用（Pass by reference）底层其实就是传指针，无论对象多大，传的永远只是那几个字节的地址，速度极快。

* 加 const 是为了“安全”：传引用虽然快，但把原本的地址暴露出去了，万一函数内部手贱修改了原本的数据怎么办？加上 const 修饰符，就完美实现了：既有 C 语言指针的极速执行效率，又有极高的安全性（只读不写）。

#### 📌 总结法则：
* 所有的参数传递，尽量传引用。如果怕被改，就加 const。

* 函数的返回值，能传引用就传引用。但如果返回的是函数内部新创建的局部变量（Local object），绝对不能传引用（因为函数结束局部变量就销毁了，传回去的是悬空引用/野指针），必须老老实实传值。
