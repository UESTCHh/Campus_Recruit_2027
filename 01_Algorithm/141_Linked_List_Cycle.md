# 算法实战复盘：环形链表 (LeetCode 141)

> **打卡日期**：2026-03-31 (Day 5 上午)
> **核心主题**：快慢指针 (双指针法)、龟兔赛跑算法、防死循环防御。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给你一个链表的头节点 `head`，判断链表中是否有环。
* **核心痛点：** 如果链表有环，使用普通的 `while (cur != nullptr)` 遍历会永远陷入死循环，导致程序崩溃。
* **大厂面试限制：** 不允许使用哈希表（`std::unordered_set`）来记录访问过的节点（拒绝空间复杂度 $O(n)$ 的投机取巧），必须使用 $O(1)$ 的内存空间。

---

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n)$
**推导过程：** * 如果链表没有环，快指针（每次走 2 步）会以 $O(n)$ 的时间到达终点。
* 如果链表有环，快慢指针都会进入环。由于快指针相对于慢指针的速度差是 1（每次多走一步），这就好比快指针在环里以 1 的速度去追击慢指针。最坏情况下，快指针需要绕环一圈才能追上，耗时依然是线性的 $O(n)$。

### 2. 空间复杂度：$O(1)$
**推导过程：** 全程只开辟了 `fast` 和 `slow` 两个指针变量，没有任何随数据规模增长的动态空间，严格满足大厂的 $O(1)$ 内存要求。

---

## 🎯 三、 核心思想：龟兔赛跑 (Floyd 判圈算法)

想象在一个环形操场上跑步。如果操场是直的（无环），跑得快的人会直接冲过终点线；如果操场是环形的，跑得快的人最终一定会从后面“套圈”追上跑得慢的人。

* **慢指针 (`slow`)**：每次向前走 **1** 步。
* **快指针 (`fast`)**：每次向前走 **2** 步。
* **判定条件：** * 如果 `fast` 或者 `fast->next` 遇到了 `nullptr`，说明跑到了尽头，**绝对无环**。
  * 如果 `fast == slow`，说明快指针套圈成功，**绝对有环**！

---

## 💻 四、 核心代码

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 * int val;
 * ListNode *next;
 * ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    bool hasCycle(ListNode *head) {
        // 边界防御：空链表或只有一个节点且无环
        if (head == nullptr || head->next == nullptr) {
            return false;
        }

        ListNode* slow = head;
        ListNode* fast = head;

        // 快指针不仅自己不能是空，它的下一个也不能是空（否则 fast->next->next 会段错误）
        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;          // 慢指针走一步
            fast = fast->next->next;    // 快指针走两步

            // 龟兔相遇，证明有环
            if (slow == fast) {
                return true;
            }
        }

        // 快指针冲出了跑道，说明是直道（无环）
        return false;
    }
};
```

## 💡 五、 面试直击灵魂的防暴击拷问
* **灵魂拷问：** 为什么快指针每次走 2 步，慢指针走 1 步？快指针每次走 3 步、4 步不行吗？
    * **绝杀回答：**“之所以选择 2 步和 1 步，是为了保证快指针相对于慢指针的相对速度恒定为 1。这意味着在环里，快指针是一步一步地逼近慢指针的，绝对不可能发生‘跳过’慢指针的情况，它们必然会相遇。如果快指针每次走 3 步（相对速度为 2），在环长为偶数等特定情况下，快指针可能会一直在慢指针的前后横跳，完美错过，从而导致死循环。”
