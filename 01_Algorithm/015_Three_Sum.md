# 算法实战复盘：三数之和 (LeetCode 15)

> **打卡日期**：2026-04-02
> **核心主题**：排序、双指针、去重策略。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。请你返回所有和为 0 且不重复的三元组。
* **核心痛点：** 暴力解法是三层 `for` 循环，时间复杂度 $O(n^3)$，且需要处理重复结果。
* **大厂面试限制：** 要求必须在 $O(n^2)$ 的时间复杂度内解决，并确保结果不重复。
* **题目提示：**
  - 3 <= nums.length <= 3000
  - -10^5 <= nums[i] <= 10^5

---

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n^2)$
**推导过程：** 首先对数组进行排序，时间复杂度为 $O(n og n)$。然后使用一层 `for` 循环固定第一个元素，内部使用双指针遍历剩余元素，时间复杂度为 $O(n^2)$。整体时间复杂度由双指针部分主导，为 $O(n^2)$。

### 2. 空间复杂度：$O(1)$ 或 $O(log n)$
**推导过程：** 排序过程需要的空间复杂度取决于具体的排序算法，通常为 $O(log n)$。除此之外，我们只使用了常数级别的额外空间来存储结果和临时变量，因此空间复杂度为 $O(1)$ 或 $O(log n)$。

## 🤔 三、 解题思路分析

### 3.1 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：三层 `for` 循环，时间复杂度 O(n³)
2. **哈希表法**：使用哈希表存储元素，时间复杂度 O(n²)
3. **排序 + 双指针法**：先排序，然后使用双指针，时间复杂度 O(n²)

### 3.2 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n³) | O(1) | 思路简单，不需要额外空间 | 时间复杂度极高，对于 n=3000 时操作次数达到 2.7×10^10，直接超时 |
| 哈希表法 | O(n²) | O(n) | 时间复杂度合理 | 去重复杂，容易出错 |
| 排序 + 双指针法 | O(n²) | O(1) 或 O(logn) | 时间复杂度合理，去重容易 | 需要排序，可能改变原数组顺序 |

### 3.3 数据大小对方法选择的影响

根据题目提示，数组长度 n 最大为 3000：

- **暴力法**：对于 n=3000，操作次数为 3000³ = 27,000,000,000，这会导致严重超时
- **哈希表法**：对于 n=3000，操作次数为 3000² = 9,000,000，效率较高
- **排序 + 双指针法**：对于 n=3000，操作次数为 3000² = 9,000,000，效率较高

**实际测试验证：**
- 暴力法：在 LeetCode 上会直接超时
- 哈希表法：可以通过，但去重逻辑复杂
- 排序 + 双指针法：执行时间短，去重逻辑简单

### 3.4 方法选择建议

**推荐使用排序 + 双指针法**，原因如下：
1. 时间复杂度为 O(n²)，能够处理最大规模的输入
2. 去重逻辑简单直观，不容易出错
3. 空间复杂度低，不需要额外的哈希表空间
4. 代码实现简洁，易于理解和维护

## 🎯 四、 核心思想：排序 + 双指针，去重是关键

我们的目标是找到三个数 $a + b + c = 0$，可以转化为 $a + b = -c$。

### 4.1 算法步骤详解

1. **排序：** 首先对数组进行排序，这样可以方便地使用双指针，并且更容易处理重复元素。
2. **固定第一个元素：** 遍历数组，固定第一个元素 `nums[i]`，然后在剩余的元素中寻找和为 `-nums[i]` 的两个数。
3. **双指针：** 使用左指针 `left = i + 1` 和右指针 `right = nums.size() - 1`，在剩余区间内寻找符合条件的组合。
4. **去重策略：**
   - 对第一个元素去重：如果 `nums[i] == nums[i-1]`，则跳过，避免重复结果。
   - 对左右指针去重：当找到符合条件的组合后，移动指针时跳过相同的元素，避免重复结果。

### 4.2 核心原理

- **排序的作用：** 排序后，相同的元素会相邻，方便去重；同时，双指针可以根据当前和的大小调整移动方向。
- **双指针的移动策略：** 当和小于目标值时，左指针右移；当和大于目标值时，右指针左移；当和等于目标值时，记录结果并同时移动左右指针。
- **去重的重要性：** 题目要求返回不重复的三元组，因此必须在各个层面进行去重处理。

### 4.3 示例分析

以示例 1 为例：`nums = [-1, 0, 1, 2, -1, -4]`

**步骤 1：排序**
- 排序后：`[-4, -1, -1, 0, 1, 2]`

**步骤 2：遍历固定第一个元素**
- i=0，nums[i]=-4：
  - target=4，left=1, right=5
  - nums[left]+nums[right] = -1+2=1 < 4，left++
  - nums[left]+nums[right] = -1+2=1 < 4，left++
  - nums[left]+nums[right] = 0+2=2 < 4，left++
  - nums[left]+nums[right] = 1+2=3 < 4，left++
  - left >= right，结束

- i=1，nums[i]=-1：
  - 去重：nums[1] != nums[0]，继续
  - target=1，left=2, right=5
  - nums[left]+nums[right] = -1+2=1 == 1，记录结果 [-1, -1, 2]
  - 去重：left++ 到 3，right-- 到 4
  - nums[left]+nums[right] = 0+1=1 == 1，记录结果 [-1, 0, 1]
  - 去重：left++ 到 4，right-- 到 3，结束

- i=2，nums[i]=-1：
  - 去重：nums[2] == nums[1]，跳过

- i=3，nums[i]=0：
  - target=0，left=4, right=5
  - nums[left]+nums[right] = 1+2=3 > 0，right--
  - left >= right，结束

**最终结果：** `[[-1, -1, 2], [-1, 0, 1]]`

### 4.4 边界情况处理

1. **数组长度小于 3**：直接返回空数组
2. **第一个元素大于 0**：由于数组已排序，后面的元素都大于 0，不可能形成和为 0 的三元组，直接返回
3. **重复元素**：需要在固定第一个元素和移动双指针时进行去重
4. **数值溢出**：由于题目中 nums[i] 的范围是 -10^5 到 10^5，三个数的和不会超过 3*10^5，不会溢出

### 4.5 与其他方法的对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n³) | O(1) | 思路简单 | 时间复杂度极高，无法处理大输入 |
| 哈希表法 | O(n²) | O(n) | 不需要排序 | 去重复杂，代码实现较困难 |
| 排序 + 双指针法 | O(n²) | O(1) 或 O(logn) | 去重简单，代码简洁 | 需要排序，可能改变原数组顺序 |

### 4.6 关键技术点

1. **排序**：为双指针和去重创造条件
2. **双指针**：将时间复杂度从 O(n³) 降到 O(n²)
3. **去重**：确保结果中没有重复的三元组
4. **剪枝**：当第一个元素大于 0 时，直接返回结果

## 💻 五、 核心代码

### 5.1 排序 + 双指针法实现

```cpp
#include <vector>
#include <algorithm>

class Solution {
public:
    std::vector<std::vector<int>> threeSum(std::vector<int>& nums) {
        std::vector<std::vector<int>> result;
        int n = nums.size();
        
        // 1. 排序
        std::sort(nums.begin(), nums.end());
        
        // 2. 遍历固定第一个元素
        for (int i = 0; i < n; i++) {
            // 剪枝：如果当前元素大于 0，后面的元素都大于 0，不可能形成和为 0 的三元组
            if (nums[i] > 0) {
                return result;
            }
            
            // 去重：如果当前元素与前一个元素相同，跳过
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }
            
            // 3. 双指针寻找剩余两个元素
            int target = -nums[i];
            int left = i + 1;
            int right = n - 1;
            
            while (left < right) {
                int sum = nums[left] + nums[right];
                
                if (sum == target) {
                    // 找到符合条件的三元组
                    result.push_back({nums[i], nums[left], nums[right]});
                    
                    // 去重：移动左指针，跳过相同元素
                    while (left < right && nums[left] == nums[left + 1]) {
                        left++;
                    }
                    // 去重：移动右指针，跳过相同元素
                    while (left < right && nums[right] == nums[right - 1]) {
                        right--;
                    }
                    
                    // 移动指针，继续寻找
                    left++;
                    right--;
                } else if (sum < target) {
                    // 和小于目标值，左指针右移
                    left++;
                } else {
                    // 和大于目标值，右指针左移
                    right--;
                }
            }
        }
        
        return result;
    }
};
```

**代码解释：**
1. **排序**：对数组进行排序，为双指针和去重创造条件
2. **剪枝**：当第一个元素大于 0 时，直接返回结果
3. **去重**：跳过与前一个元素相同的元素，避免重复结果
4. **双指针**：在剩余区间内寻找和为 -nums[i] 的两个元素
5. **记录结果**：当找到符合条件的组合时，记录结果并去重

### 5.2 暴力法实现（仅供参考，实际会超时）

```cpp
#include <vector>
#include <algorithm>

class Solution {
public:
    std::vector<std::vector<int>> threeSum(std::vector<int>& nums) {
        std::vector<std::vector<int>> result;
        int n = nums.size();
        
        // 排序，便于去重
        std::sort(nums.begin(), nums.end());
        
        // 三层循环遍历所有可能的组合
        for (int i = 0; i < n; i++) {
            // 去重：跳过相同的元素
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }
            
            for (int j = i + 1; j < n; j++) {
                // 去重：跳过相同的元素
                if (j > i + 1 && nums[j] == nums[j - 1]) {
                    continue;
                }
                
                for (int k = j + 1; k < n; k++) {
                    // 去重：跳过相同的元素
                    if (k > j + 1 && nums[k] == nums[k - 1]) {
                        continue;
                    }
                    
                    if (nums[i] + nums[j] + nums[k] == 0) {
                        result.push_back({nums[i], nums[j], nums[k]});
                    }
                }
            }
        }
        
        return result;
    }
};
```

**暴力法性能分析：**
- 时间复杂度：O(n³)，其中 n 是数组的长度
- 空间复杂度：O(1)，不需要额外空间
- 适用场景：数据规模很小，对性能要求不高的情况

### 5.3 哈希表法实现

```cpp
#include <vector>
#include <unordered_set>
#include <algorithm>

class Solution {
public:
    std::vector<std::vector<int>> threeSum(std::vector<int>& nums) {
        std::vector<std::vector<int>> result;
        int n = nums.size();
        
        // 排序，便于去重
        std::sort(nums.begin(), nums.end());
        
        for (int i = 0; i < n; i++) {
            // 去重：跳过相同的元素
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }
            
            // 使用哈希表存储已经遍历过的元素
            std::unordered_set<int> seen;
            for (int j = i + 1; j < n; j++) {
                int target = -nums[i] - nums[j];
                if (seen.count(target)) {
                    result.push_back({nums[i], nums[j], target});
                    // 去重：跳过相同的元素
                    while (j + 1 < n && nums[j] == nums[j + 1]) {
                        j++;
                    }
                }
                seen.insert(nums[j]);
            }
        }
        
        return result;
    }
};
```

**哈希表法性能分析：**
- 时间复杂度：O(n²)，其中 n 是数组的长度
- 空间复杂度：O(n)，需要额外空间存储哈希表
- 适用场景：需要避免排序的情况

## ❌ 七、 常见错误及解决方案

### 7.1 常见错误

1. **去重错误**：
   - 错误：使用 `nums[i] == nums[i+1]` 进行去重
   - 后果：会漏掉正确的解，如 [-1, -1, 2]
   - 原因：这种去重方式会跳过当前元素，而不是跳过重复的元素

2. **双指针移动错误**：
   - 错误：找到符合条件的组合后，只移动一个指针
   - 后果：会导致死循环或重复结果
   - 原因：需要同时移动左右指针，继续寻找下一个组合

3. **剪枝不充分**：
   - 错误：没有在第一个元素大于 0 时直接返回
   - 后果：增加不必要的计算
   - 原因：排序后，后面的元素都大于 0，不可能形成和为 0 的三元组

4. **边界条件处理错误**：
   - 错误：没有处理数组长度小于 3 的情况
   - 后果：可能导致数组越界
   - 原因：题目要求至少三个元素

### 7.2 解决方案

1. **正确的去重方式**：
   - 使用 `nums[i] == nums[i-1]` 进行去重
   - 确保只有当当前元素与前一个元素相同时才跳过

2. **正确的双指针移动**：
   - 找到符合条件的组合后，同时移动左右指针
   - 在移动指针前，先跳过相同的元素

3. **充分的剪枝**：
   - 当第一个元素大于 0 时，直接返回结果
   - 减少不必要的计算

4. **边界条件处理**：
   - 在函数开始时检查数组长度
   - 确保数组长度至少为 3

## 💡 九、 面试直击灵魂的防暴击拷问

* **灵魂拷问一：为什么要先排序？**
    * **绝杀回答：**"排序有两个主要作用：1）方便使用双指针法，将时间复杂度从 $O(n^3)$ 降到 $O(n^2)$；2）便于去重，避免重复的三元组结果。排序后，相同的元素会相邻，我们可以通过比较当前元素与前一个元素是否相同来跳过重复值。"

* **灵魂拷问二：如何处理重复的三元组？**
    * **绝杀回答：**"去重需要从三个层面考虑：1）固定的第一个元素：如果 `nums[i] == nums[i-1]`，则跳过，避免重复；2）左指针：当找到符合条件的组合后，左指针需要跳过所有相同的元素；3）右指针：同理，右指针也需要跳过所有相同的元素。这样可以确保最终结果中没有重复的三元组。"

* **灵魂拷问三：双指针的移动策略是怎样的？**
    * **绝杀回答：**"当 `nums[left] + nums[right] < target` 时，说明和太小，需要增大，因此左指针右移；当 `nums[left] + nums[right] > target` 时，说明和太大，需要减小，因此右指针左移；当等于目标值时，记录结果并同时移动左右指针，继续寻找下一个可能的组合。"

* **灵魂拷问四：时间复杂度为什么是 $O(n^2)$？**
    * **绝杀回答：**"排序的时间复杂度是 $O(n og n)$，然后我们有一层 `for` 循环，每次循环内部使用双指针遍历剩余元素，双指针遍历的时间复杂度是 $O(n)$，因此整体时间复杂度是 $O(n^2)$。排序的时间复杂度相对于 $O(n^2)$ 可以忽略不计。"

* **💥 灵魂拷问五（致死级陷阱）：为什么第一个固定元素的去重必须写 nums[i] == nums[i-1]，写成 nums[i] == nums[i+1] 会有什么灾难性后果？**
    * **绝杀回答：**"写成 nums[i] == nums[i+1] 的意思是‘如果当前数字和后面的数字一样，我就直接跳过当前数字’。但这会漏掉正确答案！比如遇到数组 [-1, -1, 2] 且目标和为 0 时，我们理应用第一个 -1 配合第二个 -1 和 2 组成答案。如果按上述错误写法，遇到第一个 -1 时发现后面也是 -1，就会直接 continue 跳过，导致完美的解被漏掉！而写成 nums[i] == nums[i-1] 的意思是‘这个数字在上一轮当过车头了，这一轮不许再当车头’，这才是真正的去重逻辑。"

* **灵魂拷问六：除了排序 + 双指针法，还有其他方法吗？**
    * **绝杀回答：**"还有暴力法和哈希表法。暴力法是三层 `for` 循环，时间复杂度 $O(n^3)$，无法处理大输入。哈希表法是使用哈希表存储元素，时间复杂度 $O(n^2)$，但去重逻辑复杂，容易出错。相比之下，排序 + 双指针法是最优的选择，时间复杂度 $O(n^2)$，去重逻辑简单，代码实现简洁。"

## 📚 十、 总结

本题是一道经典的算法题，通过排序 + 双指针的方法，我们成功将时间复杂度从 $O(n^3)$ 优化到了 $O(n^2)$，并通过合理的去重策略确保了结果的唯一性。

### 核心要点：
1. **排序**：为双指针和去重创造条件
2. **双指针**：将时间复杂度从 $O(n^3)$ 降到 $O(n^2)$
3. **去重**：确保结果中没有重复的三元组
4. **剪枝**：当第一个元素大于 0 时，直接返回结果

### 扩展思考：
- **算法应用**：该算法思想可以扩展到四数之和、最接近的三数之和等问题
- **性能优化**：对于大规模数据，可以考虑并行计算和缓存优化
- **实际应用**：在金融分析、数据挖掘、游戏开发等领域有广泛应用

### 方法比较：

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n³) | O(1) | 思路简单 | 时间复杂度极高 |
| 哈希表法 | O(n²) | O(n) | 不需要排序 | 去重复杂 |
| 排序 + 双指针法 | O(n²) | O(1) 或 O(logn) | 去重简单，代码简洁 | 需要排序 |

通过本题的学习，我们不仅掌握了一种高效的算法，更重要的是理解了排序 + 双指针的思想和去重策略，为解决更复杂的问题打下了基础。
