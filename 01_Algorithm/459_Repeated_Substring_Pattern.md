## 1. 问题描述

### 1.1 中文描述

给定一个非空的字符串 `s`，检查是否可以通过由它的一个子串重复多次构成。

### 1.2 英文描述

Given a non-empty string `s`, check if it can be constructed by taking a substring of it and appending multiple copies of the substring together.

### 1.3 示例

**示例 1**：
- 输入：`s = "abab"`
- 输出：`true`
- 解释：可由子串 "ab" 重复两次构成。

**示例 2**：
- 输入：`s = "aba"`
- 输出：`false`

**示例 3**：
- 输入：`s = "abcabcabcabc"`
- 输入：`s = "abcabcabcabc"`
- 输出：`true`
- 解释：可由子串 "abc" 重复四次构成。 (或者子串 "abcabc" 重复两次构成。)

## 2. 解题思路分析

### 2.1 暴力解法

**思路**：
1. 枚举所有可能的子串长度 `len`，其中 `len` 是字符串长度 `n` 的约数
2. 对于每个可能的 `len`，检查是否可以通过重复该长度的子串得到原字符串

**步骤**：
1. 遍历可能的子串长度 `len`，从 1 到 `n/2`
2. 如果 `n` 能被 `len` 整除，则检查前 `len` 个字符组成的子串是否可以重复 `n/len` 次得到原字符串
3. 如果找到这样的子串，返回 `true`；否则，返回 `false`

**时间复杂度**：O(n²)，其中 n 是字符串长度
**空间复杂度**：O(1)

### 2.2 KMP 算法解法（最优解法）

**思路**：
利用 KMP 算法中的前缀函数（next 数组）来判断字符串是否由重复子串构成。

**核心原理**：
对于一个由重复子串构成的字符串 `s`，其最长相等前后缀的长度 `len` 满足：
- `s.length() % (s.length() - len) == 0`

**步骤**：
1. 计算字符串 `s` 的前缀函数数组 `next`
2. 获取最长相等前后缀的长度 `len = next.back()`
3. 如果 `len > 0` 且 `s.length() % (s.length() - len) == 0`，则返回 `true`；否则，返回 `false`

**时间复杂度**：O(n)，其中 n 是字符串长度
**空间复杂度**：O(n)，用于存储前缀函数数组

### 2.3 字符串拼接解法

**思路**：
如果字符串 `s` 由重复子串构成，那么将 `s` 与自身拼接后，去除首尾字符，得到的字符串一定包含 `s`。

**核心原理**：
假设 `s = t * k`（`t` 是子串，`k` 是重复次数），那么 `s + s = t * 2k`。去除首尾字符后，得到的字符串是 `t * (2k - 2) + t[1:] + t[:-1]`，其中一定包含 `t * k = s`。

**步骤**：
1. 将字符串 `s` 与自身拼接，得到 `s + s`
2. 去除首尾字符，得到 `new_s = (s + s).substr(1, 2n - 2)`
3. 检查 `new_s` 是否包含 `s`，如果包含，则返回 `true`；否则，返回 `false`

**时间复杂度**：O(n)，其中 n 是字符串长度
**空间复杂度**：O(n)，用于存储拼接后的字符串

### 2.4 解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力解法 | O(n²) | O(1) | 实现简单 | 时间复杂度高，对于长字符串效率低 |
| KMP 算法 | O(n) | O(n) | 时间复杂度低，效率高 | 需要理解 KMP 算法的前缀函数 |
| 字符串拼接法 | O(n) | O(n) | 实现简单，思路巧妙 | 需要额外的空间存储拼接后的字符串 |

## 3. 算法实现步骤

### 3.1 暴力解法实现

```cpp
class Solution {
public:
    bool repeatedSubstringPattern(string s) {
        int n = s.size();
        // 枚举可能的子串长度，从 1 到 n/2
        for (int len = 1; len <= n / 2; ++len) {
            // 检查 n 是否能被 len 整除
            if (n % len == 0) {
                bool flag = true;
                // 检查前 len 个字符组成的子串是否可以重复构成原字符串
                for (int i = len; i < n; ++i) {
                    if (s[i] != s[i % len]) {
                        flag = false;
                        break;
                    }
                }
                if (flag) {
                    return true;
                }
            }
        }
        return false;
    }
};
```

### 3.2 KMP 算法实现

```cpp
class Solution {
public:
    bool repeatedSubstringPattern(string s) {
        int n = s.size();
        vector<int> next(n, -1);
        
        // 计算前缀函数数组
        for (int i = 1; i < n; ++i) {
            int j = next[i - 1];
            // 回退 j 直到找到匹配的前缀或 j == -1
            while (j != -1 && s[j + 1] != s[i]) {
                j = next[j];
            }
            // 如果找到匹配的前缀，j 增加 1
            if (s[j + 1] == s[i]) {
                j += 1;
            }
            next[i] = j;
        }
        
        // 获取最长相等前后缀的长度
        int len = next[n - 1];
        // 判断是否满足条件
        return len != -1 && n % (n - len - 1) == 0;
    }
};
```

### 3.3 字符串拼接解法实现

```cpp
class Solution {
public:
    bool repeatedSubstringPattern(string s) {
        string doubled = s + s;
        string sub = doubled.substr(1, doubled.size() - 2);
        return sub.find(s) != string::npos;
    }
};
```

### 3.4 代码详解

**暴力解法**：
1. 遍历可能的子串长度 `len`，从 1 到 `n/2`
2. 对于每个 `len`，检查 `n` 是否能被 `len` 整除
3. 如果能整除，则检查前 `len` 个字符组成的子串是否可以重复构成原字符串
4. 检查方法是比较每个位置 `i` 的字符是否等于 `i % len` 位置的字符

**KMP 算法**：
1. 计算前缀函数数组 `next`，其中 `next[i]` 表示以 `i` 结尾的子串的最长相等前后缀的长度减一
2. 获取最长相等前后缀的长度 `len = next.back() + 1`（因为 `next` 数组存储的是长度减一）
3. 判断 `len > 0` 且 `n % (n - len) == 0`，如果满足，则返回 `true`

**字符串拼接法**：
1. 将字符串 `s` 与自身拼接，得到 `s + s`
2. 去除首尾字符，得到 `new_s = (s + s).substr(1, 2n - 2)`
3. 检查 `new_s` 是否包含 `s`，如果包含，则返回 `true`；否则，返回 `false`

## 4. 时间复杂度与空间复杂度分析

### 4.1 暴力解法

**时间复杂度**：O(n²)
- 外层循环遍历可能的子串长度，最多执行 n/2 次
- 内层循环检查子串是否可以重复构成原字符串，最多执行 n 次
- 总时间复杂度为 O(n²)

**空间复杂度**：O(1)
- 只使用了常数级别的额外空间，没有使用额外的数组或数据结构

### 4.2 KMP 算法

**时间复杂度**：O(n)
- 计算前缀函数数组的过程是线性的，每个字符最多被处理两次
- 后续的判断步骤是常数时间
- 总时间复杂度为 O(n)

**空间复杂度**：O(n)
- 需要存储长度为 n 的前缀函数数组
- 空间复杂度为 O(n)

## 7. 常见错误与注意事项

### 7.1 错误 1：子串长度的范围

**错误**：子串长度的范围设置错误，例如从 0 开始或超过 n/2。
**解决方案**：子串长度的范围应该是 1 到 n/2，因为子串至少需要重复一次。

### 7.2 错误 2：前缀函数的理解

**错误**：对 KMP 算法中的前缀函数理解错误，导致计算错误。
**解决方案**：前缀函数 `next[i]` 表示以 `i` 结尾的子串的最长相等前后缀的长度减一，需要正确计算。

### 7.3 错误 3：条件判断错误

**错误**：在 KMP 算法中，条件判断错误，例如使用 `n % (n - len) == 0` 而不是 `n % (n - len - 1) == 0`。
**解决方案**：正确理解前缀函数的含义，使用正确的条件判断。

### 7.4 错误 4：边界条件处理

**错误**：没有正确处理单字符字符串等边界情况。
**解决方案**：确保代码能够正确处理各种边界情况，例如单字符字符串、长度为 2 的字符串等。

## 16. 深入理解 KMP 算法的前缀函数

### 16.1 前缀函数的计算过程详解

以模式串 "AABAAF" 为例，详细说明前缀函数的计算过程：

| 索引 i | 子串 | s[i] | j (初始值) | 回退过程 | s[j] 与 s[i] 比较 | 最终 j | next[i] |
|--------|------|------|------------|----------|-------------------|--------|---------|
| 0      | "A"  | 'A'  | -1         | 无       | 无                | -1     | -1      |
| 1      | "AA" | 'A'  | next[0] = -1 | 无       | s[0] == s[1]      | 0      | 0       |
| 2      | "AAB" | 'B'  | next[1] = 0  | s[0] != s[2] | 不匹配             | -1     | -1      |
| 3      | "AABA" | 'A'  | next[2] = -1 | 无       | s[0] == s[3]      | 0      | 0       |
| 4      | "AABAA" | 'A'  | next[3] = 0  | 无       | s[1] == s[4]      | 1      | 1       |
| 5      | "AABAAF" | 'F'  | next[4] = 1  | s[2] != s[5] → j = next[1] = 0 → s[1] != s[5] → j = next[0] = -1 | 不匹配 | -1     | -1      |

### 16.2 前缀函数的几何意义

前缀函数可以看作是字符串的自相似性度量。对于字符串 `s`，`π[i]` 表示子串 `s[0..i]` 中最大的 k，使得前 k 个字符与后 k 个字符相同。

这种自相似性在模式匹配中非常重要，因为当匹配失败时，我们可以利用这种相似性来避免重复比较。

### 16.3 前缀函数的应用

除了用于 KMP 算法外，前缀函数还可以用于：

1. **计算字符串的周期**：如 LeetCode 459 问题
2. **查找最长重复子串**：通过遍历前缀函数数组，找到最大值
3. **字符串压缩**：基于重复模式的压缩
4. **构建自动机**：用于正则表达式匹配

### 16.4 前缀函数的优化

对于某些特殊的模式串，前缀函数的计算可以进一步优化：

1. **周期性模式**：如果模式串是周期性的，前缀函数的计算可以更快
2. **重复字符**：如果模式串由重复字符组成，前缀函数的计算可以简化
3. **随机模式**：对于随机模式，前缀函数的值通常较小

## 18. 暴力解法的优化

### 18.1 早期终止策略

暴力解法的一个重要优化是早期终止：

```cpp
bool repeatedSubstringPattern(string s) {
    int n = s.size();
    for (int len = 1; len <= n / 2; ++len) {
        if (n % len == 0) {
            bool flag = true;
            for (int i = len; i < n; ++i) {
                if (s[i] != s[i % len]) {
                    flag = false;
                    break; // 早期终止
                }
            }
            if (flag) {
                return true;
            }
        }
    }
    return false;
}
```

### 18.2 剪枝策略

对于某些情况，可以提前剪枝：

1. **长度检查**：只考虑能整除 n 的长度
2. **首字符检查**：如果子串的首字符与原字符串的首字符不同，直接跳过
3. **尾字符检查**：如果子串的尾字符与原字符串的尾字符不同，直接跳过

### 18.3 实际性能

虽然暴力解法的时间复杂度为 O(n²)，但在实际应用中，对于短字符串，它的性能可能与 KMP 算法相当，因为它的常数因子较小。

class Solution {
public:
    bool repeatedSubstringPattern(string s) {
        int n = s.size();
        // 枚举可能的子串长度，从 1 到 n/2
        for (int len = 1; len <= n / 2; ++len) {
            // 检查 n 是否能被 len 整除
            if (n % len == 0) {
                bool flag = true;
                // 检查前 len 个字符组成的子串是否可以重复构成原字符串
                for (int i = len; i < n; ++i) {
                    if (s[i] != s[i % len]) {
                        flag = false;
                        break;
                    }
                }
                if (flag) {
                    return true;
                }
            }
        }
        return false;
    }
};

### 25.2 KMP 算法

```cpp
class Solution {
public:
    bool repeatedSubstringPattern(string s) {
        int n = s.size();
        vector<int> next(n, -1);
        
        // 计算前缀函数数组
        for (int i = 1; i < n; ++i) {
            int j = next[i - 1];
            // 回退 j 直到找到匹配的前缀或 j == -1
            while (j != -1 && s[j + 1] != s[i]) {
                j = next[j];
            }
            // 如果找到匹配的前缀，j 增加 1
            if (s[j + 1] == s[i]) {
                j += 1;
            }
            next[i] = j;
        }
        
        // 获取最长相等前后缀的长度
        int len = next[n - 1];
        // 判断是否满足条件
        return len != -1 && n % (n - len - 1) == 0;
    }
};
```

class Solution {
public:

    }

## 16. KMP 算法的深入分析

### 16.1 前缀函数的详细计算

前缀函数是 KMP 算法的核心，它记录了模式串中每个位置的最长相等前后缀长度。以下是前缀函数的详细计算过程：

1. **初始化**：`next[0] = -1`，因为长度为 1 的字符串没有非空前后缀。
2. **遍历模式串**：对于每个位置 `i`（从 1 开始），计算 `next[i]`。
3. **回退过程**：当 `s[i] != s[j+1]` 时，回退 `j` 到 `next[j]`，直到 `j == -1` 或 `s[i] == s[j+1]`。
4. **更新 j**：如果 `s[i] == s[j+1]`，则 `j++`。
5. **记录结果**：`next[i] = j`。

### 16.2 前缀函数的几何意义

前缀函数可以看作是字符串的自相似性度量。对于字符串 `s`，`next[i]` 表示子串 `s[0..i]` 中最大的 `k`，使得前 `k` 个字符与后 `k` 个字符相同。

这种自相似性在模式匹配中非常重要，因为当匹配失败时，我们可以利用这种相似性来避免重复比较。

### 16.3 前缀函数的性质

1. **非递减性**：`next[i] ≤ next[i-1] + 1`
2. **回退性质**：如果 `next[i] = k`，则 `next[k]` 是 `s[0..k]` 的最长相等前后缀的长度
3. **周期性**：如果字符串 `s` 的长度为 `n`，且 `n % (n - next[n-1] - 1) == 0`，则 `s` 是由长度为 `n - next[n-1] - 1` 的子串重复构成的

### 16.4 KMP 算法的匹配过程

KMP 算法的匹配过程如下：

1. **初始化**：`i = 0`（文本串指针），`j = -1`（模式串指针）。
2. **遍历文本串**：对于每个位置 `i`，执行以下操作：
   - 当 `j != -1` 且 `text[i] != pattern[j+1]` 时，回退 `j` 到 `next[j]`。
   - 如果 `text[i] == pattern[j+1]`，则 `j++`。
   - 如果 `j == pattern.length - 1`，则匹配成功，记录位置并回退 `j` 到 `next[j]`。
3. **返回结果**：返回所有匹配的位置。

## 17. 字符串拼接法的深入分析

### 17.1 原理证明

**定理**：字符串 `s` 由重复子串构成当且仅当 `s` 是 `(s + s).substr(1, 2n - 2)` 的子串。

**证明**：
- **必要性**：假设 `s = t * k`（`t` 是子串，`k ≥ 2`），则 `s + s = t * 2k`。去除首尾字符后，得到的字符串包含 `t * k = s`。
- **充分性**：假设 `s` 是 `(s + s).substr(1, 2n - 2)` 的子串，设其起始位置为 `i`（`1 ≤ i ≤ n - 1`），则 `s[0..n-1] = s[i..i+n-1]`。通过数学归纳法可以证明 `s` 由长度为 `i` 的子串重复构成。

### 17.2 实现细节

在 C++ 中，字符串拼接法的实现需要注意以下几点：

1. **字符串拼接**：使用 `+` 运算符或 `append()` 方法
2. **子字符串提取**：使用 `substr(start, length)` 方法，其中 `start` 是起始位置，`length` 是子串长度
3. **字符串查找**：使用 `find()` 方法，返回子串的起始位置，如未找到返回 `string::npos`

### 17.3 时间复杂度分析

字符串拼接法的时间复杂度分析：

- **字符串拼接**：O(n)，其中 n 是字符串长度
- **子字符串提取**：O(n)，因为需要复制 n 个字符
- **字符串查找**：O(n)，因为底层实现通常使用 KMP 算法

总时间复杂度为 O(n)。

### 17.4 空间复杂度分析

字符串拼接法的空间复杂度分析：

- **拼接后的字符串**：O(n)，需要存储长度为 2n 的字符串
- **子字符串**：O(n)，需要存储长度为 2n-2 的字符串

总空间复杂度为 O(n)。

### 18.1 早期终止策略

暴力解法的一个重要优化是早期终止：

```cpp
bool repeatedSubstringPattern(string s) {
    int n = s.size();
    for (int len = 1; len <= n / 2; ++len) {
        if (n % len == 0) {
            bool flag = true;
            for (int i = len; i < n; ++i) {
                if (s[i] != s[i % len]) {
                    flag = false;
                    break; // 早期终止
                }
            }
            if (flag) {
                return true;
            }
        }
    }
    return false;
}
```

### 18.2 剪枝策略

对于某些情况，可以提前剪枝：

3. **尾字符检查**：如果子串的尾字符与原字符串的尾字符不同，直接跳过

## 26. 扩展问题

### 26.1 找出所有可能的重复子串

**题目**：给定一个字符串，找出所有可能的重复子串，即可以通过重复多次构成原字符串的子串。

**解题思路**：
1. 遍历所有可能的子串长度 `len`，其中 `len` 是字符串长度 `n` 的约数
2. 对于每个 `len`，检查是否可以通过重复该长度的子串得到原字符串
3. 如果可以，将该子串加入结果集

**示例**：
输入：`s = "abcabcabcabc"`
输出：`["abc", "abcabc"]`

### 26.2 最短重复子串

**题目**：给定一个由重复子串构成的字符串，找出最短的重复子串。

**解题思路**：
1. 利用 KMP 算法计算前缀函数数组
2. 最短重复子串的长度为 `n - len`，其中 `len` 是最长相等前后缀的长度
3. 提取长度为 `n - len` 的子串

**示例**：
输入：`s = "abcabcabcabc"`
输出：`"abc"`

### 26.3 重复次数

**题目**：给定一个由重复子串构成的字符串，计算重复的次数。

**解题思路**：
1. 利用 KMP 算法计算前缀函数数组
2. 重复次数为 `n / (n - len)`，其中 `len` 是最长相等前后缀的长度

**示例**：
输入：`s = "abcabcabcabc"`
输出：`4`

### 26.4 最长重复子串

**题目**：给定一个字符串，找出最长的重复子串，即在字符串中出现至少两次的最长子串。

**解题思路**：
1. 可以使用二分查找结合哈希的方法
2. 也可以使用后缀数组的方法
3. 还可以使用 KMP 算法的变种

**示例**：
输入：`s = "banana"`
输出：`"ana"`
