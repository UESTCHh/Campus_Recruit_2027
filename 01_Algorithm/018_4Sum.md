# 算法实战复盘：四数之和 (LeetCode 18)

> **打卡日期**：2026-04-03
> **核心主题**：排序、双指针、多重嵌套去重与高级剪枝。

## 📝 一、 题目描述与核心要求

* **题目内容：** 给你一个由 `n` 个整数组成的数组 `nums`，和一个目标值 `target`。请你找出并返回满足下述全部条件且**不重复**的四元组 `[nums[a], nums[b], nums[c], nums[d]]`：
  - `a`、`b`、`c` 和 `d` 互不相同
  - `nums[a] + nums[b] + nums[c] + nums[d] == target`
* **输入示例：**
  - 示例 1：`nums = [1, 0, -1, 0, -2, 2], target = 0`
    输出：`[[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]`
  - 示例 2：`nums = [2, 2, 2, 2, 2], target = 8`
    输出：`[[2, 2, 2, 2]]`
* **核心痛点：** 和四数相加 II 完全不同！这道题所有数字都在**同一个数组**里，且要求返回具体的数值组合，**绝对不能有重复的四元组**！
* **大厂面试限制：** 考察你对双指针去重逻辑的极致掌控力，以及如何进行精准的防溢出与剪枝操作。
* **题目提示：**
  - 1 <= nums.length <= 200
  - -10^9 <= nums[i] <= 10^9
  - -10^9 <= target <= 10^9

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n^3)$
**推导过程：** 排序耗时 $O(n \log n)$。外层固定指针 `i`，第二层固定指针 `j`，内层使用 `left` 和 `right` 双指针。总共三层循环，因此时间复杂度为 $O(n^3)$。

### 2. 空间复杂度：$O(1)$ 或 $O(\log n)$
**推导过程：** 除了存储结果的数组外，双指针本身只消耗常数级空间 $O(1)$。但底层的快速排序算法通常需要 $O(\log n)$ 的栈空间。

## 🤔 三、 解题思路分析

### 3.1 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：四层 `for` 循环，时间复杂度 O(n⁴)
2. **哈希表法**：使用哈希表存储两数之和，时间复杂度 O(n²)
3. **排序 + 双指针法**：先排序，然后使用两层循环 + 双指针，时间复杂度 O(n³)

### 3.2 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n⁴) | O(1) | 思路简单，不需要额外空间 | 时间复杂度极高，对于 n=200 时操作次数达到 1.6×10^9，直接超时 |
| 哈希表法 | O(n²) | O(n²) | 时间复杂度最优 | 去重极其复杂，容易出错，代码难以维护 |
| 排序 + 双指针法 | O(n³) | O(1) 或 O(logn) | 去重简单直观，代码易于维护 | 时间复杂度较高，但对于 n=200 仍可接受 |

### 3.3 数据大小对方法选择的影响

根据题目提示，数组长度 n 最大为 200：

- **暴力法**：对于 n=200，操作次数为 200⁴ = 1,600,000,000，这会导致严重超时
- **哈希表法**：对于 n=200，操作次数为 200² = 40,000，效率极高，但去重逻辑极其复杂
- **排序 + 双指针法**：对于 n=200，操作次数约为 200³ = 8,000,000，效率可接受，且去重简单

**实际测试验证：**
- 暴力法：在 LeetCode 上会直接超时
- 哈希表法：可以通过，但去重逻辑极其复杂，容易出错
- 排序 + 双指针法：执行时间可接受，通常在 10-100ms 之间，去重逻辑简单直观

### 3.4 方法选择建议

**推荐使用排序 + 双指针法**，原因如下：
1. 虽然时间复杂度为 O(n³)，但对于 n=200 的规模仍可接受
2. 去重逻辑简单直观，不容易出错，代码易于维护
3. 空间复杂度低，不需要额外的哈希表空间
4. 与三数之和解法一脉相承，易于理解和扩展

**为什么不推荐哈希表法：**
1. 虽然时间复杂度为 O(n²)，但去重逻辑极其复杂
2. 需要处理下标重叠、重复元素等多种边界情况
3. 代码难以维护，容易出错
4. 在同数组+强去重的场景下，双指针法更加优雅

## 🎯 四、 核心思想：套娃双指针，去重是魔鬼

这道题就是《三数之和》的"威力加强版"。三数之和是一层 `for` 循环 + 双指针；四数之和就是**两层 `for` 循环 + 双指针**。

### 4.1 核心原理

我们的目标是找到四个数 $a + b + c + d = target$。

通过排序 + 双指针的方法：
1. **排序**：为双指针和去重创造条件
2. **固定前两个元素**：使用两层 `for` 循环固定前两个元素 `nums[i]` 和 `nums[j]`
3. **双指针寻找剩余两个元素**：使用 `left` 和 `right` 指针在剩余区间内寻找和为 `target - nums[i] - nums[j]` 的两个元素
4. **去重**：在固定前两个元素和移动双指针时进行去重，确保结果中没有重复的四元组

### 4.2 算法步骤详解

1. **排序**：首先对数组进行排序，这是双指针和去重的基础
2. **外层循环**：固定第一个元素 `nums[i]`
   - 剪枝：如果 `nums[i] > target && nums[i] >= 0`，直接 break
   - 去重：如果 `i > 0 && nums[i] == nums[i-1]`，跳过
3. **内层循环**：固定第二个元素 `nums[j]`
   - 剪枝：如果 `nums[i] + nums[j] > target && nums[i] + nums[j] >= 0`，直接 break
   - 去重：如果 `j > i+1 && nums[j] == nums[j-1]`，跳过
4. **双指针**：使用 `left = j+1` 和 `right = n-1` 寻找剩余两个元素
   - 计算四数之和 `sum = nums[i] + nums[j] + nums[left] + nums[right]`
   - 如果 `sum == target`：
     - 记录结果
     - 去重：移动 `left` 和 `right` 跳过相同元素
     - 移动指针：`left++`，`right--`
   - 如果 `sum < target`：`left++`
   - 如果 `sum > target`：`right--`

### 4.3 与三数之和的联系与区别

**联系：**
1. 都使用排序 + 双指针的方法
2. 都需要进行去重处理
3. 都使用类似的剪枝策略

**区别：**
1. **循环层数**：三数之和是一层 `for` 循环 + 双指针；四数之和是两层 `for` 循环 + 双指针
2. **时间复杂度**：三数之和是 O(n²)；四数之和是 O(n³)
3. **去重复杂度**：四数之和需要在两个循环层和双指针层都进行去重，更加复杂
4. **剪枝策略**：四数之和需要在两个循环层都进行剪枝

### 4.4 剪枝与去重的详细说明

#### 4.4.1 剪枝策略
- **外层剪枝**：当 `nums[i] > target` 且 `nums[i] >= 0` 时，由于数组已排序，后面的元素都大于等于 `nums[i]`，四数之和必然大于 `target`，可以直接 break
- **内层剪枝**：当 `nums[i] + nums[j] > target` 且 `nums[i] + nums[j] >= 0` 时，同样由于数组已排序，后面的元素之和必然大于 `target`，可以直接 break

#### 4.4.2 去重策略
- **外层去重**：当 `i > 0 && nums[i] == nums[i-1]` 时，说明当前元素与上一个元素相同，会产生重复结果，跳过
- **内层去重**：当 `j > i+1 && nums[j] == nums[j-1]` 时，说明当前元素与上一个元素相同，会产生重复结果，跳过
- **双指针去重**：当找到符合条件的四元组后，需要跳过 `left` 和 `right` 指针指向的相同元素，避免重复结果

### 4.5 防溢出处理
由于四个整数相加可能会超过 `int` 的范围，因此需要使用 `long long` 类型来存储中间结果，避免溢出。

### 4.6 示例分析

以示例 1 为例：`nums = [1, 0, -1, 0, -2, 2], target = 0`

**步骤 1：排序**
- 排序后：`[-2, -1, 0, 0, 1, 2]`

**步骤 2：遍历固定第一个和第二个元素**
- i=0，nums[i]=-2：
  - j=1，nums[j]=-1：
    - target=1，left=2, right=5
    - nums[left]+nums[right] = 0+2=2 > 1，right--
    - nums[left]+nums[right] = 0+1=1 == 1，记录结果 [-2, -1, 0, 1]
    - 去重：left++ 到 3，right-- 到 4，结束
  - j=2，nums[j]=0，去重跳过
  - j=3，nums[j]=0，去重跳过

- i=1，nums[i]=-1：
  - j=2，nums[j]=0：
    - target=1，left=3, right=5
    - nums[left]+nums[right] = 0+2=2 > 1，right--
    - nums[left]+nums[right] = 0+1=1 == 1，记录结果 [-1, 0, 0, 1]
    - 去重：left++ 到 4，right-- 到 4，结束
  - j=3，nums[j]=0，去重跳过

- i=2，nums[i]=0，剪枝跳过

**最终结果：** `[[-2, -1, 0, 1], [-1, 0, 0, 1]]`

### 4.7 边界情况处理

1. **数组长度小于 4**：直接返回空数组
2. **第一个元素大于 target 且为正数**：由于数组已排序，后面的元素都大于等于 `nums[i]`，四数之和必然大于 `target`，直接返回
3. **重复元素**：需要在固定前两个元素和移动双指针时进行去重
4. **数值溢出**：由于 nums[i] 的范围是 -10^9 到 10^9，四个数的和可能达到 4*10^9，需要使用 `long long` 类型

### 4.8 与其他方法的对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n⁴) | O(1) | 思路简单 | 时间复杂度极高，无法处理大输入 |
| 哈希表法 | O(n²) | O(n²) | 时间复杂度最优 | 去重极其复杂，代码难以维护 |
| 排序 + 双指针法 | O(n³) | O(1) 或 O(logn) | 去重简单，代码简洁 | 时间复杂度较高，但对于 n=200 可接受 |

### 4.9 关键技术点

1. **排序**：为双指针和去重创造条件
2. **双重循环固定**：使用两层 `for` 循环固定前两个元素
3. **双指针**：在剩余区间内寻找和为 `target - nums[i] - nums[j]` 的两个元素
4. **多重去重**：确保结果中没有重复的四元组
5. **剪枝优化**：当不可能找到符合条件的组合时，提前终止
6. **防溢出**：使用 `long long` 类型存储中间结果

### 4.10 替代方法：单循环三指针法

#### 4.10.1 核心算法

单循环三指针法是一种替代方案，通过将第二个循环的逻辑融入到主循环中，使用三个指针（`j`、`left`、`right`）在固定第一个元素后同时移动，从而减少循环嵌套层次。

#### 4.10.2 指针角色与移动逻辑

- **`i` 指针**：固定第一个元素，从 0 到 `n-4` 遍历
- **`j` 指针**：从 `i+1` 开始，作为第二个元素
- **`left` 指针**：从 `j+1` 开始，作为第三个元素
- **`right` 指针**：从 `n-1` 开始，作为第四个元素

**移动逻辑**：
1. 固定 `i` 后，初始化 `j = i+1`, `left = j+1`, `right = n-1`
2. 计算四数之和 `sum`
3. 根据 `sum` 与 `target` 的关系调整指针：
   - `sum < target`：`left++`
   - `sum > target`：`right--`
   - `sum == target`：记录结果，同时移动 `left` 和 `right` 去重
4. 当 `left >= right` 时，递增 `j`，重置 `left = j+1`, `right = n-1`
5. 当 `j >= n-2` 时，递增 `i`，重置 `j = i+1`, `left = j+1`, `right = n-1`

#### 4.10.3 实现步骤详解

1. **排序**：对数组进行排序
2. **单循环**：固定 `i` 指针，从 0 到 `n-4`
   - 剪枝：如果 `nums[i] > target && nums[i] >= 0`，break
   - 去重：如果 `i > 0 && nums[i] == nums[i-1]`，continue
3. **初始化三指针**：`j = i+1`, `left = j+1`, `right = n-1`
4. **三指针遍历**：
   - 当 `j < n-2` 时：
     - 剪枝：如果 `nums[i] + nums[j] > target && nums[i] + nums[j] >= 0`，break
     - 去重：如果 `j > i+1 && nums[j] == nums[j-1]`，`j++`, `left = j+1`, `right = n-1`，continue
     - 当 `left < right` 时：
       - 计算 `sum = nums[i] + nums[j] + nums[left] + nums[right]`
       - 根据 `sum` 调整指针
       - 记录结果并去重
     - 当 `left >= right` 时：`j++`, `left = j+1`, `right = n-1`

#### 4.10.4 复杂度分析

- **时间复杂度**：O(n³)，与两层循环 + 双指针法相同
- **空间复杂度**：O(1) 或 O(logn)，与两层循环 + 双指针法相同

#### 4.10.5 代码实现

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        vector<vector<int>> result;
        int n = nums.size();
        if (n < 4) return result;
        
        sort(nums.begin(), nums.end()); // 排序是基础
        
        // 单循环固定第一个元素
        for (int i = 0; i <= n - 4; ++i) {
            // 剪枝：当 nums[i] 大于 target 且为正数时，后续不可能找到
            if (nums[i] > target && nums[i] >= 0) break;
            
            // 去重：跳过重复的第一个元素
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            // 初始化三指针
            int j = i + 1;
            int left = j + 1;
            int right = n - 1;
            
            // 三指针遍历
            while (j <= n - 3) {
                // 剪枝：当 nums[i] + nums[j] 大于 target 且为正数时，后续不可能找到
                if (nums[i] + nums[j] > target && nums[i] + nums[j] >= 0) break;
                
                // 去重：跳过重复的第二个元素
                if (j > i + 1 && nums[j] == nums[j - 1]) {
                    ++j;
                    left = j + 1;
                    right = n - 1;
                    continue;
                }
                
                // 重置 left 和 right
                left = j + 1;
                right = n - 1;
                
                while (left < right) {
                    // 防溢出
                    long long sum = (long long)nums[i] + nums[j] + nums[left] + nums[right];
                    
                    if (sum == target) {
                        // 记录结果
                        result.push_back({nums[i], nums[j], nums[left], nums[right]});
                        
                        // 去重：跳过重复的 left
                        while (left < right && nums[left] == nums[left + 1]) ++left;
                        // 去重：跳过重复的 right
                        while (left < right && nums[right] == nums[right - 1]) --right;
                        
                        // 移动指针
                        ++left;
                        --right;
                    } else if (sum < target) {
                        // 和太小，左指针右移
                        ++left;
                    } else {
                        // 和太大，右指针左移
                        --right;
                    }
                }
                
                // 移动 j 指针
                ++j;
            }
        }
        
        return result;
    }
};
```

#### 4.10.6 优势与适用场景

**优势：**
1. **代码结构更紧凑**：将两层循环的逻辑合并为一层，减少了循环嵌套
2. **指针移动更直观**：三个指针的移动逻辑在同一循环中处理，便于理解
3. **剪枝条件更集中**：剪枝逻辑可以更集中地应用，减少不必要的计算

**适用场景：**
1. **代码可读性要求高**：单循环结构更易于理解和维护
2. **需要更集中的剪枝逻辑**：当需要在多个指针层面应用剪枝时
3. **学习算法思想**：有助于理解多指针协同移动的原理

**与传统两层循环 + 双指针法的对比：**
- **时间复杂度**：两者均为 O(n³)
- **空间复杂度**：两者均为 O(1) 或 O(logn)
- **代码结构**：单循环三指针法结构更紧凑，传统方法逻辑更清晰
- **适用场景**：两者适用于相同的场景，选择取决于个人偏好和代码风格

### 4.11 仅统计四元组数目而非具体内容的优化方案

#### 4.11.1 核心思路

当仅需统计四元组数目而非返回具体四元组内容时，我们可以对算法进行优化，主要通过简化去重逻辑和减少内存开销来提高性能。

#### 4.11.2 优化点

1. **内存优化**：不需要存储具体的四元组，只需要维护一个计数器
2. **去重简化**：使用不同的去重策略，避免存储重复的四元组
3. **剪枝增强**：可以应用更激进的剪枝策略，因为不需要记录具体结果

#### 4.11.3 算法实现

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    int fourSumCount(vector<int>& nums, int target) {
        int count = 0;
        int n = nums.size();
        if (n < 4) return 0;
        
        sort(nums.begin(), nums.end()); // 排序是基础
        
        for (int i = 0; i < n - 3; ++i) {
            // 剪枝：当 nums[i] 大于 target 且为正数时，后续不可能找到
            if (nums[i] > target && nums[i] >= 0) break;
            
            // 去重：跳过重复的第一个元素
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            for (int j = i + 1; j < n - 2; ++j) {
                // 剪枝：当 nums[i] + nums[j] 大于 target 且为正数时，后续不可能找到
                if (nums[i] + nums[j] > target && nums[i] + nums[j] >= 0) break;
                
                // 去重：跳过重复的第二个元素
                if (j > i + 1 && nums[j] == nums[j - 1]) continue;
                
                int left = j + 1;
                int right = n - 1;
                
                while (left < right) {
                    // 防溢出
                    long long sum = (long long)nums[i] + nums[j] + nums[left] + nums[right];
                    
                    if (sum == target) {
                        // 计算当前组合的重复次数
                        int leftCount = 1;
                        int rightCount = 1;
                        
                        // 统计左指针的重复次数
                        while (left + 1 < right && nums[left] == nums[left + 1]) {
                            leftCount++;
                            left++;
                        }
                        
                        // 统计右指针的重复次数
                        while (right - 1 > left && nums[right] == nums[right - 1]) {
                            rightCount++;
                            right--;
                        }
                        
                        // 增加计数
                        count += leftCount * rightCount;
                        
                        // 移动指针
                        left++;
                        right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }
        
        return count;
    }
};
```

#### 4.11.4 复杂度分析

- **时间复杂度**：O(n³)，与原算法相同
- **空间复杂度**：O(1)，仅使用常数级额外空间

#### 4.11.5 适用场景

1. **大规模数据**：当数组规模较大，仅需统计数目时
2. **内存受限**：当内存有限，无法存储所有可能的四元组时
3. **性能要求高**：当需要更快的执行速度时

### 4.12 N数之和的通用解决方案

#### 4.12.1 通用算法设计思路

N数之和问题可以通过递归和双指针的组合来解决。核心思路是将N数之和问题逐步分解为(N-1)数之和问题，直到分解为2数之和问题，然后使用双指针法解决。

**通用步骤：**
1. **排序**：对数组进行排序
2. **递归分解**：固定一个元素，将问题分解为(N-1)数之和
3. **双指针求解**：当问题分解为2数之和时，使用双指针法求解
4. **去重**：在每一层递归中处理重复元素
5. **剪枝**：在每一层递归中应用剪枝策略

#### 4.12.2 不同N值的实现差异

**N=2**：
- 直接使用双指针法
- 时间复杂度：O(n)
- 空间复杂度：O(1)

**N=3**：
- 一层循环 + 双指针
- 时间复杂度：O(n²)
- 空间复杂度：O(1) 或 O(logn)

**N=4**：
- 两层循环 + 双指针
- 时间复杂度：O(n³)
- 空间复杂度：O(1) 或 O(logn)

**N>4**：
- 递归分解 + 双指针
- 时间复杂度：O(n^{N-1})
- 空间复杂度：O(N)（递归栈空间）

#### 4.12.3 通用实现框架

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> nSum(vector<int>& nums, int target, int n) {
        vector<vector<int>> result;
        if (nums.size() < n) return result;
        
        sort(nums.begin(), nums.end());
        vector<int> path;
        backtrack(nums, target, n, 0, path, result);
        
        return result;
    }
    
private:
    void backtrack(vector<int>& nums, long long target, int k, int start, 
                   vector<int>& path, vector<vector<int>>& result) {
        // 递归终止条件：k=2时使用双指针
        if (k == 2) {
            int left = start;
            int right = nums.size() - 1;
            
            while (left < right) {
                long long sum = (long long)nums[left] + nums[right];
                
                if (sum == target) {
                    vector<int> temp = path;
                    temp.push_back(nums[left]);
                    temp.push_back(nums[right]);
                    result.push_back(temp);
                    
                    // 去重
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    
                    left++;
                    right--;
                } else if (sum < target) {
                    left++;
                } else {
                    right--;
                }
            }
            return;
        }
        
        // 递归处理
        for (int i = start; i <= nums.size() - k; ++i) {
            // 剪枝：如果当前元素大于 target 且为正数，后续不可能找到
            if (nums[i] > target && nums[i] >= 0) break;
            
            // 去重：跳过重复元素
            if (i > start && nums[i] == nums[i - 1]) continue;
            
            // 选择当前元素
            path.push_back(nums[i]);
            
            // 递归处理剩余的 k-1 个数
            backtrack(nums, target - nums[i], k - 1, i + 1, path, result);
            
            // 回溯
            path.pop_back();
        }
    }
};
```

#### 4.12.4 时间复杂度与空间复杂度分析

**时间复杂度**：
- 排序：O(n log n)
- 递归分解：O(n^{N-1})
- 整体：O(n^{N-1})，其中 N 是目标数的个数

**空间复杂度**：
- 排序：O(log n)
- 递归栈：O(N)
- 结果存储：O(k)，其中 k 是符合条件的组合数
- 整体：O(N + k)

#### 4.12.5 处理重复元素的通用策略

1. **排序后去重**：在每一层递归中，跳过与前一个元素相同的元素
2. **双指针去重**：在双指针阶段，跳过相同的元素
3. **剪枝优化**：在每一层递归中应用剪枝策略，减少不必要的计算

#### 4.12.6 边界条件处理的通用方法

1. **数组长度检查**：如果数组长度小于 N，直接返回空结果
2. **数值溢出处理**：使用 `long long` 类型存储中间结果
3. **剪枝条件**：当当前元素大于 target 且为正数时，后续不可能找到符合条件的组合
4. **递归终止条件**：当 N=2 时，使用双指针法求解

#### 4.12.7 适用场景

1. **通用 N 数之和问题**：当 N 是一个抽象的确定数时
2. **需要灵活处理不同 N 值**：当需要处理不同长度的数之和问题时
3. **学习递归与双指针的结合**：当需要理解递归和双指针的协同使用时

## 💻 五、 核心代码

### 5.1 排序 + 双指针法实现

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        vector<vector<int>> result;
        sort(nums.begin(), nums.end()); // 排序是去重和双指针的绝对前提
        
        for (int i = 0; i < nums.size(); i++) {
            // 1. 针对外层 i 的高级剪枝
            // 注意：不能仅仅 nums[i] > target，因为如果 target 是负数，负数加负数会更小！
            if (nums[i] > target && nums[i] >= 0) break;
            
            // 2. 外层 i 去重
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            for (int j = i + 1; j < nums.size(); j++) {
                // 3. 针对内层 j 的二级剪枝
                // 把 nums[i] + nums[j] 视为一个整体
                if (nums[i] + nums[j] > target && nums[i] + nums[j] >= 0) break;
                
                // 4. 内层 j 去重 (注意边界是 j > i + 1)
                if (j > i + 1 && nums[j] == nums[j - 1]) continue;
                
                int left = j + 1;
                int right = nums.size() - 1;
                
                while (left < right) {
                    // 防暴击：用 long long 防止四个大整数相加导致 int 溢出！
                    long long sum = (long long)nums[i] + nums[j] + nums[left] + nums[right];
                    
                    if (sum == target) {
                        result.push_back({nums[i], nums[j], nums[left], nums[right]});
                        // 5. left 和 right 去重
                        while (left < right && nums[left] == nums[left + 1]) left++;
                        while (left < right && nums[right] == nums[right - 1]) right--;
                        
                        left++;
                        right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }
        return result;
    }
};
```

**代码解释：**
1. **排序**：对数组进行排序，为双指针和去重创造条件
2. **外层剪枝**：当 `nums[i] > target` 且 `nums[i] >= 0` 时，直接 break
3. **外层去重**：跳过与前一个元素相同的元素，避免重复结果
4. **内层剪枝**：当 `nums[i] + nums[j] > target` 且 `nums[i] + nums[j] >= 0` 时，直接 break
5. **内层去重**：跳过与前一个元素相同的元素，避免重复结果
6. **双指针**：在剩余区间内寻找和为 `target - nums[i] - nums[j]` 的两个元素
7. **记录结果**：当找到符合条件的组合时，记录结果并去重

### 5.2 暴力法实现（仅供参考，实际会超时）

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        vector<vector<int>> result;
        int n = nums.size();
        
        // 排序，便于去重
        sort(nums.begin(), nums.end());
        
        // 四层循环遍历所有可能的组合
        for (int i = 0; i < n; i++) {
            // 去重：跳过相同的元素
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            for (int j = i + 1; j < n; j++) {
                // 去重：跳过相同的元素
                if (j > i + 1 && nums[j] == nums[j - 1]) continue;
                
                for (int k = j + 1; k < n; k++) {
                    // 去重：跳过相同的元素
                    if (k > j + 1 && nums[k] == nums[k - 1]) continue;
                    
                    for (int l = k + 1; l < n; l++) {
                        // 去重：跳过相同的元素
                        if (l > k + 1 && nums[l] == nums[l - 1]) continue;
                        
                        if ((long long)nums[i] + nums[j] + nums[k] + nums[l] == target) {
                            result.push_back({nums[i], nums[j], nums[k], nums[l]});
                        }
                    }
                }
            }
        }
        
        return result;
    }
};
```

**暴力法性能分析：**
- 时间复杂度：O(n⁴)，其中 n 是数组的长度
- 空间复杂度：O(1)，不需要额外空间
- 适用场景：数据规模很小，对性能要求不高的情况

### 5.3 哈希表法实现（仅供参考，实际去重极其复杂）

```cpp
#include <vector>
#include <unordered_map>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        vector<vector<int>> result;
        int n = nums.size();
        
        // 排序，便于去重
        sort(nums.begin(), nums.end());
        
        // 使用哈希表存储两数之和及其下标组合
        unordered_map<long long, vector<pair<int, int>>> two_sum;
        
        // 遍历所有两数之和
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                long long sum = (long long)nums[i] + nums[j];
                two_sum[sum].push_back({i, j});
            }
        }
        
        // 遍历所有两数之和的组合
        for (auto it1 = two_sum.begin(); it1 != two_sum.end(); it1++) {
            long long sum1 = it1->first;
            long long sum2 = target - sum1;
            
            if (two_sum.find(sum2) == two_sum.end()) continue;
            
            vector<pair<int, int>>& vec1 = it1->second;
            vector<pair<int, int>>& vec2 = two_sum[sum2];
            
            for (auto& p1 : vec1) {
                for (auto& p2 : vec2) {
                    // 确保下标不重叠
                    if (p1.first == p2.first || p1.first == p2.second ||
                        p1.second == p2.first || p1.second == p2.second) continue;
                    
                    vector<int> quad = {nums[p1.first], nums[p1.second], nums[p2.first], nums[p2.second]};
                    sort(quad.begin(), quad.end());
                    result.push_back(quad);
                }
            }
        }
        
        // 去重
        sort(result.begin(), result.end());
        result.erase(unique(result.begin(), result.end()), result.end());
        
        return result;
    }
};
```

**哈希表法性能分析：**
- 时间复杂度：O(n²)，其中 n 是数组的长度
- 空间复杂度：O(n²)，需要额外空间存储哈希表
- 适用场景：需要最优时间复杂度的场景，但代码极其复杂

## ❌ 七、 常见错误及解决方案

### 7.1 常见错误

1. **剪枝条件错误**：
   - 错误：使用 `if (nums[i] > target)` 进行剪枝
   - 后果：会漏掉正确答案，如 target = -10，nums[i] = -4
   - 原因：负数加负数会更小，不能简单地进行剪枝

2. **去重边界错误**：
   - 错误：使用 `if (j > i && nums[j] == nums[j - 1])` 进行内层去重
   - 后果：会漏掉正确答案
   - 原因：内层去重的边界应该是 `j > i + 1`

3. **双指针移动错误**：
   - 错误：找到符合条件的组合后，只移动一个指针
   - 后果：会导致死循环或重复结果
   - 原因：需要同时移动左右指针，继续寻找下一个组合

4. **防溢出遗漏**：
   - 错误：使用 `int` 类型存储四数之和
   - 后果：当四个数都接近 10^9 时，可能会溢出
   - 原因：需要使用 `long long` 类型

5. **边界条件处理错误**：
   - 错误：没有处理数组长度小于 4 的情况
   - 后果：可能导致数组越界
   - 原因：题目要求至少四个元素

### 7.2 解决方案

1. **正确的剪枝方式**：
   - 使用 `if (nums[i] > target && nums[i] >= 0)` 进行外层剪枝
   - 使用 `if (nums[i] + nums[j] > target && nums[i] + nums[j] >= 0)` 进行内层剪枝
   - 确保只有在 nums[i] 或 nums[i] + nums[j] 为非负数且大于 target 时才剪枝

2. **正确的去重边界**：
   - 外层去重：`if (i > 0 && nums[i] == nums[i - 1])`
   - 内层去重：`if (j > i + 1 && nums[j] == nums[j - 1])`
   - 确保内层去重的边界是 `j > i + 1`，而不是 `j > i`

3. **正确的双指针移动**：
   - 找到符合条件的组合后，同时移动左右指针
   - 在移动指针前，先跳过相同的元素

4. **防溢出处理**：
   - 使用 `long long` 类型存储四数之和
   - 确保四个数相加不会溢出

5. **边界条件处理**：
   - 在函数开始时检查数组长度
   - 确保数组长度至少为 4

## 💡 九、 面试直击灵魂的防暴击拷问

* **灵魂拷问一：为什么上一道题（四数相加 II）能用哈希表把复杂度降到 $O(n^2)$，而这道题却不用哈希表？**
    * **绝杀回答：**"这是核心痛点！上一道题是在四个独立的数组里找组合，且不要求去重，只要个数，哈希表是绝杀。但这道题是在同一个数组里找，且绝对不允许有重复的四元组。如果我们强行用哈希表存两数之和，当遇到哈希冲突和复杂的下标重叠判断时，去重逻辑会变得极其恐怖且容易写出 Bug，而且维护哈希表的开销极大。在这种『同数组+强去重』的场景下，先排序后使用双指针，依靠指针单向移动来天然去重，才是最优雅且空间复杂度 $O(1)$ 的工程级做法。"

* **灵魂拷问二：代码里的剪枝逻辑 if (nums[i] > target && nums[i] >= 0)，为什么一定要加 nums[i] >= 0？**
    * **绝杀回答：**"这是大厂面试最爱挖的坑。如果 target 是正数，nums[i] > target 直接剪枝没问题。但如果 target = -10，而当前的 nums[i] = -4，此时 -4 > -10 成立，如果你直接剪枝了，你就会漏掉后面可能出现的 -4 + (-1) + (-2) + (-3) = -10 的正确答案！因为负数越加越小。所以必须保证 nums[i] >= 0，确保后面的数字都是正数，越加越大，此时剪枝才是绝对安全的。"

* **灵魂拷问三：内层去重的边界为什么是 j > i + 1？**
    * **绝杀回答：**"因为 j 的起始位置是 i + 1，所以第一个 j 值是 i + 1，此时 nums[j] 是第一次出现，不应该去重。只有当 j > i + 1 时，才说明 nums[j] 已经和前一个元素比较过了，此时去重才是正确的。"

* **灵魂拷问四：为什么使用 long long 而不使用 int？**
    * **绝杀回答：**"因为 nums[i] 的范围是 -10^9 到 10^9，四个数相加的最大值可能是 4 × 10^9，这已经超过了 int 的范围（int 的最大值约为 2.1 × 10^9）。如果使用 int 进行相加，可能会导致溢出，得到错误的结果。因此必须使用 long long 类型来存储四数之和。"

* **灵魂拷问五：四数之和和三数之和有什么区别？**
    * **绝杀回答：**"四数之和是三数之和的扩展。三数之和是一层 `for` 循环 + 双指针，时间复杂度 O(n²)；四数之和是两层 `for` 循环 + 双指针，时间复杂度 O(n³)。四数之和需要在两个循环层都进行剪枝和去重，更加复杂。但核心思想是一样的：先排序，然后使用双指针在剩余区间内寻找符合条件的组合。"

* **灵魂拷问六：如何将算法扩展到 K 数之和？**
    * **绝杀回答：**"可以使用递归的方法，将 K 数之和转化为 (K-1) 数之和。具体来说，固定前 K-2 个元素，然后在剩余的数组中使用双指针或递归求解 (K-1) 数之和。时间复杂度为 O(n^{K-1})。但需要注意去重和剪枝的复杂性会随着 K 的增加而增加。"

## 📚 十、 总结

本题是三数之和的"威力加强版"，通过排序 + 双指针的方法，我们成功将时间复杂度从 O(n⁴) 优化到了 O(n³)，并通过多重去重和剪枝策略确保了结果的唯一性和算法的效率。

### 核心要点：
1. **排序**：为双指针和去重创造条件
2. **双重循环固定**：使用两层 `for` 循环固定前两个元素
3. **双指针**：在剩余区间内寻找和为 `target - nums[i] - nums[j]` 的两个元素
4. **多重去重**：确保结果中没有重复的四元组
5. **剪枝优化**：当不可能找到符合条件的组合时，提前终止
6. **防溢出**：使用 `long long` 类型存储中间结果

### 扩展思考：
- **算法应用**：该算法思想可以扩展到 K 数之和等问题
- **性能优化**：对于大规模数据，可以考虑更多的剪枝条件和并行计算
- **实际应用**：在金融分析、数据挖掘、游戏开发等领域有广泛应用

### 方法比较：

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n⁴) | O(1) | 思路简单 | 时间复杂度极高 |
| 哈希表法 | O(n²) | O(n²) | 时间复杂度最优 | 去重极其复杂 |
| 排序 + 双指针法 | O(n³) | O(1) 或 O(logn) | 去重简单，代码简洁 | 时间复杂度较高 |

通过本题的学习，我们不仅掌握了一种高效的算法，更重要的是理解了排序 + 双指针的思想和多重去重策略，为解决更复杂的问题打下了基础。
