# 算法实战复盘：四数之和 (LeetCode 18)

> **打卡日期**：2026-04-03
> **核心主题**：排序、双指针、多重嵌套去重与高级剪枝。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给你一个由 `n` 个整数组成的数组 `nums`，和一个目标值 `target`。请你找出并返回满足下述全部条件且**不重复**的四元组 `[nums[a], nums[b], nums[c], nums[d]]`：
  - `a`、`b`、`c` 和 `d` 互不相同
  - `nums[a] + nums[b] + nums[c] + nums[d] == target`
* **输入示例：**
  - 示例 1：`nums = [1, 0, -1, 0, -2, 2], target = 0`
    输出：`[[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]`
  - 示例 2：`nums = [2, 2, 2, 2, 2], target = 8`
    输出：`[[2, 2, 2, 2]]`
* **核心痛点：** 和四数相加 II 完全不同！这道题所有数字都在**同一个数组**里，且要求返回具体的数值组合，**绝对不能有重复的四元组**！
* **大厂面试限制：** 考察你对双指针去重逻辑的极致掌控力，以及如何进行精准的防溢出与剪枝操作。

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n^3)$
**推导过程：** 排序耗时 $O(n \log n)$。外层固定指针 `i`，第二层固定指针 `j`，内层使用 `left` 和 `right` 双指针。总共三层循环，因此时间复杂度为 $O(n^3)$。

### 2. 空间复杂度：$O(1)$ 或 $O(\log n)$
**推导过程：** 除了存储结果的数组外，双指针本身只消耗常数级空间 $O(1)$。但底层的快速排序算法通常需要 $O(\log n)$ 的栈空间。

## 🎯 三、 核心思想：套娃双指针，去重是魔鬼

这道题就是《三数之和》的“威力加强版”。三数之和是一层 `for` 循环 + 双指针；四数之和就是**两层 `for` 循环 + 双指针**。

### 3.1 算法步骤详解

1. **排序**：首先对数组进行排序，这是双指针和去重的基础
2. **外层循环**：固定第一个元素 `nums[i]`
   - 剪枝：如果 `nums[i] > target && nums[i] >= 0`，直接 break
   - 去重：如果 `i > 0 && nums[i] == nums[i-1]`，跳过
3. **内层循环**：固定第二个元素 `nums[j]`
   - 剪枝：如果 `nums[i] + nums[j] > target && nums[i] + nums[j] >= 0`，直接 break
   - 去重：如果 `j > i+1 && nums[j] == nums[j-1]`，跳过
4. **双指针**：使用 `left = j+1` 和 `right = n-1` 寻找剩余两个元素
   - 计算四数之和 `sum = nums[i] + nums[j] + nums[left] + nums[right]`
   - 如果 `sum == target`：
     - 记录结果
     - 去重：移动 `left` 和 `right` 跳过相同元素
     - 移动指针：`left++`，`right--`
   - 如果 `sum < target`：`left++`
   - 如果 `sum > target`：`right--`

### 3.2 剪枝与去重的详细说明

#### 3.2.1 剪枝策略
- **外层剪枝**：当 `nums[i] > target` 且 `nums[i] >= 0` 时，由于数组已排序，后面的元素都大于等于 `nums[i]`，四数之和必然大于 `target`，可以直接 break
- **内层剪枝**：当 `nums[i] + nums[j] > target` 且 `nums[i] + nums[j] >= 0` 时，同样由于数组已排序，后面的元素之和必然大于 `target`，可以直接 break

#### 3.2.2 去重策略
- **外层去重**：当 `i > 0 && nums[i] == nums[i-1]` 时，说明当前元素与上一个元素相同，会产生重复结果，跳过
- **内层去重**：当 `j > i+1 && nums[j] == nums[j-1]` 时，说明当前元素与上一个元素相同，会产生重复结果，跳过
- **双指针去重**：当找到符合条件的四元组后，需要跳过 `left` 和 `right` 指针指向的相同元素，避免重复结果

### 3.3 防溢出处理
由于四个整数相加可能会超过 `int` 的范围，因此需要使用 `long long` 类型来存储中间结果，避免溢出。

## 💻 四、 核心代码

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        vector<vector<int>> result;
        sort(nums.begin(), nums.end()); // 排序是去重和双指针的绝对前提
        
        for (int i = 0; i < nums.size(); i++) {
            // 1. 针对外层 i 的高级剪枝
            // 注意：不能仅仅 nums[i] > target，因为如果 target 是负数，负数加负数会更小！
            if (nums[i] > target && nums[i] >= 0) break;
            
            // 2. 外层 i 去重
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            for (int j = i + 1; j < nums.size(); j++) {
                // 3. 针对内层 j 的二级剪枝
                // 把 nums[i] + nums[j] 视为一个整体
                if (nums[i] + nums[j] > target && nums[i] + nums[j] >= 0) break;
                
                // 4. 内层 j 去重 (注意边界是 j > i + 1)
                if (j > i + 1 && nums[j] == nums[j - 1]) continue;
                
                int left = j + 1;
                int right = nums.size() - 1;
                
                while (left < right) {
                    // 防暴击：用 long long 防止四个大整数相加导致 int 溢出！
                    long long sum = (long long)nums[i] + nums[j] + nums[left] + nums[right];
                    
                    if (sum == target) {
                        result.push_back({nums[i], nums[j], nums[left], nums[right]});
                        // 5. left 和 right 去重
                        while (left < right && nums[left] == nums[left + 1]) left++;
                        while (left < right && nums[right] == nums[right - 1]) right--;
                        
                        left++;
                        right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }
        return result;
    }
};
```

## 💡 五、 面试直击灵魂的防暴击拷问
* **灵魂拷问一：为什么上一道题（四数相加 II）能用哈希表把复杂度降到 $O(n^2)$，而这道题却不用哈希表？**
    * **绝杀回答：**“这是核心痛点！上一道题是在四个独立的数组里找组合，且不要求去重，只要个数，哈希表是绝杀。但这道题是在同一个数组里找，且绝对不允许有重复的四元组。如果我们强行用哈希表存两数之和，当遇到哈希冲突和复杂的下标重叠判断时，去重逻辑会变得极其恐怖且容易写出 Bug，而且维护哈希表的开销极大。在这种『同数组+强去重』的场景下，先排序后使用双指针，依靠指针单向移动来天然去重，才是最优雅且空间复杂度 $O(1)$ 的工程级做法。”
* **灵魂拷问二：代码里的剪枝逻辑 if (nums[i] > target && nums[i] >= 0)，为什么一定要加 nums[i] >= 0？**
    * **绝杀回答：**“这是大厂面试最爱挖的坑。如果 target 是正数，nums[i] > target 直接剪枝没问题。但如果 target = -10，而当前的 nums[i] = -4，此时 -4 > -10 成立，如果你直接剪枝了，你就会漏掉后面可能出现的 -4 + (-1) + (-2) + (-3) = -10 的正确答案！因为负数越加越小。所以必须保证 nums[i] >= 0，确保后面的数字都是正数，越加越大，此时剪枝才是绝对安全的。”
