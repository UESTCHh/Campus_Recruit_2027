# 算法实战复盘：四数相加 II (LeetCode 454)

> **打卡日期**：2026-04-03
> **核心主题**：哈希表、分组降维打击、$O(n^4)$ 到 $O(n^2)$ 的极致优化。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给你四个整数数组 `nums1`、`nums2`、`nums3` 和 `nums4`，数组长度都是 `n`，请你计算有多少个元组 `(i, j, k, l)` 能满足 `nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0`, `0 <= i, j, k, l < n`。
* **输入示例：**
  - 示例 1：`nums1 = [1, 2], nums2 = [-2, -1], nums3 = [-1, 2], nums4 = [0, 2]`
    输出：`2`
    解释：两个元组分别是 `(0, 0, 0, 1)` 和 `(1, 1, 0, 0)`
  - 示例 2：`nums1 = [0], nums2 = [0], nums3 = [0], nums4 = [0]`
    输出：`1`
* **核心痛点：** 最暴力的解法是四层 `for` 循环，时间复杂度 $O(n^4)$。如果 $n=200$，直接超时爆炸。
* **大厂面试限制：** 题目只要求返回"有多少个"，**不要求去重**，这是本题能使用哈希表进行疯狂优化的核心眼眼！
* **题目提示：**
  - n == nums1.length == nums2.length == nums3.length == nums4.length
  - 1 <= n <= 200
  - -2^28 <= nums1[i], nums2[i], nums3[i], nums4[i] <= 2^28

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n^2)$
**推导过程：** 我们将四个数组分为两组（A+B 和 C+D）。遍历 A 和 B 组计算所有的和需要 $O(n^2)$。遍历 C 和 D 组并在哈希表中查询需要 $O(n^2)$。整体时间复杂度被完美压制在了 $O(n^2)$。

### 2. 空间复杂度：$O(n^2)$
**推导过程：** 我们需要一个哈希表来存储 A 和 B 组所有可能的和及其出现的次数。最坏情况下，A 和 B 的组合有 $n^2$ 种不同的和，所以空间复杂度为 $O(n^2)$。这是典型的**空间换时间**。

## 🤔 三、 解题思路分析

### 3.1 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：四层 `for` 循环，时间复杂度 O(n⁴)
2. **1+3 分组法**：将一个数组与其他三个数组分组，时间复杂度 O(n³)
3. **2+2 分组法**：将四个数组分为两组，时间复杂度 O(n²)

### 3.2 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n⁴) | O(1) | 思路简单，不需要额外空间 | 时间复杂度极高，对于 n=200 时操作次数达到 1.6×10^9，直接超时 |
| 1+3 分组法 | O(n³) | O(n²) | 比暴力法快 | 时间复杂度仍然较高，对于 n=200 时操作次数达到 8×10^6，可能超时 |
| 2+2 分组法 | O(n²) | O(n²) | 时间复杂度最优，效率极高 | 需要额外空间存储哈希表 |

### 3.3 数据大小对方法选择的影响

根据题目提示，数组长度 n 最大为 200：

- **暴力法**：对于 n=200，操作次数为 200⁴ = 1,600,000,000，这会导致严重超时
- **1+3 分组法**：对于 n=200，操作次数为 200³ = 8,000,000，可能会超时
- **2+2 分组法**：对于 n=200，操作次数为 2×200² = 80,000，效率极高

**实际测试验证：**
- 暴力法：在 LeetCode 上会直接超时
- 1+3 分组法：可能会超时
- 2+2 分组法：执行时间极短，通常在 0-10ms 之间

### 3.4 哈希表结构的分析

在本题中，我们可以使用以下几种哈希表结构：

| 哈希表结构 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------------|------------|------------|------|------|
| `std::unordered_map` | 平均 O(1) | O(n²) | 查询速度快，不需要排序 | 底层实现复杂，有一定的常数开销 |
| `std::map` | O(logn) | O(n²) | 自动排序 | 查询和插入时间复杂度较高 |
| `std::unordered_set` | 平均 O(1) | O(n²) | 自动去重，查询速度快 | 无法存储次数信息 |

**选择建议：**
- 本题需要同时存储和值及其出现次数，所以必须使用 `std::unordered_map`
- 如果只需要存储和值，可以使用 `std::unordered_set`
- 只有在需要排序功能时，才考虑使用 `std::map`

## 🎯 四、 核心思想：分组对抗，哈希寻人

我们要找 $A + B + C + D = 0$。通过移项，可以变成 **$A + B = -(C + D)$**。

这就把一个四维的恐怖问题，降维成了两个二维问题！

### 4.1 算法步骤详解

1. **分组**：将四个数组分为两组，A+B 和 C+D
2. **构建哈希表**：
   - 遍历 A 和 B 数组，计算所有可能的和 `a + b`
   - 在哈希表中记录每个和出现的次数，Key 是和的值，Value 是出现的次数
3. **查找匹配**：
   - 遍历 C 和 D 数组，计算所有可能的和 `c + d`
   - 计算目标值 `target = 0 - (c + d)`
   - 在哈希表中查找 `target`，如果找到，将对应的次数加到结果中

### 4.2 分组策略分析

#### 4.2.1 为什么选择 2+2 分组？
- **1+3 分组**：计算 A 需要 O(n)，计算 B+C+D 需要 O(n³)，总时间复杂度 O(n³)
- **2+2 分组**：计算 A+B 和 C+D 各需要 O(n²)，总时间复杂度 O(n²)
- **结论**：2+2 分组是性能最优的选择，将时间复杂度从 O(n⁴) 降到了 O(n²)

### 4.3 哈希表的使用

- **数据结构选择**：使用 `unordered_map`，平均查找时间复杂度 O(1)
- **键值设计**：
  - Key：A+B 的和值
  - Value：该和值出现的次数
- **空间权衡**：最坏情况下，A+B 的组合有 n² 种不同的和，空间复杂度 O(n²)，这是典型的空间换时间策略

### 4.4 示例分析

以示例 1 为例：
- `nums1 = [1, 2], nums2 = [-2, -1], nums3 = [-1, 2], nums4 = [0, 2]`

**步骤 1：构建哈希表**
- A+B 的可能和：
  - 1 + (-2) = -1
  - 1 + (-1) = 0
  - 2 + (-2) = 0
  - 2 + (-1) = 1
- 哈希表内容：`{-1: 1, 0: 2, 1: 1}`

**步骤 2：查找匹配**
- C+D 的可能和：
  - (-1) + 0 = -1 → 目标值 1，哈希表中存在，次数 1
  - (-1) + 2 = 1 → 目标值 -1，哈希表中存在，次数 1
  - 2 + 0 = 2 → 目标值 -2，哈希表中不存在
  - 2 + 2 = 4 → 目标值 -4，哈希表中不存在
- 总次数：1 + 1 = 2，与示例输出一致

### 4.5 与其他方法的对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n⁴) | O(1) | 思路简单，不需要额外空间 | 时间复杂度极高，对于 n=200 时直接超时 |
| 1+3 分组法 | O(n³) | O(n²) | 比暴力法快 | 时间复杂度仍然较高 |
| 2+2 分组法 | O(n²) | O(n²) | 时间复杂度最优，效率极高 | 需要额外空间存储哈希表 |

### 4.6 关键技术点

1. **分组降维**：将四维问题转化为两个二维问题，通过 $A + B = -(C + D)$ 的变形，实现降维打击
2. **2+2 分组**：选择最优的分组策略，使两边的时间复杂度都降到 $O(n^2)$
3. **哈希表计数**：使用哈希表记录 A+B 的和值及其出现次数，实现快速查找
4. **空间换时间**：以 $O(n^2)$ 的空间复杂度换取时间复杂度的优化
5. **无需去重**：题目只要求返回组合个数，无需去重，这是使用哈希表的关键前提

通过这种方法，我们成功将时间复杂度从 $O(n^4)$ 优化到了 $O(n^2)$，实现了高效的求解。

## 💻 五、 核心代码

### 5.1 哈希表法实现

```cpp
#include <vector>
#include <unordered_map>

using namespace std;

class Solution {
public:
    int fourSumCount(vector<int>& nums1, vector<int>& nums2, vector<int>& nums3, vector<int>& nums4) {
        // key: a+b的数值，value: a+b数值出现的次数
        unordered_map<int, int> umap; 
        
        // 1. 遍历前两个数组，统计两个数组元素之和，和出现的次数，放到map中
        for (int a : nums1) {
            for (int b : nums2) {
                umap[a + b]++;
            }
        }
        
        int count = 0; // 统计 a+b+c+d = 0 出现的次数
        
        // 2. 遍历后两个数组，找到如果 0-(c+d) 在map中出现过的话，就把map中key对应的value也就是出现次数统计出来
        for (int c : nums3) {
            for (int d : nums4) {
                int target = 0 - (c + d);
                if (umap.find(target) != umap.end()) {
                    count += umap[target];
                }
            }
        }
        
        return count;
    }
};
```

**哈希表法解释：**
1. **构建哈希表**：遍历 nums1 和 nums2，计算所有可能的和 `a + b`，并记录每个和出现的次数
2. **查找匹配**：遍历 nums3 和 nums4，计算所有可能的和 `c + d`，然后查找 `0 - (c + d)` 在哈希表中出现的次数
3. **累加结果**：将找到的次数累加到结果中

### 5.2 暴力法实现

```cpp
#include <vector>

using namespace std;

class Solution {
public:
    int fourSumCount(vector<int>& nums1, vector<int>& nums2, vector<int>& nums3, vector<int>& nums4) {
        int count = 0;
        
        // 四层循环遍历所有可能的组合
        for (int a : nums1) {
            for (int b : nums2) {
                for (int c : nums3) {
                    for (int d : nums4) {
                        if (a + b + c + d == 0) {
                            count++;
                        }
                    }
                }
            }
        }
        
        return count;
    }
};
```

**暴力法解释：**
1. **四层循环**：使用四层 for 循环遍历所有可能的元素组合
2. **检查条件**：对于每组元素，检查它们的和是否等于 0
3. **累加结果**：如果和等于 0，累加计数器

**暴力法性能分析：**
- 时间复杂度：O(n⁴)，其中 n 是数组的长度
- 空间复杂度：O(1)，不需要额外空间
- 适用场景：数据规模很小，对性能要求不高的情况

### 5.3 1+3 分组法实现

```cpp
#include <vector>
#include <unordered_map>

using namespace std;

class Solution {
public:
    int fourSumCount(vector<int>& nums1, vector<int>& nums2, vector<int>& nums3, vector<int>& nums4) {
        // 遍历 nums1，计算 nums2+nums3+nums4 的和
        unordered_map<int, int> umap;
        
        // 统计 nums2+nums3+nums4 的所有可能和及其出现次数
        for (int b : nums2) {
            for (int c : nums3) {
                for (int d : nums4) {
                    umap[b + c + d]++;
                }
            }
        }
        
        int count = 0;
        
        // 遍历 nums1，查找目标值
        for (int a : nums1) {
            int target = -a;
            if (umap.find(target) != umap.end()) {
                count += umap[target];
            }
        }
        
        return count;
    }
};
```

**1+3 分组法解释：**
1. **构建哈希表**：遍历 nums2、nums3 和 nums4，计算所有可能的和 `b + c + d`，并记录每个和出现的次数
2. **查找匹配**：遍历 nums1，计算目标值 `-a`，然后查找目标值在哈希表中出现的次数
3. **累加结果**：将找到的次数累加到结果中

**1+3 分组法性能分析：**
- 时间复杂度：O(n³)，其中 n 是数组的长度
- 空间复杂度：O(n²)，需要额外空间存储哈希表
- 适用场景：数据规模较小，内存有限的情况

## 💡 六、 面试直击灵魂的防暴击拷问

* **灵魂拷问一：为什么分成 A+B 和 C+D 两组，而不是 A 算一组，B+C+D 算一组？**

    * **绝杀回答：**"如果是 1对3 分组，计算 A 只要 O(n)，但计算 B+C+D 依然需要 O(n^3)，整体时间复杂度还是 O(n^3)。只有采用 2对2 的极致对半分组，才能让两边的时间复杂度都降到 O(n^2)，这是性能收益最大化的切分方式。"

* **灵魂拷问二：这道题能用双指针做吗？为什么？**

    * **绝杀回答：**"不能，或者说极其不合适。双指针法的前提是数组必须有序。但这道题是四个独立的数组，如果分别排序后用多指针，不仅逻辑极其复杂，而且题目要求的是'组合的个数'，相同的元素是可以重复使用的，双指针在这种不需要去重且跨多个数组的场景下，远不如哈希表直接映射来得快准狠。"

* **灵魂拷问三：为什么哈希表要存储和值出现的次数，而不是只存储和值是否存在？**

    * **绝杀回答：**"因为 A+B 的同一个和值可能出现多次，比如 nums1=[1,1] 和 nums2=[-1,-1]，那么 A+B=-0 的和值出现了 4 次。当我们在 C+D 中找到一个和值为 0 的组合时，需要找到 4 个 A+B 的组合使它们相加等于 0。所以必须存储和值出现的次数，而不是仅仅判断和值是否存在。"

## 📚 七、 总结

本题是哈希表的经典应用，通过分组和哈希表的结合，我们成功将时间复杂度从 $O(n^4)$ 优化到了 $O(n^2)$。

### 核心要点：
1. **分组降维**：将四维问题转化为两个二维问题，通过 $A + B = -(C + D)$ 的变形，实现降维打击
2. **2+2 分组**：选择最优的分组策略，使两边的时间复杂度都降到 $O(n^2)$
3. **哈希表计数**：使用哈希表记录 A+B 的和值及其出现次数，实现快速查找
4. **空间换时间**：以 $O(n^2)$ 的空间复杂度换取时间复杂度的优化
5. **无需去重**：题目只要求返回组合个数，无需去重，这是使用哈希表的关键前提

### 扩展思考：
- **哈希表的应用**：哈希表在解决计数、查找等问题时非常高效，是算法面试中的常用工具
- **分组策略**：在处理多数组或多变量问题时，合理的分组策略可以显著降低时间复杂度
- **时间复杂度与空间复杂度的权衡**：在选择算法时，需要根据具体情况权衡时间和空间复杂度
- **问题变形**：如果题目要求去重，或者返回具体的元组，解法会有所不同

### 方法比较：

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
