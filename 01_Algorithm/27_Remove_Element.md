# 算法实战复盘：移除元素 (LeetCode 27)

> **打卡日期**：2026-03-26 (Day 2)
> **核心主题**：数组内存连续性、同向双指针 (快慢指针)、In-place (原地) 操作。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给定一个数组 `nums` 和一个值 `val`，你需要 **原地 (In-place)** 移除所有数值等于 `val` 的元素，并返回移除后数组的新长度。
* **核心限制（大厂极爱考）：**
  1. **不能使用额外的数组空间**：必须仅使用 $O(1)$ 的额外空间并原地修改输入数组。
  2. **元素的顺序可以改变**：你不需要考虑数组中超出新长度后面的元素。
* **底层陷阱：** 数组在内存中是连续分布的，**数组的元素绝对不能被真正的“物理删除”，只能被“覆盖”**。如果遇到目标值就把后面的元素整体往前挪一位（嵌套两个 `for` 循环），时间复杂度会飙升到极端的 $O(n^2)$。

---

## 📊 二、 复杂度深度剖析 (面试必推导)

### 1. 时间复杂度：$O(n)$
**推导过程（为什么是线性时间？）：**
* 假设数组 `nums` 的总长度为 $N$。
* 我们的算法由一个单独的 `for` 循环构成，由 `fast` 指针主导遍历。
* 无论 `nums[fast]` 是否等于 `val`，`fast` 指针都会从索引 $0$ 一直走到索引 $N-1$。这意味着循环体精确执行了 $N$ 次。
* 在循环体内部，只有简单的条件判断 `if (nums[fast] != val)` 和一次数组赋值覆盖 `nums[slow++] = nums[fast]`，这些操作都是一步到位的常数时间，即 $O(1)$。
* **结论：** 总时间 = 循环次数 $\times$ 单次操作时间 = $N \times O(1) = O(N)$。抹去常数项和系数，最终的时间复杂度就是极其优秀的 $O(n)$。我们用一层循环，完成了原本需要两层循环（找元素+搬运整个后续数组）才能做完的事！

### 2. 空间复杂度：$O(1)$
**推导过程（为什么叫原地操作 In-place？）：**
* 在整个程序的生命周期内，我们仅仅向操作系统申请了两个整型变量（也就是两个指针的内存）：`int fast` 和 `int slow`。
* 无论输入的数组是 10 个元素，还是 10 亿个元素，我们所消耗的额外内存空间永远只是这两个 `int` 类型的字节（通常是 8 个字节）。没有开辟任何新的数组、哈希表或队列。
* **结论：** 额外空间消耗是一个绝对的常数，不随数据规模 $N$ 变化，因此空间复杂度为 $O(1)$。完美契合题目极其严苛的内存限制要求。

## 🎯 三、 核心思想：双指针的降维打击



这道题的精髓在于 **“角色分工”** 。我们让两个指针同向奔赴，但赋予它们完全不同的使命：

* **快指针 (`fast`)：无情的搜寻者。** 它的任务是跑遍整个数组，像排雷一样，跳过所有等于 `val` 的“废弃数据”，只去抓取我们需要保留的“好数据”。
* **慢指针 (`slow`)：老实的搬砖工。** 它的任务是指向“新数组”当前应该安放元素的最新位置。只要快指针扔过来一个“好数据”，慢指针就把它接住并覆盖在自己当前的位置，然后默默向前走一步，准备接下一个。

---

## 💻 四、 核心代码

```cpp
class Solution {
public:
    int removeElement(vector<int>& nums, int val) {
        int fast = 0; // 侦察兵：负责寻找不在黑名单里的好数据
        int slow = 0; // 搬砖工：负责记录好数据应该安放的位置

        // 快指针以恒定的速度，雷打不动地遍历整个数组
        for (fast = 0; fast < nums.size(); fast++) {
            
            // 💡 核心逻辑：如果快指针发现当前数据不是我们要删的 val（说明是好数据）
            if (nums[fast] != val) {
                // 就把这个好数据，覆盖到慢指针当前所在的位置
                // 并且慢指针向前走一步，准备迎接下一个好数据
                nums[slow] = nums[fast]; 
                slow++; 
                
                // 也可以优雅地简写为一行：nums[slow++] = nums[fast];
            }
        }
        
        // 当快指针走完整个数组，慢指针停留的位置，恰好就是新数组的长度！
        return slow;
    }
};
```
## 💡 五、 面试核心考点与防暴击指南
* **灵魂拷问：** 如果调用 C++ 标准库的 vector.erase() 来删除元素，时间复杂度是多少？必须秒答： 是 $O(n^2)$！很多初学者以为调个库函数就是 $O(1)$，其实 erase() 的底层实现是：一旦删除了某个元素，它会自动把后面所有的元素都往前整体挪动一位。如果数组里有大量需要删除的元素，每次 erase 都要搬运一次后面的大部队，这会造成极其恐怖的性能开销。这也正是为什么要用双指针法的原因！
  
* **细节拉满：** 最终返回的为什么是 slow 而不是 slow + 1 或者 slow - 1？因为 slow 代表的是“下一个将要被写入的位置索引”。比如当 slow 变成 3 时，意味着索引 0、1、2 都已经装满了“好数据”，此时数组的有效长度恰好就是 3！所以直接 return slow; 即可。
