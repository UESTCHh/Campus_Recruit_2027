# 算法实战复盘：有效的字母异位词 (LeetCode 242)

> **打卡日期**：2026-04-01 (Day 6 上午)
> **核心主题**：哈希表、数组作为极简哈希表、ASCII 码映射。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给定两个字符串 `s` 和 `t`，编写一个函数来判断 `t` 是否是 `s` 的字母异位词。（若 `s` 和 `t` 中每个字符出现的次数都相同，则称 `s` 和 `t` 互为字母异位词，字母异位词是通过重新排列不同单词或短语的字母而形成的单词或短语，并使用所有原字母一次。）。
* **核心痛点：** 字符串可能很长，如果对每个字符都去另一个字符串里遍历寻找，时间复杂度会爆炸。
* **大厂面试限制：** 要求极高的执行效率。虽然可以用 `std::unordered_map` 甚至直接对字符串进行排序 `std::sort` 来解决，但面试官最想看到的是**用固定大小的数组来模拟哈希表**的极致性能。
* **题目提示：**
  - 1 <= s.length, t.length <= 5 * 10^4
  - s 和 t 仅包含小写字母

---

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n)$
**推导过程：** 只需要完整遍历字符串 `s` 和 `t` 各一次（长度为 $n$），然后再遍历一次大小恒为 26 的哈希数组。总操作次数线性正比于字符串长度，完美达到 $O(n)$。

### 2. 空间复杂度：$O(1)$
**推导过程：** 无论输入的字符串有多长，我们始终只需要一个大小为 26 的 `int` 数组来记录字母频率。内存开销是常数级别的，所以空间复杂度为 $O(1)$。

## 🤔 三、 解题思路分析

### 3.1 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：对于字符串 `t` 中的每个字符，在字符串 `s` 中查找并标记，时间复杂度 O(n²)
2. **排序法**：对两个字符串进行排序，然后比较是否相等，时间复杂度 O(nlogn)
3. **哈希表法**：使用哈希表记录字符出现的频率，时间复杂度 O(n)

### 3.2 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n²) | O(1) | 思路简单，不需要额外空间 | 时间复杂度高，对于长字符串效率极低 |
| 排序法 | O(nlogn) | O(1) 或 O(n) | 代码简洁，逻辑清晰 | 排序的时间复杂度较高，对于长字符串效率不够理想 |
| 哈希表法 | O(n) | O(1) (固定大小数组) | 时间复杂度最优，效率极高 | 需要理解哈希映射的概念 |

### 3.3 数据大小对方法选择的影响
- **小型数据**（字符串长度 < 100）：三种方法的差异不大，排序法可能更简单
- **中型数据**（字符串长度 100-1000）：哈希表法开始展现优势
- **大型数据**（字符串长度 > 1000）：暴力法会严重超时，排序法也会有明显延迟，哈希表法是唯一选择
根据题目提示，字符串长度可以达到 5 * 10^4，这是一个相当大的数据规模，需要我们选择高效的算法：

- **暴力法**：时间复杂度 O(n²)，对于 n = 5 * 10^4，操作次数会达到 2.5 * 10^9，这会导致严重超时
- **排序法**：时间复杂度 O(nlogn)，对于 n = 5 * 10^4，操作次数约为 5 * 10^4 * 16 ≈ 8 * 10^5，虽然可以通过，但效率不够理想
- **哈希表法**：时间复杂度 O(n)，对于 n = 5 * 10^4，操作次数约为 5 * 10^4，效率最高，是处理大数据的最佳选择

**实际测试验证：**
- 暴力法：在 LeetCode 上会直接超时
- 排序法：在 LeetCode 上可以通过，但执行时间较长
- 哈希表法：在 LeetCode 上执行时间极短，通常在 0-4ms 之间

### 3.4 为什么选择哈希法

哈希法的优势在于：
1. **时间复杂度最优**：O(n)，线性时间处理
2. **空间使用合理**：使用固定大小的数组，空间复杂度 O(1)
3. **适合处理大输入**：无论字符串多长，处理时间都线性增长
4. **易于理解和实现**：一旦掌握哈希映射的概念，代码非常简洁

## 🎯 四、 核心思想：极简的哈希映射

### 4.1 哈希映射的概念

哈希表的本质，就是通过一个哈希函数（Hash Function），把任意的“键（Key）”映射为一个数组的“索引（Index）”。这样我们就可以通过索引快速访问对应的值，而不需要遍历整个数据结构。

### 4.2 本题的哈希函数

在这道题中，因为题目明确指出“只包含小写字母”，由于 a 到 z 的 ASCII 值是连续的，我们拥有一个天然且完美的哈希函数：**`字符 - 'a'`**。
- 字母 `'a'` 对应索引 `0` (`'a' - 'a'`)
- 字母 `'b'` 对应索引 `1` (`'b' - 'a'`)
- ...
- 字母 `'z'` 对应索引 `25` (`'z' - 'a'`)

### 4.3 战术执行步骤

1. **初始化哈希表**：定义一个大小为 26 的数组 `record`，全部初始化为 0。
2. **统计频率**：
   - 遍历字符串 `s`，遇到字母就把对应位置的数字 **+1**
   - 遍历字符串 `t`，遇到字母就把对应位置的数字 **-1**
3. **检查结果**：最终检查 `record` 数组，如果所有元素都是 0，说明它是完美的异位词；只要有一个非 0，直接判负！

### 4.4 示例演示

以 `s = "anagram", t = "nagaram"` 为例：

1. 初始化 `record = [0, 0, 0, ..., 0]`（26个0）
2. 遍历 `s`：
   - 'a' → record[0] = 1
   - 'n' → record[13] = 1
   - 'a' → record[0] = 2
   - 'g' → record[6] = 1
   - 'r' → record[17] = 1
   - 'a' → record[0] = 3
   - 'm' → record[12] = 1
3. 遍历 `t`：
   - 'n' → record[13] = 0
   - 'a' → record[0] = 2
   - 'g' → record[6] = 0
   - 'a' → record[0] = 1
   - 'r' → record[17] = 0
   - 'a' → record[0] = 0
   - 'm' → record[12] = 0
4. 检查 `record` 数组，所有元素都是 0，返回 true

## 💻 五、 核心代码

### 5.1 哈希表法（数组实现）

```cpp
class Solution {
public:
    bool isAnagram(string s, string t) {
        // 剪枝优化：如果长度不同，直接返回false
        // 因为异位词必须长度相同
        if (s.length() != t.length()) return false;

        // 定义一个大小为26的数组作为哈希表
        // 索引0-25分别对应字母a-z
        int record[26] = {0}; 

        // 遍历两个字符串，同时统计频率
        // s中的字符使对应位置+1，t中的字符使对应位置-1
        // 这样如果是异位词，最终所有位置都会回到0
        for (int i = 0; i < s.length(); i++) {
            // 计算字符对应的索引：字符 - 'a'
            // 例如 'a' - 'a' = 0, 'b' - 'a' = 1, 以此类推
            record[s[i] - 'a']++; // s 负责增加词频
            record[t[i] - 'a']--; // t 负责抵消词频
        }

        // 检查哈希表是否完全归零
        // 如果有任何位置不为0，说明两个字符串的字符频率不同
        for (int i = 0; i < 26; i++) {
            if (record[i] != 0) {
                return false;
            }
        }

        // 所有位置都为0，说明是异位词
        return true;
    }
};
```

### 5.2 排序法实现

```cpp
class Solution {
public:
    bool isAnagram(string s, string t) {
        // 剪枝优化：如果长度不同，直接返回false
        if (s.length() != t.length()) return false;
        
        // 对两个字符串进行排序
        sort(s.begin(), s.end());
        sort(t.begin(), t.end());
        
        // 比较排序后的字符串是否相等
        return s == t;
    }
};
```

### 5.3 暴力法实现

```cpp
class Solution {
public:
    bool isAnagram(string s, string t) {
        // 剪枝优化：如果长度不同，直接返回false
        if (s.length() != t.length()) return false;
        
        // 复制s到一个临时字符串
        string temp = s;
        
        // 遍历t中的每个字符
        for (char c : t) {
            // 在临时字符串中查找并删除该字符
            size_t pos = temp.find(c);
            if (pos == string::npos) {
                // 如果找不到，说明不是异位词
                return false;
            }
            temp.erase(pos, 1);
        }
        
        // 所有字符都找到了，是异位词
        return true;
    }
};
```

## 💡 六、 面试直击灵魂的防暴击拷问
* **灵魂拷问一：为什么这道题不用 std::unordered_map<char, int>？它的时间复杂度不也是 $O(1)$ 吗？**
  * **绝杀回答：**“虽然 unordered_map 的平均查找时间也是 $O(1)$，但它的底层维护了复杂的哈希桶和链表结构，计算哈希值、处理哈希冲突以及动态分配内存都需要极大的常数级时间开销。在这道题已知键的范围极小且连续（仅 26 个小写字母）的情况下，直接开辟一段连续的栈内存数组做直接映射，运行速度会比 unordered_map 快几十倍。在底层基础架构开发中，能用数组解决的哈希映射，绝对不用 map。”

* **灵魂拷问二：如果输入字符串包含 Unicode 字符（比如中文）怎么办？** 
  * **绝杀回答：**“如果包含 Unicode，字符集的空间会变得极其庞大（上百万个代码点），此时再用固定大小的数组就会造成极其恐怖的内存浪费（且很多位置用不上）。这个时候，我们就必须切换回 std::unordered_map，利用其动态扩容的特性来按需记录字符频率。”

* **灵魂拷问三：如果面试官要求你不使用额外空间，怎么办？** 
  * **绝杀回答：**“如果不允许使用额外空间，我们可以使用排序法。虽然排序的时间复杂度是 O(nlogn)，但它不需要额外的空间（除了排序本身可能需要的栈空间）。我们可以对两个字符串进行排序，然后比较它们是否相等。”

* **灵魂拷问四：如何处理大小写敏感的情况？** 
  * **绝杀回答：**“如果题目要求大小写敏感，我们可以将哈希表的大小扩展到 52（26个大写字母 + 26个小写字母），然后使用 `tolower` 或 `toupper` 函数将所有字符转换为统一大小写后再处理。或者，我们可以直接使用两个不同的哈希表分别处理大写和小写字母。”

## 📚 七、 总结

本题是哈希表的经典入门题，通过巧妙利用数组作为极简哈希表，我们可以在 O(n) 的时间复杂度和 O(1) 的空间复杂度下解决问题。

### 核心要点：
1. **哈希映射**：利用 `字符 - 'a'` 作为哈希函数，将字符映射到数组索引
2. **频率统计**：通过一次遍历同时统计两个字符串的字符频率
3. **剪枝优化**：首先检查字符串长度是否相同，不同则直接返回 false
4. **去重处理**：由于题目要求字符出现次数相同，不需要额外去重

### 扩展思考：
- 哈希表是一种非常重要的数据结构，在很多算法问题中都有广泛应用
- 对于有限范围的键，使用数组作为哈希表是最高效的选择
- 在实际开发中，我们需要根据具体情况选择合适的数据结构，权衡时间和空间复杂度

通过本题的学习，我们不仅掌握了一种高效的算法，更重要的是理解了哈希表的基本原理和应用场景，为解决更复杂的问题打下了基础。
