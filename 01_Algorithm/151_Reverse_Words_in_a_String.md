# 算法实战复盘：反转字符串中的单词 (LeetCode 151)

> **打卡日期**：2026-04-06
> **核心主题**：双指针法、字符串处理、原地操作。

---

## 📝 一、 题目描述与核心要求

### 1. 中文描述
给你一个字符串 `s` ，请你反转字符串中 **单词** 的顺序。

**单词** 是由非空格字符组成的字符串。`s` 中使用至少一个空格将字符串中的 **单词** 分隔开。

返回 **单词顺序颠倒且单词之间用单个空格连接的结果字符串**。

**注意：**
- 输入字符串 `s` 中可能会存在前导空格、尾随空格或者单词间的多个空格。
- 返回的结果字符串中，单词间应当仅用单个空格分隔，且不包含任何额外的空格。

### 2. 英文描述
Given an input string `s`, reverse the order of the **words**.

A **word** is defined as a sequence of non-space characters. The **words** in `s` will be separated by at least one space.

Return *a string of the words in reverse order concatenated by a single space*.

**Note:**
- `s` may contain leading or trailing spaces or multiple spaces between two words. 
- The returned string should only have a single space separating the words. Do not include any extra spaces.

### 3. 输入示例
- **示例 1**：
  - 输入：`s = "the sky is blue"`
  - 输出：`"blue is sky the"`

- **示例 2**：
  - 输入：`s = "  hello world  "`
  - 输出：`"world hello"`

- **示例 3**：
  - 输入：`s = "a good   example"`
  - 输出：`"example good a"`

### 4. 核心痛点
- 处理字符串中的前导空格、尾随空格和多个连续空格
- 反转单词顺序的同时保持单词内部字符顺序不变
- 确保返回的字符串中单词间仅用单个空格分隔
- 优化时间和空间复杂度

### 5. 题目提示
- 1 <= s.length <= 10^4
- s contains English letters (upper-case and lower-case), digits, and spaces ' '.
- There is at least one word in s.

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：O(n)
**推导过程**：无论使用哪种方法，都需要遍历字符串中的每个字符至少一次，因此时间复杂度为 O(n)，其中 n 是字符串的长度。

### 2. 空间复杂度：O(n) 或 O(1)
**推导过程**：
- 使用额外空间的方法：需要 O(n) 的空间来存储结果
- 原地修改的方法：只需要 O(1) 的额外空间（不考虑输出字符串的空间）

## 🤔 三、 解题思路分析

### 1. 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：分割字符串为单词数组，反转数组，然后重新拼接
2. **双指针法**：原地修改字符串，先反转整个字符串，再反转每个单词
3. **栈方法**：将单词压入栈中，然后弹出拼接

### 2. 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n) | O(n) | 思路简单，易于实现 | 需要额外空间存储单词数组 |
| 双指针法 | O(n) | O(1) | 原地修改，空间复杂度最优 | 实现较复杂，需要处理空格 |
| 栈方法 | O(n) | O(n) | 思路清晰 | 需要额外空间存储栈 |

### 3. 数据大小对方法选择的影响

根据题目提示，字符串长度最大为 10^4：

- **暴力法**：需要 O(n) 的额外空间，对于 n=10^4，这会占用约 10KB 的额外内存，在现代计算机中可以接受
- **双指针法**：只使用常数级额外空间，对于任何大小的字符串都适用
- **栈方法**：需要 O(n) 的额外空间，与暴力法类似

**实际测试验证**：
- 暴力法：实现简单，运行时间可接受
- 双指针法：空间使用少，但实现较复杂
- 栈方法：实现相对简单，但空间使用较多

### 4. 方法选择建议

**推荐使用双指针法**，原因如下：
1. 空间复杂度 O(1)，符合原地修改的思想
2. 时间复杂度 O(n)，效率高
3. 不需要额外空间，适合处理大字符串
4. 能够锻炼字符串处理和双指针技巧

## 🎯 四、 核心思想：双指针法

双指针法是解决字符串中单词反转问题的最优方法。它通过以下步骤实现：

### 1. 核心原理

1. **移除多余空格**：使用双指针法移除字符串中的前导空格、尾随空格和多个连续空格
2. **反转整个字符串**：将整个字符串反转
3. **反转每个单词**：将每个单词单独反转，恢复单词内部的字符顺序

### 2. 算法步骤详解

1. **移除多余空格**：
   - 初始化指针 `slow = 0`，遍历字符串
   - 跳过前导空格
   - 对于非空格字符，复制到 `slow` 位置，然后 `slow++`
   - 如果遇到空格且前一个字符不是空格，添加一个空格，然后 `slow++`
   - 最后调整字符串长度为 `slow`

2. **反转整个字符串**：
   - 使用双指针法反转从 0 到 `slow-1` 的字符

3. **反转每个单词**：
   - 初始化指针 `start = 0`，遍历字符串
   - 找到每个单词的结束位置 `end`
   - 反转从 `start` 到 `end-1` 的字符
   - 更新 `start = end + 1`

### 3. 示例分析

以示例 1 为例：`s = "the sky is blue"`

**步骤 1**：移除多余空格（此例无多余空格）
- 字符串保持不变：`"the sky is blue"`

**步骤 2**：反转整个字符串
- 得到：`"eulb si yks eht"`

**步骤 3**：反转每个单词
- 反转 "eulb" → "blue"
- 反转 "si" → "is"
- 反转 "yks" → "sky"
- 反转 "eht" → "the"
- 最终结果：`"blue is sky the"`

### 4. 边界情况处理

1. **空字符串**：题目保证至少有一个单词，所以不需要处理
2. **只有一个单词的字符串**：反转整个字符串后再反转该单词，结果不变
3. **多个连续空格**：在移除多余空格步骤中处理
4. **前导和尾随空格**：在移除多余空格步骤中处理

### 5. 与其他方法的对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 双指针法 | O(n) | O(1) | 原地修改，空间复杂度最优 | 实现较复杂 |
| 暴力法 | O(n) | O(n) | 思路简单，易于实现 | 需要额外空间 |
| 栈方法 | O(n) | O(n) | 思路清晰 | 需要额外空间 |

### 6. 关键技术点

1. **双指针移除空格**：使用快慢指针移除多余空格
2. **字符串反转**：使用双指针法反转字符串的指定范围
3. **单词边界识别**：正确识别单词的开始和结束位置
4. **原地修改**：不使用额外空间，直接修改原字符串

## 💻 五、 核心代码

### 1. 双指针法实现（原地修改，推荐）

```cpp
#include <string>
using namespace std;

class Solution {
public:
    string reverseWords(string s) {
        // 步骤1：反转整个字符串
        reverse(s.begin(), s.end());

        int n = s.size();
        int idx = 0; // 记录当前有效字符的位置
        
        // 步骤2：遍历字符串，处理每个单词
        for (int start = 0; start < n; ++start) {
            if (s[start] != ' ') { // 找到单词的开始位置
                // 在单词前添加空格（除了第一个单词）
                if (idx != 0) s[idx++] = ' ';

                // 步骤3：复制单词到有效位置
                int end = start;
                while (end < n && s[end] != ' ') {
                    s[idx++] = s[end++];
                }

                // 步骤4：反转当前单词
                reverse(s.begin() + idx - (end - start), s.begin() + idx);

                // 步骤5：更新start，跳过已处理的单词
                start = end;
            }
        }
        
        // 步骤6：删除多余的字符
        s.erase(s.begin() + idx, s.end());
        return s;
    }
};
```

**代码解释**：
1. **反转整个字符串**：使用标准库的 `reverse` 函数将整个字符串反转，这样单词的顺序就颠倒了
2. **遍历字符串**：`start` 指针用于寻找单词的开始位置
3. **处理单词**：当找到非空格字符时，说明找到了一个单词的开始
4. **添加空格**：如果不是第一个单词，在前面添加一个空格
5. **复制单词**：将单词从 `start` 位置复制到 `idx` 位置
6. **反转单词**：将复制后的单词反转，恢复其正确的字符顺序
7. **更新指针**：将 `start` 移动到单词的结束位置，继续寻找下一个单词
8. **删除多余字符**：使用 `erase` 函数删除 `idx` 之后的多余字符

### 2. 详细步骤解析

以示例 2 为例：`s = "  hello world  "`

**步骤 1**：反转整个字符串
- 原字符串：`"  hello world  "`
- 反转后：`"  dlrow olleh  "`

**步骤 2**：遍历字符串，处理每个单词
- 初始化 `idx = 0`
- `start = 0`：遇到空格，跳过
- `start = 1`：遇到空格，跳过
- `start = 2`：遇到非空格字符 'd'，开始处理第一个单词

**步骤 3**：处理第一个单词 "dlrow"
- `idx = 0`，不需要添加空格
- 复制单词：将 "dlrow" 复制到 `idx` 位置
- `idx` 变为 5
- 反转单词：将 "dlrow" 反转为 "world"
- 更新 `start = 7`（单词结束位置）

**步骤 4**：处理第二个单词 "olleh"
- `idx = 5`，添加一个空格，`idx` 变为 6
- 复制单词：将 "olleh" 复制到 `idx` 位置
- `idx` 变为 11
- 反转单词：将 "olleh" 反转为 "hello"
- 更新 `start = 13`（单词结束位置）

**步骤 5**：处理剩余空格
- `start = 13`：遇到空格，跳过
- `start = 14`：遇到空格，跳过
- 遍历结束

**步骤 6**：删除多余字符
- `idx = 11`，删除从 11 到末尾的字符
- 最终结果：`"world hello"`

所以解题思路如下：

   * 移除多余空格
   * 将整个字符串反转
   * 将每个单词反转
```cpp
class Solution {
    public:
    void reverse(string& s, int start, int end){ //翻转，区间写法：左闭右闭 []
        for (int i = start, j = end; i < j; i++, j--) {
            swap(s[i], s[j]);
        }
    }

    void removeExtraSpaces(string& s) {//去除所有空格并在相邻单词之间添加空格, 快慢指针。
        int slow = 0;   //整体思想参考https://programmercarl.com/0027.移除元素.html
        for (int i = 0; i < s.size(); ++i) { //
            if (s[i] != ' ') { //遇到非空格就处理，即删除所有空格。
                if (slow != 0) s[slow++] = ' '; //手动控制空格，给单词之间添加空格。slow != 0说明不是第一个单词，需要在单词前添加空格。
                while (i < s.size() && s[i] != ' ') { //补上该单词，遇到空格说明单词结束。
                    s[slow++] = s[i++];
                }
            }
        }
        s.resize(slow); //slow的大小即为去除多余空格后的大小。
    }

    string reverseWords(string s) {
        removeExtraSpaces(s); //去除多余空格，保证单词之间之只有一个空格，且字符串首尾没空格。
        reverse(s, 0, s.size() - 1);
        int start = 0; //removeExtraSpaces后保证第一个单词的开始下标一定是0。
        for (int i = 0; i <= s.size(); ++i) {
            if (i == s.size() || s[i] == ' ') { //到达空格或者串尾，说明一个单词结束。进行翻转。
                reverse(s, start, i - 1); //翻转，注意是左闭右闭 []的翻转。
                start = i + 1; //更新下一个单词的开始下标start
            }
        }
        return s;
    }
    };
```
> **💡 面试直击灵魂的追问：`s.resize(slow);` 真的把多余的内存还给操作系统了吗？**
> **绝杀回答：** “没有！在 C++ 标准库中，`std::string` 和 `std::vector` 的 `resize()` 方法只会改变对象的逻辑大小（`size`），并在末尾添加 `\0`，但**绝不会减小其物理容量（`capacity`）**。这是为了避免频繁分配内存。如果我们需要彻底释放这部分多余的内存，必须使用 **Shrink-to-Fit 惯用法**：`s.shrink_to_fit();` （C++11 引入）。这样能向操作系统申请一块精确大小的新内存并转移数据，真正做到空间极限压榨。”

### 3. 暴力法实现（使用额外空间）

```cpp
#include <string>
#include <vector>
#include <sstream>
using namespace std;

class Solution {
public:
    string reverseWords(string s) {
        // 分割字符串为单词数组
        vector<string> words;
        stringstream ss(s);
        string word;
        while (ss >> word) {
            words.push_back(word);
        }
        
        // 反转单词数组
        reverse(words.begin(), words.end());
        
        // 拼接单词
        string result;
        for (int i = 0; i < words.size(); i++) {
            if (i != 0) {
                result += " ";
            }
            result += words[i];
        }
        
        return result;
    }
};
```

**暴力法性能分析**：
- 时间复杂度：O(n)，需要遍历字符串和单词数组
- 空间复杂度：O(n)，需要额外的单词数组和结果字符串
- 适用场景：实现简单，适合处理中等大小的字符串

### 4. 栈方法实现（使用额外空间）

```cpp
#include <string>
#include <stack>
using namespace std;

class Solution {
public:
    string reverseWords(string s) {
        stack<string> st;
        string word;
        
        // 将单词压入栈中
        for (int i = 0; i < s.size(); i++) {
            if (s[i] != ' ') {
                word += s[i];
            } else if (!word.empty()) {
                st.push(word);
                word = "";
            }
        }
        
        // 处理最后一个单词
        if (!word.empty()) {
            st.push(word);
        }
        
        // 弹出栈拼接单词
        string result;
        while (!st.empty()) {
            result += st.top();
            st.pop();
            if (!st.empty()) {
                result += " ";
            }
        }
        
        return result;
    }
};
```

**栈方法性能分析**：
- 时间复杂度：O(n)，需要遍历字符串和栈
- 空间复杂度：O(n)，需要额外的栈和结果字符串
- 适用场景：思路清晰，适合处理中等大小的字符串

## ❌ 七、 常见错误与注意事项

### 1. 常见错误

1. **空格处理错误**：没有正确处理前导空格、尾随空格和多个连续空格
2. **单词边界识别错误**：无法正确识别单词的开始和结束位置
3. **反转逻辑错误**：在反转整个字符串后，没有正确反转每个单词
4. **字符串长度调整错误**：在移除多余空格后，没有正确调整字符串长度
5. **原地修改时的内存访问错误**：在修改字符串时，越界访问内存

### 2. 注意事项

1. **空格处理**：使用双指针法移除多余空格时，要注意单词之间只保留一个空格
2. **字符串长度**：在移除多余空格后，需要使用 `resize` 方法调整字符串长度
3. **反转顺序**：先反转整个字符串，再反转每个单词
4. **边界条件**：处理只有一个单词的情况和空字符串的情况
5. **性能优化**：对于大字符串，双指针法是最优选择

### 2. 解题思路

#### 2.1 反转字符串 II

**问题描述**：给定一个字符串 s 和一个整数 k，从字符串开头算起，每 2k 个字符反转前 k 个字符。如果剩余字符少于 k 个，则反转剩余字符。如果剩余字符小于 2k 但大于或等于 k 个，则反转前 k 个字符，其余字符保持原样。

**解题思路**：
- 使用双指针法，每次处理 2k 个字符
- 对于每个 2k 区间，反转前 k 个字符
- 处理剩余字符的情况

**解题思路**：
- 使用双指针法，从两端向中间交换字符
- 时间复杂度 O(n)，空间复杂度 O(1)

#### 2.3 反转元音字母

**问题描述**：编写一个函数，以字符串作为输入，反转该字符串中的元音字母。

**解题思路**：
- 使用双指针法，左指针从左向右，右指针从右向左
- 当左右指针都指向元音字母时，交换它们
- 继续移动指针，直到左指针超过右指针

## 💡 九、 面试直击灵魂的防暴击拷问

* **灵魂拷问一：为什么双指针法是解决反转字符串中单词问题的最优方法？**
    * **绝杀回答：**"双指针法是最优方法，因为它可以原地修改字符串，空间复杂度为 O(1)，同时时间复杂度为 O(n)，只需要遍历字符串几次。相比暴力法和栈方法，它不需要额外空间，适合处理大字符串。此外，双指针法还能锻炼字符串处理和指针操作的能力，是面试中的常用方法。"

* **灵魂拷问二：如何处理字符串中的多余空格？**
    * **绝杀回答：**"处理多余空格可以使用双指针法：
      1. 初始化慢指针 `slow = 0`，快指针 `i` 遍历整个字符串
      2. 跳过前导空格
      3. 对于非空格字符，复制到 `slow` 位置，然后 `slow++`
      4. 如果遇到空格且前一个字符不是空格，添加一个空格，然后 `slow++`
      5. 最后调整字符串长度为 `slow`
      这样可以移除前导空格、尾随空格和多个连续空格，只保留单词间的单个空格。"

* **灵魂拷问三：反转字符串中单词的顺序的步骤是什么？**
    * **绝杀回答：**"反转字符串中单词的顺序可以分为三个步骤：
      1. 移除多余空格：使用双指针法移除前导空格、尾随空格和多个连续空格
      2. 反转整个字符串：将整个字符串反转，使得单词顺序颠倒
      3. 反转每个单词：将每个单词单独反转，恢复单词内部的字符顺序
      这样可以在原地完成操作，不需要额外空间。"

* **灵魂拷问四：如果允许使用额外空间，还有哪些方法可以解决这个问题？**
    * **绝杀回答：**"如果允许使用额外空间，可以使用以下方法：
      1. 暴力法：分割字符串为单词数组，反转数组，然后重新拼接
      2. 栈方法：将单词压入栈中，然后弹出拼接
      3. 使用字符串流：使用 `stringstream` 分割字符串，然后反转单词顺序
      这些方法实现简单，但需要额外空间。"

* **灵魂拷问五：如何处理只有一个单词的字符串？**
    * **绝杀回答：**"对于只有一个单词的字符串，处理过程如下：
      1. 移除多余空格（如果有）
      2. 反转整个字符串：此时单词被反转
      3. 反转每个单词：将反转后的单词再次反转，恢复原样
      最终结果与原字符串相同（去除多余空格后）。"

3. **反转每个单词**：恢复单词内部的字符顺序
4. **原地修改**：不使用额外空间，直接修改原字符串
