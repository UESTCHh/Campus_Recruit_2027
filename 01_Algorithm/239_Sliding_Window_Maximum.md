## 一、 题目描述与核心要求

### 1. 题目原文
给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

返回滑动窗口中的最大值。

### 2. 示例说明

#### 示例 1：
**输入**：nums = [1,3,-1,-3,5,3,6,7], k = 3
**输出**：[3,3,5,5,6,7]

**解释**：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7

#### 示例 2：
**输入**：nums = [1], k = 1
**输出**：[1]

### 3. 约束条件
- 1 <= nums.length <= 10^5
- -10^4 <= nums[i] <= 10^4
- 1 <= k <= nums.length

### 4. 进阶要求
你能在线性时间复杂度内解决此题吗？

## 二、 解题思路分析

### 1. 问题分析
- **核心需求**：在数组上维护一个大小为 k 的滑动窗口，每次向右移动一位，返回每个窗口的最大值
- **输入特点**：数组长度可能很大（10^5），需要高效算法
- **难点**：如何高效维护窗口最大值

### 2. 初学者想法

#### 2.1 暴力解法
- **思路**：遍历数组，每次对窗口内的 k 个元素进行一次遍历找最大值
- **时间复杂度**：O(n × k)
  - 对于长度为 n 的数组 nums，当滑动窗口大小为 k 时，窗口的数量为 n - k + 1
  - 每个窗口需要遍历 k 个元素找最大值
  - 总操作次数 = (n - k + 1) × k = nk - k² + k = O(nk)
- **空间复杂度**：O(1)
- **问题**：对于 n = 10^5，k = 10^4，时间复杂度达到 10^9，会超时

#### 2.2 暴力解法复杂度详细推导

##### 2.2.1 窗口数量分析
对于长度为 n 的数组，滑动窗口大小为 k：
- 第一个窗口：[0, k-1]
- 第二个窗口：[1, k]
- ...
- 最后一个窗口：[n-k, n-1]

窗口数量 = (n - k + 1)

##### 2.2.2 单次遍历分析
对于每个窗口，需要遍历 k 个元素找最大值：
- 比较次数：k - 1 次（两两比较）
- 总比较次数 = (n - k + 1) × (k - 1) ≈ (n - k + 1) × k

##### 2.2.3 总时间复杂度
T(n, k) = (n - k + 1) × k = nk - k² + k

当 n 和 k 都是 10^5 量级时：
T(n, k) ≈ 10^5 × 10^5 = 10^10

这个数字远远超出了计算机在一秒内能够处理的范围，所以暴力解法不可行。

#### 2.3 大顶堆（优先级队列）
- **思路**：使用大顶堆维护窗口内的元素，堆顶就是最大值
- **问题**：窗口移动时，需要移除移出窗口的元素，但大顶堆只能移除堆顶元素，无法移除其他元素
- **结论**：大顶堆不适用

### 3. 数据结构选择

#### 3.1 思考过程
我们需要一个队列，这个队列应该满足：
- 放进去窗口里的元素
- 随着窗口移动，队列也一进一出
- 每次移动后，队列能告诉我们里面的最大值是什么

设计如下接口：
```cpp
class MyQueue {
public:
    void pop(int value) {
        // 从队列中移除元素
    }
    void push(int value) {
        // 向队列中添加元素
    }
    int front() {
        return que.front(); // 返回队列中的最大值
    }
};
```

#### 3.2 关键洞察
队列没有必要维护窗口里的所有元素，只需要维护有可能成为窗口里最大值的元素，同时保证队列里的元素数值是由大到小的。

维护元素单调递减的队列就叫做**单调队列**，即单调递减或单调递增的队列。

### 4. 单调队列设计

#### 4.1 核心规则
- **pop(value)**：如果窗口移除的元素 value 等于单调队列的出口元素（最大值），那么队列弹出元素，否则不用任何操作
- **push(value)**：如果 push 的元素 value 大于入口元素的数值，那么就将队列入口的元素弹出，直到 push 元素的数值小于等于队列入口元素的数值为止

#### 4.2 数据结构选择
使用 deque（双端队列）最为合适，因为 deque 可以在两端进行高效的插入和删除操作。

## 三、 核心代码实现

### 1. 方法一：自定义单调队列

#### 1.1 代码实现
```cpp
class Solution {
private:
    // 自定义单调队列（从大到小）
    class MyQueue {
    public:
        deque<int> que; // 使用deque来实现单调队列

        // 每次弹出的时候，比较当前要弹出的数值是否等于队列出口元素的数值，如果相等则弹出
        // 同时pop之前判断队列当前是否为空
        void pop(int value) {
            if (!que.empty() && value == que.front()) {
                que.pop_front();
            }
        }

        // 如果push的数值大于入口元素的数值，那么就将队列后端的数值弹出，直到push的数值小于等于队列入口元素的数值为止
        // 这样就保持了队列里的数值是单调从大到小的了
        void push(int value) {
            while (!que.empty() && value > que.back()) {
                que.pop_back();
            }
            que.push_back(value);
        }

        // 查询当前队列里的最大值，直接返回队列前端也就是front就可以了
        int front() {
            return que.front();
        }
    };

public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        MyQueue que;
        vector<int> result;

        // 先将前k的元素放进队列
        for (int i = 0; i < k; i++) {
            que.push(nums[i]);
        }
        result.push_back(que.front()); // result 记录前k的元素的最大值

        // 移动窗口
        for (int i = k; i < nums.size(); i++) {
            que.pop(nums[i - k]); // 滑动窗口移除最前面元素
            que.push(nums[i]); // 滑动窗口添加最后面的元素
            result.push_back(que.front()); // 记录对应的最大值
        }

        return result;
    }
};
```

#### 1.2 代码解析
- **自定义单调队列类**：使用 deque 作为底层容器，实现 pop、push 和 front 操作
- **pop 操作**：检查要移除的元素是否是队列出口元素（最大值），如果是才弹出
- **push 操作**：维护单调递减的顺序，弹出所有比新元素小的元素，再加入新元素
- **front 操作**：直接返回队列前端元素，即当前窗口最大值
- **主函数逻辑**：先初始化前 k 个元素，然后滑动窗口，每次先 pop 再 push，最后记录最大值

#### 1.3 工作原理示例
以 nums = [1,3,-1,-3,5,3,6,7], k = 3 为例：

1. **初始化前3个元素**：
   - push(1)：队列为空，直接加入 → [1]
   - push(3)：3 > 1，弹出1，加入3 → [3]
   - push(-1)：-1 < 3，直接加入 → [3, -1]
   - front()：3 → result = [3]

2. **窗口移动，i=3（元素-3）**：
   - pop(1)：1 ≠ 3，不弹出 → [3, -1]
   - push(-3)：-3 < -1，直接加入 → [3, -1, -3]
   - front()：3 → result = [3, 3]

3. **窗口移动，i=4（元素5）**：
   - pop(3)：3 = 3，弹出 → [-1, -3]
   - push(5)：5 > -3，弹出-3；5 > -1，弹出-1；加入5 → [5]
   - front()：5 → result = [3, 3, 5]

4. **继续这个过程**，最终得到结果 [3, 3, 5, 5, 6, 7]

### 2. 方法二：直接使用 deque

#### 2.1 代码实现
```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> dq; // 存储的是索引，不是数值
        vector<int> result;

        for (int i = 0; i < nums.size(); i++) {
            // 移除比当前元素小的队列尾部元素，保持队列单调递减
            while (!dq.empty() && nums[i] > nums[dq.back()]) {
                dq.pop_back();
            }
            dq.push_back(i); // 加入当前元素的索引

            // 移除已经不在窗口内的队列头部元素
            if (dq.front() <= i - k) {
                dq.pop_front();
            }

            // 当窗口大小为k时，记录最大值
            if (i >= k - 1) {
                result.push_back(nums[dq.front()]);
            }
        }

        return result;
    }
};
```

#### 2.2 代码解析
- **存储索引而非数值**：这样可以更方便地判断元素是否在窗口内
- **维护单调递减**：每次加入新元素前，先移除所有比它小的元素
- **移除过期元素**：检查队列头部元素是否已经不在窗口内
- **记录最大值**：当窗口大小达到 k 时，队列头部就是当前窗口的最大值

#### 2.3 优势
- 代码更简洁，不需要自定义队列类
- 存储索引而非数值，更方便处理窗口边界
- 逻辑更紧凑，易于理解和维护

### 3. 方法三：使用 multiset

#### 3.1 代码实现
```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        multiset<int> window;
        vector<int> result;

        // 初始化窗口
        for (int i = 0; i < k; i++) {
            window.insert(nums[i]);
        }
        result.push_back(*window.rbegin()); // rbegin()指向最大元素

        // 滑动窗口
        for (int i = k; i < nums.size(); i++) {
            // 移除离开窗口的元素
            window.erase(window.find(nums[i - k]));
            // 加入新元素
            window.insert(nums[i]);
            // 记录最大值
            result.push_back(*window.rbegin());
        }

        return result;
    }
};
```

#### 3.2 代码解析
- **multiset 特性**：元素自动排序，可以有重复元素
- **初始化窗口**：将前 k 个元素加入 multiset
- **滑动窗口**：移除离开窗口的元素，加入新元素
- **获取最大值**：使用 rbegin() 获取反向迭代器，指向最大元素

#### 3.3 时间复杂度分析
- 每个元素的插入和删除都是 O(log k)
- 总共有 n 个元素
- 时间复杂度：O(n log k)
- 对于 n = 10^5，log k 约为 15，总操作数约为 1.5 × 10^6，可以通过但不如单调队列高效

## 四、 时间复杂度与空间复杂度分析

### 1. 方法一：自定义单调队列
- **时间复杂度**：O(n)
  - 虽然有嵌套的 while 循环，但每个元素最多被 push 和 pop 各一次
  - 总操作次数是线性的
- **空间复杂度**：O(k)
  - 单调队列中最多存储 k 个元素

### 2. 方法二：直接使用 deque
- **时间复杂度**：O(n)
  - 每个元素最多被 push 和 pop 各一次
  - 线性时间复杂度
- **空间复杂度**：O(k)
  - deque 中最多存储 k 个元素

### 3. 方法三：使用 multiset
- **时间复杂度**：O(n log k)
  - 每个元素的插入和删除都是 O(log k)
  - 总共有 n 个元素
- **空间复杂度**：O(k)
  - multiset 中存储 k 个元素

### 4. 算法对比
| 方法 | 时间复杂度 | 空间复杂度 | 优势 | 劣势 |
|------|------------|------------|------|------|
| 自定义单调队列 | O(n) | O(k) | 时间最优，适合大数据量 | 需要自定义数据结构 |
| 直接使用 deque | O(n) | O(k) | 代码简洁，时间最优 | 需要注意索引处理 |
| 使用 multiset | O(n log k) | O(k) | 代码最简洁，易于理解 | 时间复杂度较高 |

### 5. 单调队列时间复杂度详细分析（均摊分析）

#### 5.1 为什么看似 O(n²) 的算法实际上是 O(n)？
在方法二的主循环中，有一个 `while (!dq.empty() && nums[i] > nums[dq.back()])` 的嵌套循环，这看起来像是 O(n × k)。但实际上，每个元素最多只会被 push 和 pop 各一次。

#### 5.2 均摊分析
- **push 操作**：每个元素最多被 push 一次
- **pop 操作**：每个元素最多被 pop 一次
- **总操作次数**：最多 2n 次（n 次 push + n 次 pop）
- **总时间复杂度**：O(n)

#### 5.3 具体例子
考虑 nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3：

1. 初始化：push 1, 3 → 元素 1 被弹出
2. i=2: push -1 → -1 < 3，不弹出任何元素
3. i=3: push -3 → -3 < -1，不弹出任何元素
4. i=4: push 5 → 弹出 -3, -1, 3（4次操作）
5. i=5: push 3 → 3 < 5，不弹出
6. i=6: push 6 → 弹出 3, 5（2次操作）
7. i=7: push 7 → 弹出 6（1次操作）

总操作次数 = 1 + 0 + 0 + 4 + 0 + 2 + 1 = 8 = O(n)

### 6. 为什么单调队列是最优解？
- **暴力解法**：O(n × k) = O(10^10)，不可行
- **大顶堆**：无法处理窗口滑动
- **multiset**：O(n log k) = O(10^5 × 17) = 1.7 × 10^6，可行但不是最优
- **单调队列**：O(n) = O(10^5)，最优

## 四、 算法思想对比与关键步骤解析

### 1. 三种主流解法对比

#### 1.1 暴力解法
- **核心思想**：遍历所有窗口，对每个窗口遍历找最大值
- **实现难度**：简单
- **时间复杂度**：O(n × k)
- **空间复杂度**：O(1)
- **适用场景**：n 和 k 都很小的情况

#### 1.2 大顶堆（优先级队列）
- **核心思想**：使用堆维护窗口内的元素
- **实现难度**：中等
- **时间复杂度**：O(n log k)
- **空间复杂度**：O(k)
- **致命缺陷**：无法移除特定元素，只能移除堆顶

#### 1.3 单调队列
- **核心思想**：维护一个单调递减的队列，只保留有可能成为最大值的元素
- **实现难度**：较高（需要理解单调性维护）
- **时间复杂度**：O(n)
- **空间复杂度**：O(k)
- **适用场景**：滑动窗口最值问题

### 2. 单调队列的核心设计原理

#### 2.1 为什么队列要单调递减？
- **最大值在队首**：单调递减队列，最大的元素在队列的最前面（front）
- **便于维护**：当新元素加入时，可以从队尾移除所有比它小的元素
- **自动清理**：当元素移出窗口时，只需要检查是否是队首元素

#### 2.2 为什么要在队尾移除元素？
- **保持单调性**：确保队列从大到小排序
- **去除无用元素**：比新元素小的元素永远不可能成为最大值
- **局部性原理**：最近加入的大元素更有可能成为最大值

#### 2.3 为什么存储索引而不是值？
- **判断窗口边界**：可以通过索引判断元素是否在窗口内
- **处理重复元素**：相同值的元素可以通过索引区分
- **代码简洁**：避免复杂的值比较

### 3. 关键步骤详细解析

#### 3.1 步骤一：初始化窗口
```cpp
for (int i = 0; i < k; i++) {
    // 移除比当前元素小的队列尾部元素，保持队列单调递减
    while (!dq.empty() && nums[i] > nums[dq.back()]) {
        dq.pop_back();
    }
    dq.push_back(i);
}
```
- **目的**：建立第一个窗口（[0, k-1]）的最大值
- **关键**：确保队列单调递减后，dq.front() 就是最大值

#### 3.2 步骤二：窗口滑动
```cpp
for (int i = k; i < nums.size(); i++) {
    // 移除离开窗口的元素
    if (dq.front() <= i - k) {
        dq.pop_front();
    }
    // 加入新元素
    while (!dq.empty() && nums[i] > nums[dq.back()]) {
        dq.pop_back();
    }
    dq.push_back(i);
    // 记录最大值
    result.push_back(nums[dq.front()]);
}
```
- **目的**：移动窗口并维护最大值
- **关键**：
  1. 先检查队首是否过期
  2. 加入新元素前先维护单调性
  3. 记录当前窗口最大值

#### 3.3 步骤三：记录结果
```cpp
if (i >= k - 1) {
    result.push_back(nums[dq.front()]);
}
```
- **目的**：当窗口大小达到 k 时，记录最大值
- **关键**：dq.front() 始终是当前窗口的最大值

## 五、 关键知识点总结

### 1. 单调队列
- **定义**：队列中的元素保持单调递增或单调递减的顺序
- **特性**：可以在 O(1) 时间内获取最大值或最小值
- **实现**：通常使用 deque 作为底层容器
- **应用场景**：滑动窗口最值问题

### 2. deque（双端队列）
- **特性**：可以在两端进行高效的插入和删除操作，时间复杂度都是 O(1)
- **头操作**：push_front()、pop_front()、front()
- **尾操作**：push_back()、pop_back()、back()
- **适用场景**：实现单调队列、双端队列问题

### 3. 滑动窗口技巧
- **窗口初始化**：先处理前 k 个元素
- **窗口滑动**：每次移除最左边元素，加入最右边元素
- **边界处理**：注意窗口的起始和结束位置

### 4. 时间复杂度分析技巧
- **嵌套循环不一定是 O(n^2)**：需要考虑每个元素被处理的总次数
- **amortized analysis（均摊分析）**：用于分析一些看起来复杂但实际上高效的算法
- **单调队列的均摊分析**：每个元素最多被 push 和 pop 各一次，所以总时间复杂度是 O(n)

## 六、 易错点提示

### 1. 操作顺序错误
- **错误**：先 pop 再 push，或者先 push 再 pop
- **正确**：先处理窗口的移除操作，再处理窗口的添加操作
- **注意**：在方法二中，要先处理新元素的加入，再移除过期元素

### 2. 队列中存储值而不是索引
- **问题**：当有重复元素时，可能会错误地移除元素
- **解决**：存储索引而非值，这样可以更准确地判断元素是否在窗口内

### 3. 忘记处理窗口边界
- **错误**：没有检查队列头部元素是否已经不在窗口内
- **解决**：每次窗口移动后，检查队列头部元素是否在窗口范围内

### 4. 维护单调顺序的方式错误
- **错误**：在头部移除元素，而在尾部移除
- **正确**：在尾部移除比新元素小的元素，保持单调递减的顺序

### 5. 边界条件处理
- **k = 1**：每个元素本身就是最大值
- **n = 1**：直接返回该元素
- **元素全部相同**：所有窗口最大值都是该元素

## 九、 算法执行流程

### 1. 方法二详细执行流程

以 nums = [1,3,-1,-3,5,3,6,7], k = 3 为例：

#### 初始状态
- dq = [], result = []

#### i = 0（元素 1）
- dq 为空，push_back(0) → dq = [0]
- i < 2，不记录结果

#### i = 1（元素 3）
- nums[1] = 3 > nums[0] = 1，pop_back() → dq = []
- push_back(1) → dq = [1]
- i < 2，不记录结果

#### i = 2（元素 -1）
- nums[2] = -1 < nums[1] = 3，直接 push_back(2) → dq = [1, 2]
- i >= 2，记录 nums[1] = 3 → result = [3]

#### i = 3（元素 -3）
- nums[3] = -3 < nums[2] = -1，直接 push_back(3) → dq = [1, 2, 3]
- 检查 dq.front() = 1，1 > 0（3-3），不弹出
- i >= 2，记录 nums[1] = 3 → result = [3, 3]

#### i = 4（元素 5）
- nums[4] = 5 > nums[3] = -3，pop_back() → dq = [1, 2]
- nums[4] = 5 > nums[2] = -1，pop_back() → dq = [1]
- nums[4] = 5 > nums[1] = 3，pop_back() → dq = []
- push_back(4) → dq = [4]
- 检查 dq.front() = 4，4 > 1（4-3），不弹出
- i >= 2，记录 nums[4] = 5 → result = [3, 3, 5]

#### 继续这个过程，最终得到 result = [3, 3, 5, 5, 6, 7]

## 十、 面试相关八股问题

### 1. 基础问题

#### 1.1 单调队列相关
- **问题**：什么是单调队列？
  - **答案**：单调队列是一种特殊的队列，队列中的元素保持单调递增或单调递减的顺序。

- **问题**：单调队列有什么特性？
  - **答案**：可以在 O(1) 时间内获取最大值或最小值。

- **问题**：如何实现单调队列？
  - **答案**：通常使用 deque（双端队列）作为底层容器，在加入新元素时维护单调性。

#### 1.2 本题解法相关
- **问题**：为什么不能用大顶堆（优先级队列）？
  - **答案**：大顶堆只能移除堆顶元素，无法移除其他元素，当窗口移动时，无法准确维护窗口内的元素。

- **问题**：为什么要存储索引而不是值？
  - **答案**：存储索引可以更方便地判断元素是否在窗口内，特别是在有重复元素时，可以避免错误移除。

- **问题**：时间复杂度是多少？为什么？
  - **答案**：时间复杂度是 O(n)，因为每个元素最多被 push 和 pop 各一次，总操作次数是线性的。

### 2. 进阶问题

#### 2.1 单调队列设计
- **问题**：设计单调队列时，如何处理 push 操作？
  - **答案**：当 push 新元素时，从队列尾部开始，移除所有比新元素小的元素，然后将新元素加入队列尾部。

- **问题**：设计单调队列时，如何处理 pop 操作？
  - **答案**：当需要移除元素时，只在该元素等于队列头部元素时才进行 pop_front 操作。

- **问题**：如何同时维护滑动窗口的最大值和最小值？
  - **答案**：使用两个单调队列，一个维护最大值，一个维护最小值。

#### 2.2 时间复杂度分析
- **问题**：虽然有嵌套的 while 循环，为什么时间复杂度还是 O(n)？
  - **答案**：使用均摊分析，每个元素最多被 push 和 pop 各一次，所以总操作次数是线性的。

- **问题**：如何证明单调队列的均摊时间复杂度是 O(n)？
  - **答案**：使用势能分析，定义势能函数为队列的大小，每个 push 操作势能加 1，每个 pop 操作势能减 1，总时间复杂度是 O(n)。

### 3. 实际应用问题

#### 3.1 实际场景
- **问题**：滑动窗口最大值在实际中有什么应用？
  - **答案**：可以用于网络流量监控、股票分析、任务调度、传感器数据处理等场景。

- **问题**：如果数据是实时流，如何设计在线算法？
  - **答案**：同样使用单调队列，但需要考虑数据是源源不断到来的情况。

#### 3.2 系统设计
- **问题**：如果需要处理非常大量的数据（如 10^10），如何设计分布式算法？
  - **答案**：可以将数据划分到不同节点，每个节点处理自己的滑动窗口，然后进行合并。

- **问题**：如何处理窗口大小动态变化的情况？
  - **答案**：这会更复杂，可能需要重新设计算法，或者使用更灵活的数据结构。

### 4. 代码优化问题

#### 4.1 代码优化
- **问题**：如何优化代码以提高性能？
  - **答案**：使用更高效的数据结构、减少不必要的判断、利用现代编译器优化等。

- **问题**：如何处理内存限制？
  - **答案**：使用滚动窗口的思想，只保存必要信息，不保存整个历史数据。

#### 4.2 边界情况

  - **答案**：直接返回原数组，每个元素都是最大值。

## 十二、 更多解题思路（可选）

### 1. 分块处理（块状数组）

#### 1.1 思路
将数组分成大小为 sqrt(n) 的块，每个块预处理其最大值。查询时，查询整个块的最大值和边界上的元素。

#### 1.2 时间复杂度
- 预处理：O(n)
- 查询：O(sqrt(n)) per window，共 O(n sqrt(n))
- 总时间复杂度：O(n sqrt(n))，不如单调队列高效

### 2. 线段树

#### 2.1 思路
使用线段树维护区间最大值，每次查询窗口 [i-k+1, i] 的最大值。

#### 2.2 时间复杂度
- 建树：O(n)
- 查询：O(log n) per window，共 O(n log n)
- 总时间复杂度：O(n log n)，可以通过但不如单调队列高效

### 3. 两种方法的优缺点
- **分块处理**：实现相对简单，空间复杂度低，但时间复杂度较高
- **线段树**：功能强大，可以处理更多类型的区间查询，但实现较复杂，常数因子较大
- **结论**：对于这道题，单调队列是最优选择
