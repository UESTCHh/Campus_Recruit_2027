# 算法实战复盘：反转链表 (LeetCode 206)

> **打卡日期**：2026-03-30 (Day 4)
> **核心主题**：多指针微操、断链保护、原地反转 (In-place)。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。
* **大厂面试变态限制：** 绝对不允许你额外开辟一个新链表或者数组来存数据（拒绝空间复杂度 $O(n)$ 的投机取巧），**必须在原链表上原地（In-place）修改指针的指向！**

---

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n)$
**推导过程：** 我们使用一个 `cur` 指针从原链表的头节点走到尾节点（`nullptr`）。对于链表中的 $N$ 个节点，每个节点刚好被访问一次，且反转指针的操作（4 行赋值代码）是常数时间 $O(1)$ 的底层操作。总耗时随节点数量线性增长，因此时间复杂度严格为 $O(n)$。

### 2. 空间复杂度：$O(1)$
**推导过程：** 我们仅仅向栈区申请了 `pre`、`cur`、`temp` 三个临时指针变量。没有任何随数据规模 $N$ 动态增长的数组或哈希表。符合大厂“原地反转”的严苛要求，空间复杂度极致的 $O(1)$。

---

## 🎯 三、 核心思想：多指针贴身肉搏与断链保护



想象两个人手拉手，你想让第二个人回过头来牵第一个人的手。
**核心痛点：** 一旦第二个人（`cur`）松开牵着第三个人（原本的 `next`）的手，转而去牵第一个人（`pre`）的手，**第三个人就彻底在茫茫内存中走丢了！链表当场断裂！**

**破局思路：引入 `temp` 探路指针！**
在 `cur` 转身（改变指向）之前，先派 `temp` 去抓住原本的下一个节点。确保大部队不走丢后，`cur` 再放心地完成华丽的转身。

---

## 💻 四、 核心代码 (双指针法)

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode { ... };
 */
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* pre = nullptr; // 前驱节点，反转后它会成为新的头。一开始真实头节点的前驱是空。
        ListNode* cur = head;    // 当前正在处理的节点
        ListNode* temp;          // 探路兵：负责在断链前，死死抓住下一个节点

        // 当 cur 为 nullptr 时，说明链表已经全部反转完毕
        while (cur != nullptr) {
            // 步骤 1：探路兵先走！保存 cur 的下一个节点，防止转身后彻底失联
            temp = cur->next; 

            // 步骤 2：核心反转动作！cur 放开后面的手，转头指向前面的 pre
            cur->next = pre;

            // 步骤 3：整体舰队向前平移，准备处理下一个节点
            pre = cur;  // pre 移到 cur 的位置
            cur = temp; // cur 移到刚刚探路兵保存的位置
        }
        
        // 循环结束时，cur 指向了黑洞 (nullptr)，而 pre 刚好停在原链表的最后一个节点！
        // 此时的 pre，就是反转后的新链表的真实头节点！
        return pre; 
    }
};
```

## 💡 五、 面试直击灵魂的防暴击拷问
* 灵魂拷问一：循环的条件为什么是 while(cur != nullptr) 而不是 while(cur->next != nullptr)？
绝杀回答： 如果使用 cur->next != nullptr，当 cur 走到原链表的最后一个节点时，循环就会终止！这就导致最后一个节点的 next 指针根本没有被反转（它依然指向 nullptr 而没有指向倒数第二个节点）。使用 cur != nullptr 才能保证每一个节点、包括尾节点都被拉进循环里彻底执行一次转身。

* 灵魂拷问二：最后为什么要 return pre;，而不是 return cur;？
绝杀回答： 因为 while 循环终止的唯一条件就是 cur 变成了 nullptr（掉下了悬崖）。如果你返回 cur，你返回的就是一个空指针！而此时的 pre 恰好是紧紧跟在 cur 身后的，它停在了原链表的最后一个节点上。在全部反转完成后，这个节点正式加冕为新链表的头节点。所以必须返回 pre。
