# 算法实战复盘：反转字符串 (LeetCode 344)

> **打卡日期**：2026-04-06
> **核心主题**：双指针法、原地操作、字符串处理。

---

## 📝 一、 题目描述与核心要求

### 1. 中文描述
编写一个函数，其作用是将输入的字符串反转过来。输入字符串以字符数组 `s` 的形式给出。

不要给另外的数组分配额外的空间，你必须**原地**修改输入数组、使用 O(1) 的额外空间解决这一问题。

### 2. 英文描述
Write a function that reverses a string. The input string is given as an array of characters `s`.

You must do this by modifying the input array **in-place** with O(1) extra memory.

### 3. 输入示例
- **示例 1**：
  - 输入：`s = ["h","e","l","l","o"]`
  - 输出：`["o","l","l","e","h"]`

- **示例 2**：
  - 输入：`s = ["H","a","n","n","a","h"]`
  - 输出：`["h","a","n","n","a","H"]`

### 4. 核心痛点
- 要求原地修改，不允许使用额外空间
- 需要处理不同长度的字符串
- 确保所有字符都被正确反转

### 5. 题目提示
- 1 <= s.length <= 10^5
- s[i] 都是 ASCII 码表中的可打印字符

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：O(n)
**推导过程**：无论使用哪种方法，都需要遍历字符串中的每个字符至少一次，因此时间复杂度为 O(n)，其中 n 是字符串的长度。

### 2. 空间复杂度：O(1)
**推导过程**：由于要求原地修改，不使用额外空间，因此空间复杂度为 O(1)。

## 🤔 三、 解题思路分析

### 1. 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：创建一个新的字符数组，从后往前复制原数组的元素，然后再复制回原数组
2. **双指针法**：使用两个指针，分别从字符串的两端开始，交换对应位置的字符
3. **递归法**：使用递归的方式，交换首尾字符，然后递归处理中间部分

### 2. 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n) | O(n) | 思路简单 | 违反了原地修改的要求，使用了额外空间 |
| 双指针法 | O(n) | O(1) | 原地修改，空间复杂度最优 | 无明显缺点 |
| 递归法 | O(n) | O(n) | 代码简洁 | 递归深度可能导致栈溢出，空间复杂度高 |

### 3. 数据大小对方法选择的影响

根据题目提示，字符串长度最大为 10^5：

- **暴力法**：需要额外的 O(n) 空间，对于 n=10^5，这会占用约 100KB 的额外内存，虽然不算太大，但违反了题目要求
- **双指针法**：只使用常数级额外空间，对于任何大小的字符串都适用
- **递归法**：递归深度为 n/2，对于 n=10^5，递归深度会达到 50000，这会导致栈溢出

**实际测试验证**：
- 暴力法：虽然能正确反转字符串，但会因使用额外空间而被判定为不符合要求
- 双指针法：执行时间短，空间使用少，符合题目要求
- 递归法：对于长字符串会导致栈溢出

### 4. 方法选择建议

**推荐使用双指针法**，原因如下：
1. 符合题目要求，原地修改，空间复杂度 O(1)
2. 时间复杂度 O(n)，效率高
3. 代码简洁，易于理解
4. 适用于任何长度的字符串

## 🎯 四、 核心思想：双指针法

双指针法是解决字符串反转问题的最优方法。它通过使用两个指针，分别从字符串的两端开始，交换对应位置的字符，直到指针相遇。

### 1. 核心原理

- **左指针**：从字符串的开始位置（索引 0）开始
- **右指针**：从字符串的结束位置（索引 n-1）开始
- **交换过程**：交换左指针和右指针指向的字符，然后左指针向右移动，右指针向左移动，直到左指针超过右指针

### 2. 算法步骤详解

1. **初始化指针**：左指针 `left` 初始化为 0，右指针 `right` 初始化为 `s.size() - 1`
2. **交换字符**：当 `left < right` 时，交换 `s[left]` 和 `s[right]`
3. **移动指针**：`left++`，`right--`
4. **重复步骤 2-3**：直到 `left >= right`

### 3. 示例分析

以示例 1 为例：`s = ["h","e","l","l","o"]`

**步骤 1**：初始化 `left = 0`, `right = 4`
- 交换 `s[0]` 和 `s[4]`，得到 `["o","e","l","l","h"]`
- `left = 1`, `right = 3`

**步骤 2**：交换 `s[1]` 和 `s[3]`，得到 `["o","l","l","e","h"]`
- `left = 2`, `right = 2`

**步骤 3**：`left >= right`，结束

**最终结果**：`["o","l","l","e","h"]`

### 4. 边界情况处理

1. **空字符串**：如果字符串为空，直接返回
2. **长度为 1 的字符串**：无需反转，直接返回
3. **长度为偶数的字符串**：左右指针最终会相遇
4. **长度为奇数的字符串**：中间字符无需交换，左右指针交叉后结束

### 5. 与其他方法的对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 双指针法 | O(n) | O(1) | 原地修改，空间复杂度最优 | 无明显缺点 |
| 暴力法 | O(n) | O(n) | 思路简单 | 违反了原地修改的要求 |
| 递归法 | O(n) | O(n) | 代码简洁 | 可能导致栈溢出 |

### 6. 关键技术点

1. **原地修改**：不使用额外空间，直接修改原数组
2. **双指针**：从两端向中间移动，减少遍历次数
3. **交换操作**：使用临时变量或位运算进行交换
4. **循环条件**：`left < right`，确保只处理一半的字符

## 💻 五、 核心代码

### 1. 双指针法实现

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    void reverseString(vector<char>& s) {
        int n = s.size();
        // 初始化左右指针
        int left = 0;
        int right = n - 1;
        
        // 当左指针小于右指针时，交换字符
        while (left < right) {
            // 交换 s[left] 和 s[right]
            swap(s[left], s[right]);
            // 左指针右移，右指针左移
            left++;
            right--;
        }
    }
};
```

**代码解释**：
1. **初始化指针**：`left` 从 0 开始，`right` 从数组末尾开始
2. **交换字符**：使用 `swap` 函数交换 `s[left]` 和 `s[right]`
3. **移动指针**：`left` 向右移动，`right` 向左移动
4. **循环条件**：当 `left < right` 时继续循环

### 2. 暴力法实现（不推荐，仅作对比）

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    void reverseString(vector<char>& s) {
        int n = s.size();
        // 创建临时数组
        vector<char> temp(n);
        
        // 从后往前复制原数组的元素
        for (int i = 0; i < n; i++) {
            temp[i] = s[n - 1 - i];
        }
        
        // 复制回原数组
        for (int i = 0; i < n; i++) {
            s[i] = temp[i];
        }
    }
};
```

**暴力法性能分析**：
- 时间复杂度：O(n)，需要两次遍历数组
- 空间复杂度：O(n)，需要额外的临时数组
- 适用场景：不推荐，违反了原地修改的要求

### 3. 递归法实现（不推荐，仅作对比）

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    void reverseString(vector<char>& s) {
        reverseHelper(s, 0, s.size() - 1);
    }
    
private:
    void reverseHelper(vector<char>& s, int left, int right) {
        // 递归终止条件
        if (left >= right) {
            return;
        }
        
        // 交换字符
        swap(s[left], s[right]);
        
        // 递归处理中间部分
        reverseHelper(s, left + 1, right - 1);
    }
};
```

**递归法性能分析**：
- 时间复杂度：O(n)，每个字符只处理一次
- 空间复杂度：O(n)，递归调用栈的深度为 n/2
- 适用场景：不推荐，对于长字符串可能导致栈溢出

## ❌ 七、 常见错误与注意事项

### 1. 常见错误

1. **使用额外空间**：创建新的数组或字符串来存储反转后的结果，违反了原地修改的要求
2. **循环条件错误**：使用 `left <= right` 作为循环条件，导致中间字符被交换两次
3. **指针移动错误**：只移动一个指针，导致无限循环
4. **递归深度过大**：使用递归方法处理长字符串，导致栈溢出
5. **边界条件处理错误**：没有处理空字符串或长度为 1 的字符串

### 2. 注意事项

1. **原地修改**：确保不使用额外的数组或字符串
2. **循环条件**：使用 `left < right` 作为循环条件，避免中间字符被重复交换
3. **指针移动**：每次交换后，同时移动左右指针
4. **边界情况**：处理空字符串和长度为 1 的字符串
5. **性能优化**：对于长字符串，双指针法是最优选择

## 🚀 八、 相关拓展问题及解题思路

### 1. 相关拓展问题

| 问题 | 描述 | 解题思路 |
|------|------|----------|
| 反转字符串 II | 反转字符串中每 2k 个字符中的前 k 个字符 | 双指针法，每次处理 2k 个字符 |
| 反转单词顺序 | 反转字符串中的单词顺序 | 先反转整个字符串，再反转每个单词 |

### 2. 解题思路

#### 2.1 反转字符串 II

**问题描述**：给定一个字符串 s 和一个整数 k，从字符串开头算起，每 2k 个字符反转前 k 个字符。如果剩余字符少于 k 个，则反转剩余字符。如果剩余字符小于 2k 但大于或等于 k 个，则反转前 k 个字符，其余字符保持原样。

**解题思路**：
- 使用双指针法，每次处理 2k 个字符
- 对于每个 2k 区间，反转前 k 个字符
- 处理剩余字符的情况

#### 2.2 反转单词顺序

**问题描述**：输入一个英文句子，翻转句子中单词的顺序，但单词内字符的顺序不变。

**解题思路**：
- 先反转整个字符串
- 再反转每个单词
- 处理空格和边界情况

#### 2.3 反转元音字母

**问题描述**：编写一个函数，以字符串作为输入，反转该字符串中的元音字母。

**解题思路**：
- 使用双指针法，左指针从左向右，右指针从右向左
- 当左右指针都指向元音字母时，交换它们
- 继续移动指针，直到左指针超过右指针

## 💡 九、 面试直击灵魂的防暴击拷问

* **灵魂拷问一：为什么双指针法是解决字符串反转问题的最优方法？**
    * **绝杀回答：**"双指针法是最优方法，因为它满足题目要求的原地修改，空间复杂度为 O(1)，同时时间复杂度为 O(n)，只需要遍历字符串一次。相比暴力法，它不需要额外空间；相比递归法，它不会导致栈溢出，适用于任何长度的字符串。"

* **灵魂拷问二：如何处理长度为奇数的字符串？**
    * **绝杀回答：**"对于长度为奇数的字符串，中间的字符不需要交换。双指针法的循环条件是 `left < right`，当左指针和右指针相遇时，循环结束，中间的字符自然保持不变。"

* **灵魂拷问三：交换字符的方法有哪些？**
    * **绝杀回答：**"交换字符的方法有多种：
      1. 使用临时变量：`char temp = s[left]; s[left] = s[right]; s[right] = temp;
      2. 使用位运算：`s[left] ^= s[right]; s[right] ^= s[left]; s[left] ^= s[right];`
      3. 使用 C++ 的 `swap` 函数：`swap(s[left], s[right]);`
      其中，使用 `swap` 函数代码最简洁，可读性最好。"
      > **⚠️ 架构师避坑指南：永远不要在面试中用位运算装 X 交换字符！**
        > 很多面经会教你用 `a ^= b; b ^= a; a ^= b;` 来原地交换两个变量，以节省一个临时变量的空间。但在现代 C++ 编译器（如 GCC/Clang -O3 优化）面前，这种写法不仅毫无性能优势，反而有两个致命缺陷：
        > 1. **破坏了 CPU 流水线：** 位运算存在严格的数据依赖关系（必须等上一步算完才能算下一步），彻底阻断了 CPU 的指令级并行（ILP）优化。
        > 2. **自己和自己交换会清零：** 如果 `left == right`，执行这段位运算会导致该位置的字符直接变成 `\0`（ASCII 码 0）！
        > **正解：** 老老实实使用 `std::swap(s[left], s[right]);`。现代编译器的寄存器分配机制会让它比位运算快得多，而且语义绝对清晰。

* **灵魂拷问四：如果允许使用额外空间，还有哪些方法可以解决这个问题？**
    * **绝杀回答：**"如果允许使用额外空间，可以使用以下方法：
      1. 创建一个新的字符串，从后往前复制原字符串的字符
      2. 使用栈，将字符依次入栈，然后出栈得到反转后的字符串
      3. 使用递归，通过递归调用反转字符串的子串
      但这些方法都需要额外的空间，不符合题目要求。"

* **灵魂拷问五：如何处理空字符串或长度为 1 的字符串？**
    * **绝杀回答：**"对于空字符串，不需要任何操作，直接返回。对于长度为 1 的字符串，反转后还是它本身，也不需要任何操作。在双指针法中，当字符串长度为 0 或 1 时，`left` 会大于或等于 `right`，循环不会执行，自然处理了这些边界情况。"
