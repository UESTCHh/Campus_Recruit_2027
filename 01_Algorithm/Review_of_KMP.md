# 算法实战复盘：深度解剖 KMP 算法底层逻辑与状态机思想

核心主题：单模式字符串匹配、前缀表（next 数组）、状态机回退机制、流式数据处理、算法的 Trade-off 哲学

核心目标：彻底消灭暴力匹配中极其损耗性能的“主串指针回溯”，理解 KMP 算法的设计精髓与工程落地边界，实现从“会用”到“吃透底层”的突破。

## 一、核心使用场景：KMP 到底用来解决什么问题？

### 1. 核心定义

KMP 算法专门解决「单模式字符串匹配问题（Single-Pattern Matching）」，这是字符串匹配领域最基础、最常用的场景之一，区别于“多模式匹配”（如 AC 自动机解决的场景）。

### 2. 数学描述

给定一个长度为  $n$  的**主串（Text）**（即待搜索的长字符串），和一个长度为  $m$  的**模式串（Pattern）**（即要查找的短字符串），需求是：

- 精准找到模式串在主串中**首次出现的起始位置**；

- （扩展需求）找到模式串在主串中**所有出现的位置**；

- （边界需求）若模式串不在主串中，返回约定的标识（如 -1）。

### 3. 工业级落地场景（通俗解析）

KMP 并非单纯的算法题考点，在真实工程中应用极其广泛，核心是解决“长数据中快速找短特征”的问题，以下场景均为高频落地场景：

- **日常软件操作**：文本编辑器（如 VS Code、Word）、浏览器的「Ctrl + F」精准查找功能，本质就是单模式匹配——主串是当前编辑的文档/网页内容，模式串是你输入的搜索关键词，KMP 能提升查找效率，尤其在长文档中效果明显。

- **网络安全防护**：网络中传输的 TCP 流式数据（可理解为“无限长的主串”）中，需要快速拦截包含病毒特征码、恶意攻击载荷（可理解为“固定的模式串”）的数据包。此时 KMP 的“无主串回溯”特性至关重要，能避免反复读取已处理的流数据，提升拦截效率。

- **生物信息学**：DNA 序列是由 A、T、C、G 四种碱基组成的极长序列（长度可达数十亿），生物学家需要在其中快速找到特定的目标基因片段（模式串），KMP 能在不回溯主串的前提下，线性遍历完成查找，大幅降低算力消耗。

- **补充场景**：日志分析（在海量日志中查找特定错误码）、代码语法检查（查找特定语法片段）、数据库模糊查询（简化版底层实现）等。

## 二、暴力解法（Brute-Force）：它的致命软肋在哪里？

在没有接触过 KMP 等优化算法时，暴力解法是最本能、最直观的思路，核心是“逐个尝试、逐个比对”，但它的性能缺陷在极端场景下会被无限放大。

### 1. 暴力解法的核心逻辑

利用两层嵌套循环实现：

- 外层循环：遍历主串的每一个位置，作为“匹配的起始点”（主串指针  $i$ ，从 0 到  $n-m$ ）；

- 内层循环：从当前起始点开始，逐字符比对主串和模式串（模式串指针 $j$ ，从 0 到  $m-1$ ）；

- 匹配成功：若内层循环完整执行（ $j == m$ ），返回当前主串起始位置  $i$ ；

- 匹配失败：主串指针  $i$  回溯到 $i+1$ ，模式串指针  $j$  重置为 0，重新开始比对。

### 2. 暴力解法的工程优点（不可忽视）

暴力解法并非毫无价值，在特定场景下反而有优势，这也是它至今仍被使用的原因：

- **极致的直觉性**：逻辑极其简单，代码量极少（几行代码就能实现），不需要理解复杂的前缀表、状态机等概念，上手成本为 0。

- **空间复杂度最优**：不需要任何额外的数据结构辅助，空间复杂度为  $O(1)$ ，无需向操作系统申请额外的堆内存，在内存受限场景下有一定优势。

### 3. 暴力解法的致命痛点：主串指针回溯（Backtracking）

暴力解法最大的问题，在于它对“已经匹配过的信息毫无记忆”——一旦发生失配，就彻底抛弃之前的匹配结果，让主串指针回溯、模式串指针重置，相当于“推倒重来”，这也是其性能损耗的核心根源。

#### 💣 极端恶劣数据测试（直观感受痛点）

假设：主串 = "aaaaaaaab"（8 个 a + 1 个 b，长度  $n=9$ ），模式串 = "aaab"（3 个 a + 1 个 b，长度  $m=4$ ）。

匹配过程复盘：

1. 第一次匹配：主串 $i=0$  开始，比对到  $j=3$ （主串 s[3] = 'a'，模式串 p[3] = 'b'），失配；

2. 暴力解法处理：主串指针  $i$  回溯到 1，模式串 $j$  重置为 0，重新比对；

3. 第二次匹配：主串  $i=1$  开始，比对到  $j=3$ （s[4] = 'a' vs p[3] = 'b'），再次失配；

4. 重复回溯：主串  $i$  继续回溯到 2、3、4、5，每次都要比对到  $j=3$  才失配，直到  $i=5$  时，才能匹配成功（s[5-8] = "aaab"）。

整个过程中，主串指针反复前进、倒退，大量算力被浪费在“重复比对已经验证过的 a”上。

#### 痛点延伸（工程视角）

- **时间复杂度退化**：正常场景下，暴力解法的平均时间复杂度是  $O(n+m)$ ，但在包含大量“重复前缀”的恶劣数据中，时间复杂度会急剧恶化到  $O(n \times m)$ ，当  $n$  和  $m$  均达到 10^4 级别时，程序会出现明显卡顿。

- **算力与 I/O 浪费**：若主串是 1GB 的大文件、或持续输入的网络流，主串指针回溯会导致 CPU 反复处理已读取的数据，不仅浪费算力，还会增加底层 I/O 的负担（反复读取同一部分数据），严重影响系统吞吐量。

## 三、KMP 算法架构：原理与核心突破

KMP 算法（由 Knuth、Morris、Pratt 三位科学家联合提出）的核心诞生初衷，就是**彻底消灭主串指针的回溯**，它的架构设计围绕一个核心诉求展开：**主串指针只能勇往直前，绝不回头！**

KMP 的本质的是“利用已匹配信息，避免重复比对”，核心依赖两个关键机制：前缀表（next 数组）和状态机回退机制。

### 核心要点 1：前缀表（next 数组）—— “空间换时间”的备忘录

#### （1）前缀表的核心定义

KMP 算法在开始匹配主串之前，会先对**模式串自身**进行一次预处理，计算出一个长度与模式串相同的数组——next 数组（也叫前缀函数）。

next 数组的每一个元素 next[j]，表示：模式串中前 j+1 个字符组成的前缀子串（即 p[0..j]）的**“最长相等前后缀的长度”**。

#### （2）关键概念拆解（通俗理解）

- **前缀**：字符串中从开头到某个位置（不包含末尾字符）的子串，比如 "abcde" 的前缀有 "a"、"ab"、"abc"、"abcd"；

- **后缀**：字符串中从某个位置（不包含开头字符）到末尾的子串，比如 "abcde" 的后缀有 "e"、"de"、"cde"、"bcde"；

- **最长相等前后缀**：前缀和后缀中，长度最长且内容完全相同的子串，比如 "ababa" 的最长相等前后缀是 "aba"（长度 3），"aaaaa" 的最长相等前后缀是 "aaaa"（长度 4）。

#### （3）前缀表的工程意义（核心！）

next 数组本质上是一个“状态转移备忘录”，它的核心作用是：**当主串与模式串在某个位置失配时，告诉程序“模式串指针应该回退到哪个位置”，而不是傻傻地从头再来**。

换个角度理解：失配发生时，我们已经知道“失配点之前的字符是完全匹配的”，前缀表记录了这部分匹配字符中“可复用的前缀信息”，利用这个信息，就能让模式串“精准滑动”，跳过那些已经验证过的、不可能匹配的位置，从而避免主串回溯。

#### （4）next 数组示例（快速上手）

以模式串 "aaab" 为例，计算其 next 数组：

- j=0（子串 "a"）：无前缀、无后缀，最长相等前后缀长度为 0 → next[0] = 0；

- j=1（子串 "aa"）：前缀 "a"，后缀 "a"，最长相等前后缀长度为 1 → next[1] = 1；

- j=2（子串 "aaa"）：前缀 "aa"，后缀 "aa"，最长相等前后缀长度为 2 → next[2] = 2；

- j=3（子串 "aaab"）：前缀有 "a"、"aa"、"aaa"，后缀有 "b"、"ab"、"aab"，无相等前后缀 → next[3] = 0。

因此，模式串 "aaab" 的 next 数组为 [0, 1, 2, 0]。

### 核心要点 2：状态机回退（查表滑动）—— KMP 的匹配核心流程

有了 next 数组，KMP 的匹配过程就变得简单且高效，核心是“双指针比对 + 查表回退”，全程主串指针不回溯。

匹配流程（假设主串指针为 i，模式串指针为 j，初始值均为 0）：

1. **匹配成功**：若主串 s[i] == 模式串 p[j]，则 i 和 j 同时加 1（双方携手前进）；

2. **匹配失败**：若 s[i] != p[j]，则：
        

    - 若 j == 0（模式串指针已在起点，无法回退），则 i 加 1（主串继续前进）；

    - 若 j != 0（模式串指针可回退），则 j = next[j - 1]（查表回退，不改变 i）；

3. **终止条件**：
        

    - 若 j == m（模式串完全匹配），返回 i - j（主串中匹配的起始位置）；

    - 若 i == n（主串遍历完毕，模式串未匹配），返回 -1。

#### （1）结合示例理解（延续之前的“aaaaaaaab”和“aaab”）

next 数组为 [0,1,2,0]，匹配过程：

1. i=0, j=0：s[0] = 'a' == p[0] = 'a' → i=1, j=1；

2. i=1, j=1：s[1] = 'a' == p[1] = 'a' → i=2, j=2；

3. i=2, j=2：s[2] = 'a' == p[2] = 'a' → i=3, j=3；

4. i=3, j=3：s[3] = 'a' != p[3] = 'b' → j = next[2] = 2（不回溯 i）；

5. i=3, j=2：s[3] = 'a' == p[2] = 'a' → i=4, j=3；

6. 重复上述逻辑，直到 i=8, j=3：s[8] = 'b' == p[3] = 'b' → j=4 == m=4，匹配成功，返回 i-j=5。

整个过程中，主串指针 i 从 0 到 8 线性前进，从未回溯，效率大幅提升。

#### （2）物理直觉：模式串的“精准滑动”

当失配发生时，模式串指针 j 回退到 next[j-1]，本质上等同于“将模式串向右滑动了 (j - next[j-1]) 步”。

比如上述失配场景（j=3，next[j-1]=2），模式串向右滑动 3-2=1 步，直接跳过了“已经验证过的、不可能匹配的位置”，无需重新比对，这就是 KMP 高效的核心原因。

## 四、架构视角的改进评估：为什么要用 KMP？

相比于暴力解法，KMP 算法的改进是“降维打击”级别的，尤其在工程场景中，其优势不仅体现在时间复杂度上，更体现在对特殊场景的适配性上。

### 1. 时间复杂度的跨越式飞跃

KMP 算法的时间复杂度由两部分组成：

- 预处理阶段（计算 next 数组）：遍历一次模式串，时间复杂度为  $O(m)$ ；

- 匹配阶段（遍历主串）：主串指针线性前进，不回溯，时间复杂度为  $O(n)$ 。

因此，KMP 算法的总时间复杂度为  $O(n + m)$ ，即使在最坏的极端恶劣数据（如大量重复前缀）中，也能稳定保持这个复杂度，彻底解决了暴力解法的时间复杂度退化问题。

对比：暴力解法最坏  $O(n \times m)$ ，KMP 最坏  $O(n + m)$ ，当 n=10^6、m=10^4 时，KMP 的效率是暴力解法的 100 倍以上。

### 2. 极度契合流式数据（Streaming Data）处理

这是 KMP 算法在工程中最核心的优势之一，也是它区别于其他字符串匹配算法的关键特性。

由于主串指针永远不后退，意味着主串的数据可以像“水流”一样，通过网络端口、文件句柄持续输入——我们不需要在内存中缓存整个主串，只需读取一个字符、处理一个字符，处理完毕后即可丢弃该字符，无需保留供指针回溯。

应用场景：处理超大文件（如 10GB 日志文件）、实时网络流（如 TCP 数据包）时，KMP 能大幅降低内存占用，减少 I/O 开销，提升系统的吞吐量和响应速度。

## 五、真实世界中的 Trade-off：KMP 算法的优缺点与工程落地边界

没有任何技术是绝对完美的，KMP 算法的设计遵循“空间换时间”的哲学，这也决定了它有自身的优势和局限性。在大厂的工程落地中，理解这些边界，才能做出最优的技术选择。

### ✅ KMP 的核心优势（不可替代的场景）

- **无敌的恶劣数据抗性**：在包含大量重复子串的复杂文本中（如二进制文件扫描、基因序列分析、大量重复字符的日志），KMP 的性能彻底碾压暴力法，不会出现时间复杂度退化的情况。

- **时间复杂度绝对稳定**：无论输入数据如何诡异（重复前缀、随机字符、极端长度），KMP 的执行时间都有严格的线性上限（ $O(n+m)$ ），不会出现程序卡死、响应延迟的问题，适合对稳定性要求高的场景（如网络安全拦截）。

- **流式数据适配性强**：如前所述，主串不回溯的特性，让 KMP 能完美适配超大文件、实时流数据的处理，这是暴力解法无法实现的。

### ❌ KMP 的局限性与痛点（工程落地需注意）

- **空间换时间的代价**：需要额外申请  $O(m)$  的堆内存来存储 next 数组。如果运行环境内存极其受限（如极小内存的嵌入式芯片、单片机、物联网设备），这部分额外内存可能成为沉重负担，此时暴力解法反而更合适。

- **极高的心智负担与维护成本**：next 数组的推导逻辑非常绕脑（本质是“模式串自身匹配自身”），尤其是边界条件的处理（如 next 数组起始值设为 0 还是 -1），很容易出现 Bug；后续团队维护代码时，需要开发者彻底理解 KMP 底层逻辑，维护成本较高。

- **日常短文本中优势被反噬（重要考点）**：在日常绝大多数“短文本、无规律字符”的匹配场景中（比如在 "hello world" 里找 "ll"、在 "abcdef" 里找 "def"），KMP 的预处理过程（计算 next 数组、申请内存、初始化）会带来额外的性能损耗（Overhead），反而不如暴力解法高效——因为这类场景中，往往匹配几次就能成功，不需要复杂的前缀表辅助。

### 💡 底层源码揭秘（大厂面试高频）

这也是为什么很多高级语言的标准库（如 C++ 的 std::string::find、Java 的 String.indexOf、Python 的 str.find），在底层往往不使用“纯正的 KMP 算法”。

它们的底层实现更倾向于：

- 针对短文本场景：使用经过 SIMD 指令优化的暴力法（利用 CPU 并行计算，提升比对效率）；

- 针对长文本/复杂场景：使用 Boyer-Moore (BM) 算法——这种算法在普通文本中跳跃性更强，平均时间复杂度为亚线性（ $O(n/m)$ ），比 KMP 更适合日常文本匹配；

- 极端场景（如流式数据、基因序列）：才会使用 KMP 或其变种算法。

## 六、复盘总结（核心要点提炼）

1. KMP 核心解决：单模式字符串匹配问题，核心突破是“消灭主串指针回溯”；

2. 核心依赖：前缀表（next 数组）—— 预处理模式串，记录最长相等前后缀，实现“空间换时间”；

3. 匹配逻辑：双指针比对，失配时模式串指针查表回退，主串指针不回溯，时间复杂度稳定  $O(n+m)$ ；

4. 工程落地：适合流式数据、长文本、重复前缀多的场景；不适合内存受限、短文本场景；

5. Trade-off 哲学：没有最优算法，只有最适合场景的算法——KMP 用空间代价，换来了时间稳定性和流式适配性。

## 1. KMP 算法简介

### 1.1 什么是 KMP 算法

KMP 算法是一种高效的字符串匹配算法，由 Knuth、Morris 和 Pratt 三位学者发明，因此得名 KMP。它的核心思想是利用已经匹配的信息，避免模式串的重复比较，从而提高字符串匹配的效率。

### 1.2 应用场景

KMP 算法主要应用于以下场景：
- 字符串匹配：在一个文本串中查找一个模式串的出现位置
- 模式识别：在生物序列分析、文本编辑等领域识别特定模式
- 编译器设计：词法分析中的模式匹配
- 网络安全：入侵检测系统中的特征匹配

### 1.3 KMP 算法的优势

与朴素字符串匹配算法相比，KMP 算法具有以下优势：
- **时间复杂度**：KMP 算法的时间复杂度为 O(m + n)，其中 m 是模式串长度，n 是文本串长度，而朴素算法的时间复杂度为 O(m * n)
- **避免重复比较**：当匹配失败时，KMP 算法利用已匹配的信息，避免从头开始匹配
- **空间效率**：虽然需要额外的空间来存储 next 数组，但空间复杂度仅为 O(m)，对于大多数应用场景来说是可以接受的

## 2. 核心原理

### 2.1 朴素字符串匹配的问题

朴素字符串匹配算法在每次匹配失败时，都会将模式串向右移动一位，然后从头开始匹配。这种方法在模式串和文本串有很多重复前缀时会做很多无用功。

例如，当文本串是 "AAAAAAB"，模式串是 "AAAAAB" 时，朴素算法会在每次匹配失败后重新开始，而 KMP 算法则会利用已匹配的信息，直接跳过不必要的比较。

### 2.2 KMP 的核心思想

KMP 算法的核心思想是：当出现字符串不匹配时，可以利用已经匹配的部分信息，避免模式串的重复比较。

具体来说，KMP 算法通过构建一个前缀表（next 数组），记录模式串中每个位置的最长相等前后缀长度，从而在匹配失败时，知道模式串应该回退到哪个位置继续匹配。

### 2.3 前缀和后缀的定义

- **前缀**：指不包含最后一个字符的所有以第一个字符开头的连续子串
- **后缀**：指不包含第一个字符的所有以最后一个字符结尾的连续子串

例如，对于字符串 "ABABC"：
- 前缀包括："A", "AB", "ABA", "ABAB"
- 后缀包括："B", "CB", "ABC", "BABC"

### 2.4 最长相等前后缀

最长相等前后缀是指字符串中相同的前缀和后缀的最大长度。

例如，对于字符串 "ABAB"：
- 前缀："A", "AB", "ABA"
- 后缀："B", "AB", "BAB"
- 最长相等前后缀是 "AB"，长度为 2

## 3. 前缀表与 next 数组

### 3.1 前缀表的作用

前缀表（也称为 next 数组）的作用是：当模式串与文本串匹配失败时，记录模式串应该回退到哪个位置继续匹配，而不是从头开始。

### 3.2 next 数组的定义

next 数组的每个元素 next[i] 表示：模式串中以 i 结尾的子串的最长相等前后缀的长度。

### 3.3 构建 next 数组的示例

以模式串 "ABAB" 为例，构建 next 数组：

| 索引 i | 子串 | 最长相等前后缀 | next[i] |
|--------|------|----------------|---------|
| 0      | "A"  | 无             | 0       |
| 1      | "AB" | 无             | 0       |
| 2      | "ABA" | "A"            | 1       |
| 3      | "ABAB" | "AB"           | 2       |

以模式串 "AABAAF" 为例，构建 next 数组：

| 索引 i | 子串 | 最长相等前后缀 | next[i] |
|--------|------|----------------|---------|
| 0      | "A"  | 无             | 0       |
| 1      | "AA" | "A"            | 1       |
| 2      | "AAB" | 无             | 0       |
| 3      | "AABA" | "A"            | 1       |
| 4      | "AABAA" | "AA"           | 2       |
| 5      | "AABAAF" | 无             | 0       |

### 3.4 前缀表的不同表示方式

前缀表有两种常见的表示方式：
1. **原始前缀表**：直接存储最长相等前后缀的长度
2. **前缀表统一减一**：将原始前缀表中的每个值减一，这样在回退时可以直接使用 next[i] 作为索引

## 4. 构建 next 数组的步骤

### 4.1 初始化

- 初始化两个指针：j 指向已匹配的前缀末尾，i 指向当前处理的位置
- 初始化 next 数组，next[0] = 0

### 4.2 处理不匹配的情况

当 s[i] != s[j] 时，需要回退 j 到 next[j-1] 的位置，直到 j == 0 或者 s[i] == s[j]

### 4.3 处理匹配的情况

当 s[i] == s[j] 时，j 增加 1，然后 next[i] = j

### 4.4 完整的 next 数组构建代码

```cpp
// 构建原始前缀表（不减一）
void getNext(int* next, const string& s) {
    int j = 0; // j 指向前缀的末尾
    next[0] = 0;
    for (int i = 1; i < s.size(); i++) { // i 指向后缀的末尾
        // 处理不匹配的情况
        while (j > 0 && s[i] != s[j]) {
            j = next[j - 1]; // 回退 j
        }
        // 处理匹配的情况
        if (s[i] == s[j]) {
            j++;
        }
        next[i] = j;
    }
}

// 构建前缀表（统一减一）
void getNext(int* next, const string& s) {
    int j = -1;
    next[0] = j;
    for (int i = 1; i < s.size(); i++) {
        while (j >= 0 && s[i] != s[j + 1]) {
            j = next[j];
        }
        if (s[i] == s[j + 1]) {
            j++;
        }
        next[i] = j;
    }
}
```

### 4.5 构建 next 数组的详细示例

以模式串 "AABAAF" 为例，构建 next 数组（原始前缀表）：

1. 初始化：j = 0, next[0] = 0
2. i = 1：s[1] = 'A', s[j] = 'A'，匹配，j = 1，next[1] = 1
3. i = 2：s[2] = 'B', s[j] = 'A'，不匹配，j = next[j-1] = next[0] = 0，s[2] != s[0]，j 保持 0，next[2] = 0
4. i = 3：s[3] = 'A', s[j] = 'A'，匹配，j = 1，next[3] = 1
5. i = 4：s[4] = 'A', s[j] = 'A'，匹配，j = 2，next[4] = 2
6. i = 5：s[5] = 'F', s[j] = 'B'，不匹配，j = next[j-1] = next[1] = 1，s[5] != s[1]，j = next[j-1] = next[0] = 0，s[5] != s[0]，j 保持 0，next[5] = 0

最终 next 数组为：[0, 1, 0, 1, 2, 0]

## 5. KMP 算法的匹配过程

### 5.1 匹配步骤

1. 初始化两个指针：i 指向文本串的当前位置，j 指向模式串的当前位置
2. 当 i < 文本串长度且 j < 模式串长度时：
   - 如果 s[i] == p[j]，则 i 和 j 都增加 1
   - 如果 s[i] != p[j]：
     - 如果 j > 0，则 j = next[j-1]（使用原始前缀表）
     - 如果 j == 0，则 i 增加 1
3. 当 j == 模式串长度时，匹配成功，返回 i - j
4. 如果匹配失败，返回 -1

以文本串 "AABAABAFA" 和模式串 "AABAAF" 为例：

2. s[0] = 'A', p[0] = 'A'，匹配，i = 1, j = 1
3. s[1] = 'A', p[1] = 'A'，匹配，i = 2, j = 2
4. s[2] = 'B', p[2] = 'B'，匹配，i = 3, j = 3
5. s[3] = 'A', p[3] = 'A'，匹配，i = 4, j = 4
6. s[4] = 'A', p[4] = 'A'，匹配，i = 5, j = 5
7. s[5] = 'B', p[5] = 'F'，不匹配
8. j = next[j-1] = next[4] = 2
9. s[5] = 'B', p[2] = 'B'，匹配，i = 6, j = 3
10. s[6] = 'A', p[3] = 'A'，匹配，i = 7, j = 4
11. s[7] = 'F', p[4] = 'A'，不匹配
12. j = next[j-1] = next[3] = 1
13. s[7] = 'F', p[1] = 'A'，不匹配
14. j = next[j-1] = next[0] = 0
15. s[7] = 'F', p[0] = 'A'，不匹配，i = 8
16. i == 文本串长度，匹配失败，返回 -1

## 6. 完整代码实现

### 6.1 使用原始前缀表的实现

```cpp
#include <iostream>
#include <string>
#include <vector>

using namespace std;

// 构建原始前缀表（不减一）
void getNext(vector<int>& next, const string& s) {
    int j = 0; // j 指向前缀的末尾
    next[0] = 0;
    for (int i = 1; i < s.size(); i++) { // i 指向后缀的末尾
        // 处理不匹配的情况
        while (j > 0 && s[i] != s[j]) {
            j = next[j - 1]; // 回退 j
        }
        // 处理匹配的情况
        if (s[i] == s[j]) {
            j++;
        }
        next[i] = j;
    }
}

// KMP 匹配算法
int strStr(string haystack, string needle) {
    if (needle.empty()) {
        return 0;
    }
    
    vector<int> next(needle.size());
    getNext(next, needle);
    
    int j = 0; // j 指向模式串的当前位置
    for (int i = 0; i < haystack.size(); i++) { // i 指向文本串的当前位置
        // 处理不匹配的情况
        while (j > 0 && haystack[i] != needle[j]) {
            j = next[j - 1]; // 回退 j
        }
        // 处理匹配的情况
        if (haystack[i] == needle[j]) {
            j++;
        }
        // 匹配成功
        if (j == needle.size()) {
            return i - j + 1;
        }
    }
    
    return -1;
}

int main() {
    string haystack = "AABAABAFA";
    string needle = "AABAAF";
    int result = strStr(haystack, needle);
    cout << "Result: " << result << endl; // 输出: -1
    
    haystack = "hello";
    needle = "ll";
    result = strStr(haystack, needle);
    cout << "Result: " << result << endl; // 输出: 2
    
    return 0;
}
```

## 7. 常见问题与解决方案

### 7.1 如何处理空模式串

当模式串为空时，根据题目要求，应该返回 0。这与 C 语言的 `strstr()` 和 Java 的 `indexOf()` 定义相符。

### 7.2 如何处理模式串长度大于文本串长度的情况

当模式串长度大于文本串长度时，不可能匹配成功，直接返回 -1。

### 7.3 如何理解 next 数组的回退机制

next 数组的回退机制是 KMP 算法的核心。当匹配失败时，next 数组告诉我们模式串应该回退到哪个位置继续匹配，而不是从头开始。这是因为 next 数组记录了模式串中每个位置的最长相等前后缀长度，利用这个信息可以避免重复比较。

### 7.4 如何选择前缀表的表示方式

两种前缀表表示方式各有优缺点：
- **原始前缀表**：更直观，容易理解
- **前缀表统一减一**：在代码实现中更简洁，不需要额外的边界检查

选择哪种方式取决于个人偏好和具体应用场景。

## 9. 总结

1. **提高字符串匹配的效率**：时间复杂度从 O(m * n) 降低到 O(m + n)
2. **理解算法设计的思想**：利用已有的信息来优化算法性能
3. **解决实际应用中的字符串匹配问题**：如文本搜索、模式识别等

KMP 算法的难点在于理解 next 数组的构建和回退机制，但通过详细的例子和步骤分解，我们可以逐步掌握其核心原理。希望本笔记能够帮助初学者理解 KMP 算法，并在实际应用中灵活使用。
