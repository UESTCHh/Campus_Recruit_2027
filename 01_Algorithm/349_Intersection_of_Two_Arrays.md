# 算法实战复盘：两个数组的交集 (LeetCode 349)

> **打卡日期**：2026-04-02
> **核心主题**：HashSet、集合操作、时间复杂度优化。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给定两个整数数组 `nums1` 和 `nums2`，返回它们的交集(The intersection of two arrays is defined as the set of elements that are present in both arrays.)。输出结果中的每个元素必须是唯一的，且顺序不限。
* **核心痛点：** 暴力解法是两层 `for` 循环，时间复杂度 $O(nm)$，其中 $n$ 和 $m$ 分别是两个数组的长度。
* **大厂面试限制：** 要求必须在 $O(n + m)$ 的时间复杂度内解决，且输出结果中的元素必须唯一。
* **题目提示：**
  - 1 <= nums1.length, nums2.length <= 1000
  - 0 <= nums1[i], nums2[i] <= 1000

---

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：$O(n + m)$
**推导过程：** 我们首先将第一个数组的元素存入 HashSet，时间复杂度为 $O(n)$。然后遍历第二个数组，对于每个元素，检查它是否在 HashSet 中，时间复杂度为 $O(m)$。因此整体时间复杂度为 $O(n + m)$。

### 2. 空间复杂度：$O(n)$ 或 $O(m)$
**推导过程：** 我们需要一个 HashSet 来存储第一个数组的元素，空间复杂度为 $O(n)$。如果第二个数组的长度比第一个数组小，我们可以选择存储第二个数组的元素，此时空间复杂度为 $O(m)$。因此空间复杂度为 $O(\min(n, m))$。

## 🤔 三、 解题思路分析

### 3.1 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：两层 `for` 循环，时间复杂度 O(nm)
2. **排序 + 双指针法**：对两个数组排序后，使用双指针查找交集，时间复杂度 O(nlogn + mlogm)
3. **哈希表法**：使用哈希表记录元素出现的情况，时间复杂度 O(n + m)

### 3.2 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(nm) | O(1) | 思路简单，不需要额外空间 | 时间复杂度高，对于较大数组效率极低 |
| 排序 + 双指针法 | O(nlogn + mlogm) | O(1) 或 O(logn + logm) | 不需要额外空间，适合内存受限场景 | 排序的时间复杂度较高 |
| 哈希表法 | O(n + m) | O(n) 或 O(m) | 时间复杂度最优，效率极高 | 需要额外空间存储哈希表 |

### 3.3 数据大小对方法选择的影响

根据题目提示，数组长度最大为 1000，元素值范围为 0-1000：

- **暴力法**：对于 n = m = 1000，操作次数会达到 1,000,000，虽然可以通过，但效率不够理想
- **排序 + 双指针法**：排序的时间复杂度为 O(nlogn)，对于 n = 1000，操作次数约为 1000 * 10 = 10,000，效率较好
- **哈希表法**：时间复杂度为 O(n + m)，对于 n = m = 1000，操作次数约为 2000，效率最高

**实际测试验证：**
- 暴力法：在 LeetCode 上可以通过，但执行时间较长
- 排序 + 双指针法：执行时间适中
- 哈希表法：执行时间极短，通常在 0-4ms 之间

### 3.4 哈希表结构的分析

在本题中，我们可以使用以下几种哈希表结构：

| 哈希表结构 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------------|------------|------------|------|------|
| `std::unordered_set` | 平均 O(1) | O(n) | 自动去重，查询速度快 | 底层实现复杂，有一定的常数开销 |
| `std::unordered_map` | 平均 O(1) | O(n) | 可以记录元素出现次数 | 比 `unordered_set` 更复杂，空间开销更大 |
| 布尔数组 | O(1) | O(1) (固定大小) | 访问速度极快，空间使用固定 | 只适用于元素范围已知且较小的情况 |
| `std::set` | O(logn) | O(n) | 自动排序 | 查询和插入时间复杂度较高 |

**选择建议：**
- 对于本题的元素范围（0-1000），使用布尔数组是最高效的选择
- 如果元素范围未知或较大，使用 `std::unordered_set` 是最佳选择
- 如果需要同时记录元素出现次数，使用 `std::unordered_map`
- 只有在需要排序结果时，才考虑使用 `std::set`

## 🎯 四、 核心思想：利用 HashSet 去重和快速查询

### 4.1 核心原理

1. **去重：** 使用 HashSet 存储第一个数组的元素，自动去重，确保每个元素只出现一次。
2. **快速查询：** 遍历第二个数组，对于每个元素，使用 HashSet 的 `contains` 方法（平均时间复杂度 $O(1)$）检查它是否在第一个数组中出现过。
3. **结果收集：** 将同时存在于两个数组中的元素添加到结果集合中，再次使用 HashSet 确保结果中的元素唯一。

### 4.2 示例演示

以 `nums1 = [1, 2, 2, 1], nums2 = [2, 2]` 为例：

1. **去重：** 将 nums1 的元素存入 HashSet，得到 `set1 = {1, 2}`
2. **快速查询：** 遍历 nums2 中的每个元素：
   - 第一个元素 2：在 set1 中存在，添加到结果集合
   - 第二个元素 2：在 set1 中存在，但由于结果集合使用 HashSet，不会重复添加
3. **结果收集：** 最终结果集合为 `{2}`，转换为向量 `[2]`

### 4.3 算法执行流程

1. **初始化：** 创建一个 HashSet 用于存储第一个数组的元素
2. **填充 HashSet：** 遍历第一个数组，将所有元素添加到 HashSet 中（自动去重）
3. **创建结果集合：** 创建一个 HashSet 用于存储结果
4. **遍历第二个数组：** 对于每个元素，检查是否在第一个 HashSet 中存在
5. **添加到结果：** 如果存在，将元素添加到结果集合中
6. **转换结果：** 将结果集合转换为向量并返回

### 4.4 优化策略

- **选择较小的数组：** 为了减少内存使用，应该选择将较小的数组存入 HashSet
- **使用 erase 方法：** 利用 `erase` 方法同时完成查询和去重操作，代码更简洁
- **利用布尔数组：** 当元素范围已知且较小时，使用布尔数组代替 HashSet 提高性能

## 💻 五、 核心代码

### 5.1 HashSet 实现

```cpp
#include <vector>
#include <unordered_set>

class Solution {
public:
    std::vector<int> intersection(std::vector<int>& nums1, std::vector<int>& nums2) {
        // 1. 将第一个数组的元素存入 HashSet，自动去重
        std::unordered_set<int> set1(nums1.begin(), nums1.end());
        std::unordered_set<int> result_set;
        
        // 2. 遍历第二个数组，检查元素是否在 set1 中
        for (int num : nums2) {
            if (set1.count(num)) {
                result_set.insert(num);
            }
        }
        
        // 3. 将结果集合转换为向量
        return std::vector<int>(result_set.begin(), result_set.end());
    }
};
```

### 5.2 更简洁的 HashSet 实现

```cpp
#include <vector>
#include <unordered_set>

class Solution {
public:
    std::vector<int> intersection(std::vector<int>& nums1, std::vector<int>& nums2) {
        std::unordered_set<int> set1(nums1.begin(), nums1.end());
        std::vector<int> result;
        
        for (int num : nums2) {
            if (set1.erase(num)) { // 如果元素存在，erase 会返回 1，同时删除该元素避免重复
                result.push_back(num);
            }
        }
        
        return result;
    }
};
```

### 5.3 布尔数组实现（针对元素范围 0-1000）

```cpp
#include <vector>

class Solution {
public:
    std::vector<int> intersection(std::vector<int>& nums1, std::vector<int>& nums2) {
        // 使用布尔数组标记元素是否出现
        bool hash[1001] = {false};
        std::vector<int> result;
        
        // 遍历第一个数组，标记出现的元素
        for (int num : nums1) {
            hash[num] = true;
        }
        
        // 遍历第二个数组，检查元素是否出现过
        for (int num : nums2) {
            if (hash[num]) {
                result.push_back(num);
                hash[num] = false; // 标记为 false，避免重复添加
            }
        }
        
        return result;
    }
};
```

### 5.4 排序 + 双指针法实现

```cpp
#include <vector>
#include <algorithm>

class Solution {
public:
    std::vector<int> intersection(std::vector<int>& nums1, std::vector<int>& nums2) {
        // 对两个数组进行排序
        std::sort(nums1.begin(), nums1.end());
        std::sort(nums2.begin(), nums2.end());
        
        std::vector<int> result;
        int i = 0, j = 0;
        
        // 使用双指针查找交集
        while (i < nums1.size() && j < nums2.size()) {
            if (nums1[i] == nums2[j]) {
                // 避免重复添加
                if (result.empty() || result.back() != nums1[i]) {
                    result.push_back(nums1[i]);
                }
                i++;
                j++;
            } else if (nums1[i] < nums2[j]) {
                i++; // 较小的元素指针右移
            } else {
                j++; // 较小的元素指针右移
            }
        }
        
        return result;
    }
};
```

**排序 + 双指针法解释：**
1. **排序**：首先对两个数组进行排序，时间复杂度为 O(nlogn + mlogm)
2. **初始化指针**：使用两个指针 i 和 j 分别指向两个数组的起始位置
3. **双指针遍历**：
   - 如果两个指针指向的元素相等，说明找到了一个交集元素
   - 检查结果集合是否为空或最后一个元素是否与当前元素不同，避免重复添加
   - 同时移动两个指针
   - 如果 nums1[i] < nums2[j]，移动 i 指针
   - 如果 nums1[i] > nums2[j]，移动 j 指针
4. **返回结果**：遍历结束后，返回结果集合

### 5.5 暴力法实现

```cpp
#include <vector>
#include <unordered_set>

class Solution {
public:
    std::vector<int> intersection(std::vector<int>& nums1, std::vector<int>& nums2) {
        std::unordered_set<int> result_set; // 使用 set 自动去重
        
        // 暴力遍历两个数组
        for (int i = 0; i < nums1.size(); i++) {
            for (int j = 0; j < nums2.size(); j++) {
                if (nums1[i] == nums2[j]) {
                    result_set.insert(nums1[i]); // 将交集元素添加到 set 中
                }
            }
        }
        
        // 将 set 转换为 vector
        return std::vector<int>(result_set.begin(), result_set.end());
    }
};
```

**暴力法解释：**
1. **初始化结果集合**：使用 unordered_set 来存储结果，自动去重
2. **双层循环**：使用两层 for 循环遍历两个数组的所有可能组合
3. **检查交集**：如果找到相同的元素，将其添加到结果集合中
4. **转换结果**：将 unordered_set 转换为 vector 并返回

**暴力法性能分析：**
- 时间复杂度：O(nm)，其中 n 和 m 分别是两个数组的长度
- 空间复杂度：O(k)，其中 k 是交集元素的数量
- 适用场景：数据规模很小，对性能要求不高的情况

## 💡 六、 面试直击灵魂的防暴击拷问
* **灵魂拷问一：为什么使用 HashSet 而不是数组？**
    * **绝杀回答：**"HashSet 的查询时间复杂度是 $O(1)$，而数组的查询时间复杂度是 $O(n)$。使用 HashSet 可以将时间复杂度从 $O(nm)$ 降到 $O(n + m)$，显著提高效率。此外，HashSet 还能自动去重，确保结果中的元素唯一。"

* **灵魂拷问二：如果数组中的元素范围很小，比如在 0-1000 之间，有没有更高效的方法？**
    * **绝杀回答：**"如果元素范围很小，可以使用布尔数组来代替 HashSet。例如，创建一个大小为 1001 的布尔数组，标记第一个数组中出现的元素，然后遍历第二个数组检查对应位置是否为 true。这种方法的时间复杂度仍然是 $O(n + m)$，但空间复杂度可能更低，且访问速度更快。"

* **灵魂拷问三：如何处理大数据量的情况？**
    * **绝杀回答：**"对于大数据量的情况，HashSet 仍然是合适的选择，因为它的时间复杂度是线性的。但需要注意内存使用，如果其中一个数组非常大，我们应该选择存储较小的那个数组，以减少内存消耗。此外，在 C++ 中，`std::unordered_set` 的实现通常会自动处理哈希冲突，确保在大数据量下仍然保持良好的性能。"

* **灵魂拷问四：输出结果的顺序有要求吗？**
    * **绝杀回答：**"题目要求输出结果的顺序不限，因此我们不需要对结果进行排序。如果需要按特定顺序输出，比如升序，我们可以在收集完结果后对向量进行排序，时间复杂度会增加到 $O((n + m) \log (n + m))$。"
* **💥 灵魂拷问五（大厂进阶）：如果题目明确告诉你，数组里的数字大小只在 0 到 1000 之间，你还会用 std::unordered_set 吗？**
    * **绝杀回答：**"坚决不用！虽然 unordered_set 的平均查询也是 $O(1)$，但它的底层需要计算哈希值、维护哈希桶和处理冲突产生的链表，常数级的内存和时间开销非常大。如果已知数据范围在 0-1000，我直接开辟一个 bool hash[1005] = {false}; 的固定大小数组。把数字的数值直接作为数组的下标。数组的内存映射速度在底层只需要几条汇编指令，绝对碾压任何复杂的哈希容器。这才是工业级极致压榨性能的正确做法。"

## 📚 七、 总结

本题是哈希表的经典应用，通过使用 HashSet 或布尔数组，我们可以在 $O(n + m)$ 的时间复杂度内高效解决问题。

### 核心要点：
1. **去重**：利用 HashSet 的特性自动去重，确保结果中的元素唯一
2. **快速查询**：利用 HashSet 的平均 $O(1)$ 时间复杂度查询，提高效率
3. **数据范围**：当元素范围已知且较小时，使用布尔数组可以获得更高的性能
4. **内存优化**：选择将较小的数组存入 HashSet，减少内存使用
5. **代码简洁性**：利用 `erase` 方法可以同时完成查询和去重操作，使代码更简洁

### 扩展思考：
- **哈希表的应用**：哈希表在解决查找、去重、统计等问题时非常高效，是算法面试中的常用工具
- **时间复杂度与空间复杂度的权衡**：在选择算法时，需要根据具体情况权衡时间和空间复杂度
- **数据结构的选择**：不同的数据结构适用于不同的场景，需要根据问题特点选择合适的数据结构
- **性能优化**：在实际开发中，需要根据数据规模和特点选择最适合的实现方式，以达到最佳性能

### 方法比较：

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| 暴力法 | O(nm) | O(1) | 数据规模很小，对性能要求不高 |
| 排序 + 双指针法 | O(nlogn + mlogm) | O(1) 或 O(logn + logm) | 内存受限，或需要排序结果 |
| HashSet 法 | O(n + m) | O(n) 或 O(m) | 数据规模较大，需要高效处理 |
| 布尔数组法 | O(n + m) | O(1) (固定大小) | 元素范围已知且较小 |
