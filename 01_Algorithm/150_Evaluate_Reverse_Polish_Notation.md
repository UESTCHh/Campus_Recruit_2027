## 一、 题目描述与核心要求

### 1. 题目原文
给你一个字符串数组 tokens ，表示一个根据 逆波兰表示法 表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

注意：

- 有效的算符为 '+'、'-'、'*' 和 '/' 。
- 每个操作数（运算对象）都可以是一个整数或者另一个表达式。
- 两个整数之间的除法总是 向零截断 。
- 表达式中不含除零运算。
- 输入是一个根据逆波兰表示法表示的算术表达式。
- 答案及所有中间计算结果可以用 32 位 整数表示。

### 2. 示例说明
**示例 1**：
输入：tokens = ["2","1","+","3","*"]
输出：9
解释：该算式转化为常见的中缀算术表达式为：((2 + 1) * 3) = 9

**示例 2**：
输入：tokens = ["4","13","5","/","+"]
输出：6
解释：该算式转化为常见的中缀算术表达式为：(4 + (13 / 5)) = 6

**示例 3**：
输入：tokens = ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]
输出：22
解释：该算式转化为常见的中缀算术表达式为：
  ((10 * (6 / ((9 + 3) * -11))) + 17) + 5
= ((10 * (6 / (12 * -11))) + 17) + 5
= ((10 * (6 / -132)) + 17) + 5
= ((10 * 0) + 17) + 5
= (0 + 17) + 5
= 17 + 5
= 22

### 3. 约束条件
- 1 <= tokens.length <= 10^4
- tokens[i] 是一个算符（"+"、"-"、"*" 或 "/"），或是在范围 [-200, 200] 内的一个整数

### 4. 逆波兰表达式简介
- **逆波兰表达式**：是一种后缀表达式，所谓后缀就是指算符写在后面
- **中缀表达式**：平常使用的算式，如 (1 + 2) * (3 + 4)
- **逆波兰表达式写法**：((1 2 +) (3 4 +) *)
- **优点**：
  1. 去掉括号后表达式无歧义
  2. 适合用栈操作运算：遇到数字则入栈；遇到算符则取出栈顶两个数字进行计算，并将结果压入栈中

## 二、 解题思路分析

### 1. 问题分析
- **核心需求**：计算逆波兰表达式的值
- **输入特点**：字符串数组，包含数字和运算符
- **操作特点**：运算符在操作数之后，符合后缀表达式的特性
- **计算规则**：遇到运算符时，需要取出栈顶的两个元素进行运算，注意运算顺序

### 2. 初学者想法
- **暴力解法**：直接模拟计算过程，遇到数字就保存，遇到运算符就计算
- **数据结构选择**：考虑到需要后进先出的操作特性，栈是最适合的数据结构
- **运算顺序**：注意运算符的左右操作数顺序，特别是减法和除法

### 3. 可能的解法

#### 3.1 方法一：栈
- **思路**：使用栈来存储操作数，遇到运算符时取出栈顶两个元素进行计算，将结果压回栈中
  - 遍历 tokens 数组
  - 如果当前元素是数字，转换为整数并入栈
  - 如果当前元素是运算符，取出栈顶两个元素进行运算
  - 将运算结果压回栈中
  - 最后栈中只剩下一个元素，即为表达式的结果
- **优势**：实现简单，符合逆波兰表达式的计算逻辑
- **劣势**：需要额外的栈空间

#### 3.2 方法二：递归
- **思路**：递归处理表达式，从后向前遍历
  - 如果当前元素是数字，返回其值
  - 如果当前元素是运算符，递归计算其左右操作数，然后进行运算
- **优势**：代码简洁，逻辑清晰
- **劣势**：对于长表达式，可能导致栈溢出

#### 3.3 方法三：数组模拟栈
- **思路**：使用数组来模拟栈的操作
  - 维护一个数组和一个栈顶指针
  - 遇到数字时，将数字添加到数组末尾
  - 遇到运算符时，根据栈顶指针取出元素进行运算
  - 更新栈顶指针
- **优势**：避免了栈的动态分配开销，可能更高效
- **劣势**：需要手动管理栈顶指针

### 4. 解法选择
- **首选方法**：栈
- **选择理由**：
  - 栈的后进先出特性与逆波兰表达式的计算逻辑完美匹配
  - 实现简单直观，易于理解
  - 时间复杂度为 O(n)，空间复杂度为 O(n)，效率较高
  - 递归方法可能导致栈溢出，数组模拟栈增加了实现复杂度

## 三、 核心代码实现

### 1. 方法一：栈

```cpp
class Solution {
public:
    int evalRPN(vector<string>& tokens) {
        stack<int> st;
        for (string& token : tokens) {
            if (token == "+" || token == "-" || token == "*" || token == "/") {
                int y = st.top();
                st.pop();
                int x = st.top();
                st.pop();
                if (token == "+") {
                    st.push(x + y);
                } else if (token == "-") {
                    st.push(x - y);
                } else if (token == "*") {
                    st.push(x * y);
                } else if (token == "/") {
                    st.push(x / y);
                }
            } else {
                st.push(stoi(token));
            }
        }
        return st.top();
    }
};
```

**代码解析**：
- **遍历 tokens 数组**：逐个处理每个 token
- **判断是否为运算符**：如果是运算符，取出栈顶两个元素
- **注意运算顺序**：先取出的是右操作数，后取出的是左操作数
- **执行运算**：根据运算符类型执行相应的运算
- **结果入栈**：将运算结果压回栈中
- **返回结果**：最后栈中只剩下一个元素，即为表达式的结果

### 2. 方法二：递归

```cpp
class Solution {
public:
    int evalRPN(vector<string>& tokens) {
        int index = tokens.size() - 1;
        return helper(tokens, index);
    }
    
    int helper(vector<string>& tokens, int& index) {
        string token = tokens[index];
        if (token == "+" || token == "-" || token == "*" || token == "/") {
            index--;
            int right = helper(tokens, index);
            index--;
            int left = helper(tokens, index);
            if (token == "+") {
                return left + right;
            } else if (token == "-") {
                return left - right;
            } else if (token == "*") {
                return left * right;
            } else {
                return left / right;
            }
        } else {
            return stoi(token);
        }
    }
};
```

**代码解析**：
- **从后向前遍历**：使用引用传递 index，从数组末尾开始处理
- **递归处理**：如果当前 token 是运算符，递归计算左右操作数
- **注意运算顺序**：先计算右操作数，再计算左操作数
- **返回结果**：对于数字直接返回其值，对于运算符返回运算结果

### 3. 方法三：数组模拟栈

```cpp
class Solution {
public:
    int evalRPN(vector<string>& tokens) {
        vector<int> stack;
        for (string& token : tokens) {
            if (token == "+" || token == "-" || token == "*" || token == "/") {
                int y = stack.back();
                stack.pop_back();
                int x = stack.back();
                stack.pop_back();
                if (token == "+") {
                    stack.push_back(x + y);
                } else if (token == "-") {
                    stack.push_back(x - y);
                } else if (token == "*") {
                    stack.push_back(x * y);
                } else if (token == "/") {
                    stack.push_back(x / y);
                }
            } else {
                stack.push_back(stoi(token));
            }
        }
        return stack.back();
    }
};
```

**代码解析**：
- **使用 vector 模拟栈**：vector 的 back() 和 pop_back() 方法可以模拟栈的操作
- **遍历 tokens 数组**：逐个处理每个 token
- **判断是否为运算符**：如果是运算符，取出 vector 末尾的两个元素
- **执行运算**：根据运算符类型执行相应的运算
- **结果入栈**：将运算结果添加到 vector 末尾
- **返回结果**：最后 vector 中只剩下一个元素，即为表达式的结果

### 4. 方法四：使用匿名函数的简洁写法

```cpp
class Solution {
public:
    int evalRPN(vector<string>& tokens) {
        unordered_map<string, function<int(int, int)>> map = {
            {"+", [](int a, int b) { return a + b; }},
            {"-", [](int a, int b) { return a - b; }},
            {"*", [](int a, int b) { return a * b; }},
            {"/", [](int a, int b) { return a / b; }}
        };
        stack<int> stk;
        for (string& s : tokens) {
            if (map.count(s)) {
                int right = stk.top();
                stk.pop();
                int left = stk.top();
                stk.pop();
                stk.push(map[s](left, right));
            } else {
                stk.push(stoi(s));
            }
        }
        return stk.top();
    }
};
```

**代码解析**：
- **使用 unordered_map 存储运算符**：将运算符映射到对应的 lambda 函数
- **使用 function<int(int, int)> 类型**：允许存储任何可调用对象（函数指针、lambda、函数对象）
- **lambda 表达式**：使用 lambda 表达式定义四个基本运算，代码简洁直观
- **map.count(s) 判断**：检查当前 token 是否为运算符
- **执行运算**：调用 map 中存储的函数对象执行运算

**优势**：
- 代码更加简洁，减少了重复的 if-else 判断
- 运算符和运算逻辑分离，便于维护和扩展
- 使用 lambda 表达式，使代码更具函数式编程风格

### 5. 方法五：用 C++20 的 contains 代替 count

```cpp
class Solution {
public:
    int evalRPN(vector<string> &a) {
        const int n = a.size();
        unordered_map<string, function<int(int, int)>> dict{
            {"+", plus<int>()}, {"-", minus<int>()}, {"*", multiplies<int>()}, {"/", divides<int>()}
        };
        vector<int> sta;
        for (auto &x : a) {
            if (dict.contains(x)) {
                int op2 = sta.back();
                sta.pop_back();
                int op1 = sta.back();
                sta.pop_back();
                sta.push_back(dict[x](op1, op2));
            } else {
                sta.push_back(stoi(x));
            }
        }
        return sta[0];
    }
};
```

**代码解析**：
- **使用 plus/minus/multiplies/divides 函数对象**：C++ 标准库提供的函数对象，比 lambda 更简洁
- **使用 contains 替代 count**：C++20 引入的新方法，语义更清晰，返回 bool 类型
- **vector 模拟栈**：使用 vector 的 back()、pop_back()、push_back() 方法模拟栈操作
- **dict.contains(x)**：判断当前 token 是否为运算符，比 count(x) > 0 更简洁

**优势**：
- 使用标准库函数对象 plus/minus/multiplies/divides，代码更简洁

### 6. 方法六：使用 unordered_map + lambda 表达式的简洁写法

```cpp
class Solution {
public:
    int evalRPN(vector<string>& tokens) {
        unordered_map<string,function<int(int,int)>> map={
            {"+",[](int a,int b){return a+b;}},
            {"-",[](int a,int b){return a-b;}},
            {"*",[](int a,int b){return a*b;}},
            {"/",[](int a,int b){return a/b;}}
        } ;
        stack<int> st;
        for(string& s:tokens)
        {
            if(map.count(s))
            {
                int right=st.top();st.pop();
                int left=st.top();st.pop();
                st.push(map[s](left,right));
            }
            else
                st.push(stoi(s));
        }
        return st.top();
    }
};
```

**代码解析**：
- **unordered_map 存储运算**：将四个运算符映射到对应的 lambda 表达式
- **简洁的初始化方式**：使用花括号初始化 map，代码更紧凑
- **单行栈操作**：将栈的 pop 和取值操作合并在一行，代码更简洁
- **map.count(s) 判断**：检查当前 token 是否为运算符

**优势**：
- 代码非常简洁，运算符处理逻辑清晰
- 使用 lambda 表达式定义运算，函数式风格
- 整体代码行数少，易于阅读

## 四、 时间复杂度与空间复杂度分析

### 1. 方法一：栈
- **时间复杂度**：O(n)，其中 n 是 tokens 数组的长度。每个元素只需要入栈和出栈各一次，因此时间复杂度为线性。
- **空间复杂度**：O(n)，最坏情况下，所有元素都是操作数，栈的大小为 n。

### 2. 方法二：递归
- **时间复杂度**：O(n)，每个元素只处理一次，因此时间复杂度为线性。
- **空间复杂度**：O(n)，递归调用栈的深度最坏情况下为 n。

### 3. 方法三：数组模拟栈
- **时间复杂度**：O(n)，每个元素只需要入栈和出栈各一次，因此时间复杂度为线性。
- **空间复杂度**：O(n)，最坏情况下，所有元素都是操作数，数组的大小为 n。

## 六、 易错点提示

### 1. 运算顺序问题
- **问题**：处理运算符时，操作数的顺序容易搞反
- **解决方案**：记住先弹出的是右操作数，后弹出的是左操作数，例如 x - y 而不是 y - x

### 2. 除法向零截断问题
- **问题**：C++ 中负数除法的截断方向与题目要求可能不同
- **解决方案**：题目要求向零截断，C++ 中的除法是向零截断的，所以直接使用即可

### 3. 栈为空的问题
- **问题**：在取出操作数时，栈可能为空
- **解决方案**：根据题目描述，输入是有效的逆波兰表达式，所以不需要处理这种情况

### 4. 数据类型溢出问题
- **问题**：中间计算结果可能超出 int 范围
- **解决方案**：题目说明答案及所有中间计算结果可以用 32 位整数表示，所以不需要处理溢出

### 5. 字符串转换问题
- **问题**：stoi() 函数可能抛出异常
- **解决方案**：根据题目描述，输入是有效的，所以不需要处理异常

## 九、 算法执行流程

### 1. 栈方法执行流程
1. **初始化栈**：创建一个空栈
2. **遍历 tokens 数组**：
   - 对于每个 token：
     - 如果是数字，转换为整数并入栈
     - 如果是运算符：
       - 弹出栈顶元素作为右操作数
       - 弹出栈顶元素作为左操作数
       - 根据运算符类型执行运算
       - 将运算结果压入栈中
3. **返回结果**：栈中只剩下一个元素，返回该元素

### 2. 递归方法执行流程
1. **初始化索引**：从数组末尾开始
2. **递归处理**：
   - 如果当前 token 是数字，返回其值
   - 如果当前 token 是运算符：
     - 索引减 1，递归计算右操作数
     - 索引减 1，递归计算左操作数
     - 根据运算符类型执行运算
     - 返回运算结果
3. **返回结果**：返回递归调用的结果

### 3. 数组模拟栈方法执行流程
1. **初始化数组**：创建一个空数组
2. **遍历 tokens 数组**：
   - 对于每个 token：
     - 如果是数字，转换为整数并添加到数组末尾
     - 如果是运算符：
       - 取出数组末尾元素作为右操作数
       - 取出数组末尾元素作为左操作数
       - 根据运算符类型执行运算
       - 将运算结果添加到数组末尾
3. **返回结果**：数组中只剩下一个元素，返回该元素

## 十、 面试相关八股问题

### 1. 基础问题

#### 1.1 什么是逆波兰表达式？
**答案**：逆波兰表达式是一种后缀表达式，运算符写在操作数之后。它的优点是去掉括号后表达式无歧义，适合用栈操作运算。

#### 1.2 如何计算逆波兰表达式的值？
**答案**：使用栈来计算。遇到数字则入栈；遇到运算符则取出栈顶两个数字进行计算，并将结果压入栈中。最后栈中只剩下一个元素，即为表达式的结果。

#### 1.3 为什么逆波兰表达式适合用栈来计算？
**答案**：因为逆波兰表达式的运算符在操作数之后，当遇到运算符时，需要使用最近的两个操作数进行计算，这与栈的后进先出特性完美匹配。

### 2. 进阶问题

#### 2.1 处理运算符时，为什么要注意操作数的顺序？
**答案**：因为栈是后进先出的，所以先弹出的是右操作数，后弹出的是左操作数。对于减法和除法这样的非交换运算符，操作数的顺序非常重要。

#### 2.2 如何处理除法向零截断的问题？
**答案**：在 C++ 中，整数除法已经是向零截断的，所以直接使用除法运算符即可。例如，6 / (-132) 的结果是 0，-6 / 132 的结果也是 0。

#### 2.3 如果表达式中包含括号，如何处理？
**答案**：可以先将中缀表达式转换为逆波兰表达式，然后再使用栈进行计算。转换过程也需要使用栈来处理括号和运算符的优先级。

### 3. 算法设计问题

#### 3.1 如何将中缀表达式转换为逆波兰表达式？
**答案**：使用栈来处理运算符的优先级和括号：
- 初始化一个空栈和一个输出队列
- 遍历中缀表达式的每个 token：
  - 如果是数字，添加到输出队列
  - 如果是左括号，压入栈
  - 如果是右括号，弹出栈中的运算符直到遇到左括号，将弹出的运算符添加到输出队列，然后丢弃左括号
  - 如果是运算符，弹出栈中优先级大于或等于当前运算符的运算符，添加到输出队列，然后将当前运算符压入栈
- 弹出栈中剩余的运算符，添加到输出队列
- 输出队列中的元素即为逆波兰表达式

#### 3.2 如何处理浮点数的逆波兰表达式？
**答案**：将栈的类型改为 double，使用 stod() 函数将字符串转换为浮点数，其他逻辑与整数处理相同。

#### 3.3 如何处理大整数的逆波兰表达式？
**答案**：可以使用字符串或大数库来处理大整数，避免溢出问题。

### 4. 实际应用问题

#### 4.1 逆波兰表达式在编译器中有什么应用？
**答案**：编译器在解析和计算表达式时，通常会先将中缀表达式转换为逆波兰表达式，然后再进行计算。这样可以避免处理括号和运算符优先级的复杂性。

#### 4.2 如何实现一个支持变量的逆波兰表达式计算器？
**答案**：需要维护一个变量表，存储变量名和对应的值。当遇到变量名时，从变量表中查找其值并入栈。

#### 4.3 如何优化逆波兰表达式的计算性能？
**答案**：可以考虑以下优化：
- 使用数组模拟栈，避免栈的动态分配开销
- 对于重复出现的子表达式，缓存计算结果
- 对于某些特定的表达式结构，使用并行计算
