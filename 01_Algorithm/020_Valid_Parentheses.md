# 算法实战复盘：有效的括号 (LeetCode 20)

> **打卡日期**：2026-04-15
> **核心主题**：栈、括号匹配、对称匹配。

---

## 📝 一、 题目描述与核心要求

* **题目内容：** 给定一个只包括 '('，')'，'{'，'}'，'['，']' 的字符串 s ，判断字符串是否有效。有效字符串需满足：左括号必须用相同类型的右括号闭合；左括号必须以正确的顺序闭合；每个右括号都有一个对应的相同类型的左括号。
* **核心痛点：** 如何高效地判断括号的匹配情况，避免暴力解法的高时间复杂度。
* **大厂面试限制：** 要求时间复杂度 O(n)，空间复杂度 O(n)。
* **题目提示：**
  - 1 <= s.length <= 10^4
  - s 仅由括号 '()[]{}' 组成

### 示例

| 示例 | 输入 | 输出 | 解释 |
|------|------|------|------|
| 1 | "()" | true | 有效 |
| 2 | "()[]{}" | true | 有效 |
| 3 | "(]" | false | 类型不匹配 |
| 4 | "([])" | true | 有效 |
| 5 | "([)]" | false | 顺序不正确 |

## 📊 二、 复杂度深度剖析

### 1. 时间复杂度：O(n)
**推导过程：** 遍历字符串需要 O(n) 时间，每个字符的处理（入栈或出栈）都是 O(1) 时间，因此总时间复杂度为 O(n)。

### 2. 空间复杂度：O(n)
**推导过程：** 栈的大小最多为 n/2（当所有字符都是左括号时），因此空间复杂度为 O(n)。

## 🤔 三、 解题思路分析

### 3.1 初学者的常见思路

当面对这个问题时，初学者可能会想到以下几种方法：

1. **暴力法**：通过不断删除匹配的括号对，直到字符串为空或无法继续删除
2. **栈解法**：利用栈的后进先出特性，实现括号的匹配

### 3.2 各种方法的优缺点分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| 暴力法 | O(n²) | O(1) | 思路简单，不需要额外空间 | 效率低，每次删除操作都需要移动字符串元素 |
| 栈解法 | O(n) | O(n) | 效率高，操作简单 | 需要额外的栈空间 |

### 3.3 数据大小对方法选择的影响

根据题目提示，字符串长度最大为 10^4：
- 暴力法的时间复杂度为 O(n²)，对于 n=10^4，操作次数达到 10^8，可能会超时
- 栈解法的时间复杂度为 O(n)，对于 n=10^4，操作次数为 10^4，完全可以接受

因此，栈解法是本题的最优选择。

## 🛠️ 四、 核心代码实现

### 4.1 方法一：暴力法

**思路**：通过不断删除匹配的括号对，直到字符串为空或无法继续删除。

**实现**：

```cpp
class Solution {
public:
    bool isValid(string s) {
        // 不断删除匹配的括号对
        while (true) {
            int length = s.length();
            // 删除所有"()"对
            size_t pos = s.find("()");
            while (pos != string::npos) {
                s.erase(pos, 2);
                pos = s.find("()");
            }
            // 删除所有"{}"对
            pos = s.find("{}");
            while (pos != string::npos) {
                s.erase(pos, 2);
                pos = s.find("{}");
            }
            // 删除所有"[]"对
            pos = s.find("[]");
            while (pos != string::npos) {
                s.erase(pos, 2);
                pos = s.find("[]");
            }
            // 如果字符串长度没有变化，说明没有更多匹配的括号对
            if (s.length() == length) {
                // 如果最终字符串为空，则有效，否则无效
                return length == 0;
            }
        }
    }
};
```

**复杂度分析**：
- **时间复杂度**：O(n²)，最坏情况下，每次只删除一对括号，需要n/2次遍历，每次遍历需要O(n)时间
- **空间复杂度**：O(1)，不需要额外空间

### 4.2 方法二：栈 + 哈希表

```cpp
class Solution {
public:
    bool isValid(string s) {
        // 如果字符串长度为奇数，一定无效
        if (s.length() % 2 != 0) return false;
        
        // 建立括号映射表
        unordered_map<char, char> mp = {{'(', ')'}, {'[', ']'}, {'{', '}'}};
        stack<char> st;
        
        // 遍历字符串
        for (char c : s) {
            // 如果是左括号，将对应的右括号入栈
            if (mp.count(c)) {
                st.push(mp[c]);
            } else {
                // 如果是右括号，检查栈是否为空或栈顶元素是否匹配
                if (st.empty() || st.top() != c) return false;
                // 匹配成功，弹出栈顶元素
                st.pop();
            }
        }
        
        // 所有括号必须匹配完毕
        return st.empty();
    }
};
```

### 4.3 方法三：栈 + switch语句

```cpp
class Solution {
public:
    bool isValid(string s) {
        // 如果字符串长度为奇数，一定无效
        if (s.length() % 2 != 0) return false;
        
        stack<char> st;
        
        // 遍历字符串
        for (char c : s) {
            // 处理左括号
            switch (c) {
                case '(':
                    st.push(')');
                    break;
                case '[':
                    st.push(']');
                    break;
                case '{':
                    st.push('}');
                    break;
                // 处理右括号
                default:
                    // 检查栈是否为空或栈顶元素是否匹配
                    if (st.empty() || st.top() != c) return false;
                    // 匹配成功，弹出栈顶元素
                    st.pop();
                    break;
            }
        }
        
        // 所有括号必须匹配完毕
        return st.empty();
    }
};
```

    }

## 📋 五、 算法执行流程

以方法一为例，详细说明算法的执行流程：

1. **输入检查**：首先检查字符串长度是否为奇数，如果是则直接返回false
2. **初始化**：创建一个哈希表存储括号映射关系，创建一个空栈
3. **遍历字符串**：
   - 如果当前字符是左括号，将对应的右括号入栈
   - 如果当前字符是右括号：
     - 检查栈是否为空，如果为空则返回false
     - 检查栈顶元素是否与当前右括号匹配，如果不匹配则返回false
     - 如果匹配，弹出栈顶元素
4. **最终检查**：遍历结束后，检查栈是否为空，如果为空则返回true，否则返回false

### 执行示例

#### 示例1：输入 "()"
1. 检查长度为2，偶数
2. 遍历到 '('，将 ')' 入栈
3. 遍历到 ')'，栈不为空，栈顶元素是 ')'，匹配
4. 弹出栈顶元素，栈为空
5. 遍历结束，栈为空，返回true

#### 示例2：输入 "([)]"
1. 检查长度为4，偶数
2. 遍历到 '('，将 ')' 入栈
3. 遍历到 '['，将 ']' 入栈
4. 遍历到 ')'，栈不为空，栈顶元素是 ']'，不匹配
5. 返回false

| 边界情况 | 处理策略 | 实现方法 |
|----------|----------|----------|
| 空字符串 | 认为是有效的 | 遍历结束后栈为空，返回true |
| 长度为奇数的字符串 | 一定无效 | 在代码开始时检查字符串长度是否为奇数 |
| 只有左括号的字符串 | 遍历结束后栈不为空，返回false | 在遍历结束后检查栈是否为空 |
| 只有右括号的字符串 | 第一次遇到右括号时栈为空，返回false | 在遇到右括号时检查栈是否为空 |
| 括号顺序不正确的字符串 | 遇到右括号时栈顶元素不匹配，返回false | 在遇到右括号时检查栈顶元素是否与当前右括号匹配 |

## ⚠️ 八、 常见错误与注意事项

### 8.1 常见错误

| 错误类型 | 错误表现 | 避免方法 |
|----------|----------|----------|
| 忘记检查字符串长度为奇数的情况 | 对长度为奇数的字符串进行不必要的处理 | 在代码开始时检查字符串长度是否为奇数 |
| 栈为空时尝试弹出元素 | 运行时错误（栈下溢） | 在遇到右括号时，先检查栈是否为空 |
| 括号映射错误 | 匹配逻辑错误 | 确保括号映射表正确，左括号对应正确的右括号 |
| 遍历结束后忘记检查栈是否为空 | 对于只有左括号的字符串返回true | 在遍历结束后检查栈是否为空 |

### 8.2 注意事项

1. **栈的选择**：C++中使用`stack`容器，它提供了`push`、`pop`、`top`和`empty`等方法
2. **括号匹配的顺序**：使用栈的后进先出特性来保证正确的匹配顺序
3. **效率优化**：对于大字符串，栈解法的时间复杂度为O(n)，效率很高
4. **代码可读性**：使用哈希表或switch语句可以使代码更加清晰易读

## 🚀 九、 优化与拓展

### 9.1 性能优化

- **内存优化**：对于括号类型较少的情况，可以使用switch语句或if-else语句替代哈希表，减少内存开销
- **时间优化**：在遍历过程中，一旦发现不匹配的情况，立即返回false，避免不必要的处理

### 10.2 常见追问及回答策略

#### 追问1：为什么使用栈来解决这个问题？
**回答**：栈的后进先出（LIFO）特性非常适合处理括号匹配问题。当遇到左括号时，我们将其对应的右括号入栈；当遇到右括号时，我们检查栈顶元素是否与当前右括号匹配。这种方式可以确保括号以正确的顺序闭合。

#### 追问2：如果字符串中包含其他字符，如何处理？
**回答**：如果字符串中包含其他字符，我们可以在遍历过程中忽略这些字符，只处理括号字符。具体来说，当遇到非括号字符时，直接跳过，继续处理下一个字符。

#### 追问3：如何优化空间复杂度？
**回答**：对于这个问题，栈解法的空间复杂度已经是最优的，因为在最坏情况下，我们需要存储所有的左括号。但是，我们可以使用一个计数器来替代栈，适用于只有一种括号类型的情况。例如，对于只有'('和')'的情况，我们可以使用一个计数器，遇到'('时加1，遇到')'时减1，如果计数器为负数则返回false，最后检查计数器是否为0。

#### 追问4：如何处理嵌套深度非常深的括号？
**回答**：栈解法可以处理任意深度的括号嵌套，因为栈的大小会随着嵌套深度的增加而增加。只要字符串长度不超过栈的最大容量，就可以正常处理。对于本题的约束条件（长度不超过10^4），栈解法完全可以处理。
