## 一、 题目描述与核心要求

### 1. 题目原文
给出由小写字母组成的字符串 `s`，重复项删除操作会选择两个相邻且相同的字母，并删除它们。

在 `s` 上反复执行重复项删除操作，直到无法继续删除。

在完成所有重复项删除操作后返回最终的字符串。答案保证唯一。

### 2. 示例说明
**示例**：
输入："abbaca"
输出："ca"

**解释**：
例如，在 "abbaca" 中，我们可以删除 "bb" 由于两字母相邻且相同，这是此时唯一可以执行删除操作的重复项。之后我们得到字符串 "aaca"，其中又只有 "aa" 可以执行重复项删除操作，所以最后的字符串为 "ca"。

### 3. 约束条件
- `1 <= s.length <= 10^5`
- `s` 仅由小写英文字母组成。

## 二、 解题思路分析

### 1. 问题分析
- **核心需求**：删除字符串中所有相邻的重复项，直到无法再删除为止
- **操作特点**：删除操作可能会导致新的相邻重复项出现，需要重复处理
- **数据规模**：字符串长度可达10^5，需要高效的算法

### 2. 初学者想法
- **暴力解法**：重复遍历字符串，每次删除相邻重复项，直到没有重复项为止
  - **思路**：使用一个循环，每次遍历字符串，标记需要删除的位置，然后构建新字符串
  - **问题**：时间复杂度高，对于长字符串可能超时
  - **改进**：寻找更高效的数据结构来处理这种需要频繁删除和比较的操作

- **栈的应用**：初学者可能会想到使用栈来处理括号匹配问题，类似地，这里也可以用栈来处理相邻重复项
  - **思路**：将字符依次入栈，遇到重复项则弹出
  - **优势**：操作直观，容易理解
  - **实现**：需要注意栈的后进先出特性，最后需要反转结果

- **双指针尝试**：初学者可能会尝试使用双指针来原地修改字符串
  - **思路**：左指针记录已处理部分，右指针遍历字符串
  - **挑战**：理解指针移动的逻辑，特别是遇到重复项时的处理

### 3. 可能的解法

#### 3.1 方法一：栈
- **思路**：利用栈的后进先出特性，遍历字符串，将每个字符与栈顶元素比较
  - 如果栈为空或当前字符与栈顶元素不同，将当前字符入栈
  - 如果当前字符与栈顶元素相同，将栈顶元素出栈
- **优势**：操作直观，容易理解，时间复杂度低
- **劣势**：需要额外的栈空间

#### 3.2 方法二：双指针
- **思路**：使用两个指针，左指针指向已处理部分的末尾，右指针遍历字符串
  - 比较右指针指向的字符与左指针指向的字符
  - 如果相同，左指针左移（相当于删除）
  - 如果不同，左指针右移并复制右指针指向的字符
- **优势**：原地操作，不需要额外空间
- **劣势**：逻辑稍复杂，需要理解指针移动的含义

#### 3.3 方法三：递归
- **思路**：递归处理字符串，每次删除所有相邻重复项后，对剩余字符串再次递归处理
- **优势**：代码简洁，逻辑清晰
- **劣势**：对于长字符串可能导致栈溢出，时间复杂度较高

### 4. 解法选择
- **首选方法**：栈或双指针
- **选择理由**：
  - 栈方法直观易懂，实现简单
  - 双指针方法空间效率高，适用于大规模数据
  - 递归方法在处理长字符串时可能出现栈溢出，不推荐

## 三、 核心代码实现

### 1. 方法一：栈

```cpp
class Solution {
public:
    string removeDuplicates(string s) {
        stack<char> st;
        for (char c : s) {
            if (!st.empty() && st.top() == c) {
                st.pop(); // 遇到重复项，弹出栈顶
            } else {
                st.push(c); // 不重复，压入栈
            }
        }
        
        // 从栈中构建结果字符串
        string result;
        while (!st.empty()) {
            result += st.top();
            st.pop();
        }
        reverse(result.begin(), result.end()); // 栈是后进先出，需要反转
        return result;
    }
};
```

**代码解析**：
- **遍历字符串**：逐个字符处理
- **栈操作**：比较当前字符与栈顶元素，决定压入或弹出
- **构建结果**：将栈中元素弹出并构建字符串，注意需要反转

### 2. 方法二：双指针

```cpp
class Solution {
public:
    string removeDuplicates(string s) {
        int left = -1; // 左指针，指向已处理部分的末尾
        for (int right = 0; right < s.size(); right++) {
            if (left >= 0 && s[right] == s[left]) {
                left--; // 遇到重复项，左指针左移（相当于删除）
            } else {
                left++; // 不重复，左指针右移
                s[left] = s[right]; // 复制当前字符
            }
        }
        return s.substr(0, left + 1); // 返回处理后的子串
    }
};
```

**代码解析**：
- **双指针初始化**：左指针初始化为-1，表示空
- **右指针遍历**：逐个字符处理
- **重复项处理**：遇到重复项时左指针左移
- **非重复项处理**：左指针右移并复制字符
- **返回结果**：截取处理后的子串

### 3. 方法三：递归

```cpp
class Solution {
public:
    string removeDuplicates(string s) {
        bool changed = false;
        string result;
        int i = 0;
        while (i < s.size()) {
            if (i < s.size() - 1 && s[i] == s[i + 1]) {
                // 找到重复项，跳过这两个字符
                i += 2;
                changed = true;
            } else {
                // 非重复项，添加到结果
                result += s[i];
                i++;
            }
        }
        // 如果有修改，递归处理结果
        return changed ? removeDuplicates(result) : result;
    }
};
```

**代码解析**：
- **遍历字符串**：寻找相邻重复项
- **处理重复项**：跳过重复的两个字符
- **递归处理**：如果有修改，对结果再次递归处理
- **终止条件**：当没有修改时，返回结果

## 四、 时间复杂度与空间复杂度分析

### 1. 方法一：栈
- **时间复杂度**：O(n)，其中n是字符串的长度。每个字符最多入栈和出栈各一次，因此时间复杂度为线性。
- **空间复杂度**：O(n)，最坏情况下，字符串中没有重复项，栈的大小为n。

### 2. 方法二：双指针
- **时间复杂度**：O(n)，其中n是字符串的长度。右指针遍历字符串一次，左指针最多移动n次，因此时间复杂度为线性。
- **空间复杂度**：O(1)，使用原地操作，不需要额外空间。

### 3. 方法三：递归
- **时间复杂度**：O(n^2)，最坏情况下，每次递归只删除两个字符，需要O(n)次递归，每次递归处理O(n)个字符。
- **空间复杂度**：O(n)，递归调用栈的深度最多为O(n)。

- **问题1**：如何处理字符串中的相邻重复项？
  **答案**：可以使用栈或双指针方法。栈方法通过比较栈顶元素与当前元素，遇到重复则弹出；双指针方法通过原地修改字符串，左指针记录已处理部分，右指针遍历字符串。

- **问题3**：如何处理删除后产生的新重复项？
  **答案**：栈方法和双指针方法都能自动处理这种情况。栈方法通过弹出栈顶元素，双指针方法通过左指针左移，都能重新检查新的相邻元素。

- **问题2**：如何优化算法的时间复杂度？
  **答案**：栈方法和双指针方法的时间复杂度都是O(n)，已经是最优的。可以通过选择更高效的实现方式，如使用数组模拟栈，或优化指针移动的逻辑。

## 七、 易错点提示

### 1. 栈方法
- **栈为空的处理**：在访问栈顶元素前，必须检查栈是否为空
- **结果顺序**：从栈中构建结果时，需要注意栈的后进先出特性，最后需要反转字符串
- **性能问题**：对于长字符串，栈的空间使用可能较大

### 2. 双指针方法
- **指针初始化**：左指针初始化为-1，需要注意边界条件
- **指针移动**：左指针的移动逻辑需要正确理解，特别是遇到重复项时的左移操作
- **结果截取**：返回结果时，需要正确计算子串的长度（left + 1）

### 3. 递归方法
- **栈溢出**：对于长字符串，递归深度可能导致栈溢出
- **性能问题**：递归方法的时间复杂度较高，不适合处理大规模数据
- **字符串拼接**：每次递归都创建新的字符串，可能导致内存使用较高

### 4. 通用易错点
- **边界情况**：空字符串或单字符字符串的处理
- **重复项连续出现**：例如"aaa"，需要连续删除
- **删除后产生新的重复项**：例如"abba"，删除"bb"后变成"aa"，需要再次删除

## 十、 算法执行流程

### 1. 栈方法执行流程
1. **初始化栈**：创建一个空栈
2. **遍历字符串**：
   - 对于每个字符c：
     - 如果栈不为空且栈顶元素等于c，弹出栈顶
     - 否则，将c压入栈
3. **构建结果**：
   - 弹出栈中所有元素，构建字符串
   - 反转字符串（因为栈是后进先出）
4. **返回结果**：返回构建的字符串

### 2. 双指针方法执行流程
1. **初始化指针**：左指针left = -1，右指针right = 0
2. **遍历字符串**：
   - 当right < s.size()：
     - 如果left >= 0且s[right] == s[left]，left--
     - 否则，left++，s[left] = s[right]
     - right++
3. **返回结果**：返回s.substr(0, left + 1)

### 3. 递归方法执行流程
1. **检查字符串**：遍历字符串，寻找相邻重复项
2. **构建新字符串**：
   - 跳过相邻重复项
   - 保留非重复项
3. **递归处理**：
   - 如果有重复项被删除，递归处理新字符串
   - 否则，返回当前字符串

同时，这个问题也展示了如何将抽象的算法思想应用到具体问题中。通过分析问题特点，选择合适的数据结构和算法，我们可以高效地解决各种字符串处理问题。

此外，这个问题也提醒我们要注意边界情况和性能优化。对于大规模数据，选择空间复杂度低的算法尤为重要。同时，理解算法的时间复杂度和空间复杂度分析，有助于我们评估算法的优劣，做出合理的选择。

总之，这个问题虽然看似简单，但包含了丰富的算法思想和实践技巧，是学习数据结构和算法的很好案例。
